## 一、核心渲染技术总览

### 1.1 Canvas（画布）— 底层基础

Canvas 是所有高阶渲染方案的底层宿主。

**技术原理：**
- 通过 `CanvasRenderingContext2D` 或 `WebGL` 进行像素级绘制
- `OffscreenCanvas` 支持 WebWorker 离屏渲染，不阻塞主线程
- `ImageBitmap` API 实现高效的图像数据传输（零拷贝路径）

**核心能力：**
- 2D 绑定图形：矩形、路径、渐变、文字、图像
- WebGL：硬件加速 2D/3D，可对接 Three.js、Pixi.js 等渲染库
- 视频帧提取：与 `<video>` 结合实现帧级处理

**lingobridge 关联场景：**
- 白板（Excalidraw）底层渲染
- 表单控件的手绘风格渲染
- 实时协作时的光标/图形同步层

**主要库生态：**
- Fabric.js：SVG 解析 + Canvas 双向绑定，功能最重
- Konva.js：2D Canvas 库，移动端友好
- p5.js：艺术/可视化方向，偏创意编程
- Pixi.js：WebGL 高性能 2D，适合游戏级渲染
- ZIM：Canvas 组件框架，含无障碍支持

---

### 1.2 Excalidraw（白板）

**技术原理：**
- React + Canvas 架构，完全自定义渲染层
- 手绘风格（rough.js）通过 Canvas 绑定实现「手绘感」几何图形
- 无限画布通过 Viewport 虚拟化和平移/缩放实现
- 状态机驱动：每一笔、每一个图形都是独立 state element

**核心能力：**
- 手绘风格渲染（sketchy 风格，线条抖动、几何不精确）
- 无限画布 + 平移缩放
- 图形绑定（箭头、标签自动吸附）
- 支持 Excalidraw JSON 格式导出（纯 JSON，便于协作同步）
- 支持导出 PNG、SVG、Clipboard
- i18n 国际化（多语言界面）

**协作与存储：**
- PWA 离线支持（localStorage autosave）
- 端到端加密（E2E Encryption）
- 实时协作（CRDT 类算法，支持多人同时编辑）
- `.excalidraw` 文件格式：JSON 结构，存储所有 element 数据

**lingobridge 关联场景：**
- 课堂白板/示意图绘制
- 语言学习的图示解释（关联标注、箭头说明）
- 课件素材的手绘风格图形

**集成方式：**
```bash
npm install react react-dom @excalidraw/excalidraw
```
React 组件，可嵌入到任意 React 应用中。

**知名采用方：**
Google Cloud、Meta、Notion、HackerRank、Obsidian Excalidraw 插件、Replit、Slite

## 二、文档类渲染方案

### 2.1 PDF 预览

**推荐方案：PDF.js（Mozilla）**

**技术原理：**
- 纯 JS 实现，HTML5 Canvas 渲染 PDF 页面
- Worker 线程解析 PDF 二进制（不阻塞 UI）
- 支持页面级懒加载和渐进式渲染
- 提供完整的 Annotations API、Text Layer API

**核心特点：**
- CDN 分发：jsDelivr / unpkg / cdnjs
- NPM 包名：`pdfjs-dist`
- 预构建版本：`build/generic/build/`（含 `pdf.js` + `pdf.worker.js`）
- Chrome 官方扩展（Rob--W 维护）：直接在浏览器中渲染本地 PDF
- 内置调试器：`/internal-viewer/web/debugger.html`

**渲染架构：**
```
PDF Binary → PDF.js Worker（解析）→ Canvas 2D（绘制）
                       ↓
               Text Layer（文本选择）
               Annotations（批注）
```

**性能关键点：**
- 必须在 HTTP/HTTPS 环境下运行（file:// 协议下 Worker 受限）
- 大文件分页渲染，避免一次性加载
- 高 DPI 屏需要 `devicePixelRatio` 补偿

**lingobridge 关联场景：**
- 课件 PDF 预览（语言学习材料、教材章节）
- 课后作业 PDF 展示
- 批注与高亮（Annotations API）

### 2.2 Markdown 预览

**技术方案对比：**

| 方案                    | 渲染方式           | 特点            | 适用场景        |
| --------------------- | -------------- | ------------- | ----------- |
| marked + highlight.js | HTML 字符串注入     | 最轻量，无依赖       | 简单 MD 渲染    |
| markdown-it           | Plugin 驱动      | 支持 GFM + 插件扩展 | 中等复杂度       |
| remark + rehype       | AST 转换树        | 可精确控制 AST     | 复杂格式（脚注、公式） |
| Monaco Editor         | Code Editor 模式 | 写 + 预览一体      | 代码类 MD      |
| Marked + 定制 CSS       | 主题化渲染          | 适配项目视觉        | 统一品牌风格      |

**核心问题：**
- 数学公式渲染：`rehype-katex` / `rehype-mathjax`（LaTeX → HTML）
- Mermaid 图表：`mermaid` 库（独立渲染，不走 HTML）
- 目录生成：remark 插件自动提取 Heading
- 代码高亮：`highlight.js` / `Shiki`（VS Code 同款，高亮质量更好）

**lingobridge 关联场景：**
- 课程说明文档、README
- 学生笔记展示
- 教材 Markdown 化后的预览

### 2.3 Excel（表格数据）

**推荐方案：SheetJS（社区版）**

**技术原理：**
- 纯 JS 解析 `.xlsx` / `.xls`（ECMA-376 标准）
- 输出 JSON 数据结构（行/列/单元格）
- 支持公式求值（`Formula Utilities`）
- 生成时保留样式（字体、颜色、边框、合并单元格）

**核心特点：**
- 数据提取能力强：支持复杂嵌套表格、合并单元格
- 无需 Excel 环境，浏览器端直接运行
- 支持导出生成新的 `.xlsx` 文件
- Pro 版：样式、图像、图表、PivotTable

**渲染层建议（自选）：**
SheetJS 只负责数据层，不提供渲染。搭配以下方案之一：

| 方案                | 说明                                 |
| ----------------- | ---------------------------------- |
| Handsontable      | 类 Excel UI 组件，Angular/React/Vue 支持 |
| AG Grid           | 企业级表格，性能最佳                         |
| ExcelJS           | Node.js 端可读写样式                     |
| 纯 HTML + CSS Grid | 轻量简单场景                             |

**lingobridge 关联场景：**
- 成绩单展示
- 考勤/签到数据
- 班级信息、课表（数据类）

### 2.4 Word（DOCX）

**技术方案：**

| 方案 | 语言 | 说明 |
|------|------|------|
| docx.js | JavaScript | 生成 DOCX（写入），不支持读取预览 |
| mammut.js | JavaScript | 轻量级 DOCX 解析，可渲染为 HTML |
| docx-preview | JavaScript | 读取 + 渲染 DOCX 为 HTML，基于 mammoth |
| Apache POI | Java | 后端方案，需服务器 |
| python-docx | Python | 同上 |

**推荐前端预览方案：`docx-preview`**
- 基于 `mammoth.js`（将 DOCX 转换为 HTML）
- 保留基础样式（段落、字体、列表）
- 复杂样式（页眉页脚、艺术字）无法完美还原，属正常降级
- 支持在 HTML 层面添加批注 overlay

**lingobridge 关联场景：**
- 课件 Word 文档预览
- 作业/论文提交展示
- 试卷文档（需配合 PDF 兜底）

### 2.5 PPTX（幻灯片）

**现状：** PPTX 渲染是所有格式中技术挑战最大的。

**难点分析：**
- PPTX 本质是 OOXML（ECMA-376）打包的 XML 结构，元素极多
- 一个 `.pptx` 解压后有数十个 XML 文件（slide、layout、master、media…）
- 动画效果、切换效果、母版继承关系极其复杂
- 精准还原 PowerPoint 所有特性在浏览器端几乎不可能

**现有技术方案：**

| 方案                      | 类型    | 评价             |
| ----------------------- | ----- | -------------- |
| pptxgenjs               | 生成+渲染 | 仅支持生成，无法读取现有文件 |
| officegen               | 生成    | 同上，Node.js     |
| pptx.js（ nicklassundin） | 读取+渲染 | 社区维护，不活跃，功能有限  |
| Aspose.Slides for JS    | 商业    | 最完整，但付费且重量级    |
| unoconv（LibreOffice）    | 后端转换  | 转换效果好，需服务器环境   |
| pdf.js（转换兜底）            | 降级方案  | 无法交互但保真度最高     |

**实用策略：双轨制**
1. **交互轨**：前端使用轻量库支持基础幻灯片结构预览（标题/正文/列表/图片）
2. **兜底轨**：后端用 LibreOffice / Microsoft Graph API 将 PPTX 转为 PDF，前端用 PDF.js 渲染
3. **编辑轨**：完整编辑功能建议直接嵌入 Microsoft 365 Embed（iFrame 集成）

**lingobridge 关联场景：**
- 课件幻灯片展示（主力：PDF 兜底）
- 课堂讲义（如果需要在线编辑，考虑微软 Embed）

## 三、图片渲染

### 3.1 PNG / JPG

**原生方案：**
- `<img>` 标签：最通用，`loading="lazy"` 支持懒加载
- `Canvas` 绘制：通过 `drawImage()` 实现裁剪、缩放、滤镜
- `ImageBitmap` API：高性能路径，适合 Worker 中处理
- `createObjectURL` + `URL.revokeObjectURL`：内存管理

**进阶需求：**
- 大图缩放：`CSS transform: scale()` + `will-change`
- 瀑布流：CSS Grid / Masonry 布局
- 图片裁剪：`canvas.drawImage()` + `toBlob()`
- WebP 支持：`canvastype === 'webp'` 检测

### 3.2 SVG

**渲染路径（三选一）：**

| 方案 | 说明 |
|------|------|
| 内联 `<svg>` | 直接插入 DOM，最佳性能和交互支持 |
| `<img src=".svg">` | 沙盒化，不可 JS 控制，安全性最佳 |
| `SVG <object>` | 可降级，适合大型独立 SVG 文件 |

**lingobridge 重点关注：**
- Excalidraw 导出 SVG → 在课件中作为素材使用
- 白板协作图形 → SVG 导出 → 课件插入
- 图表（Mermaid / Chart.js 输出）→ SVG → 富文本展示

**安全注意：** SVG 允许内嵌 `<script>`，用户上传的 SVG 必须做 sanitize（用 DOMPurify）

## 四、跨格式统一渲染架构建议

### 4.1 设计原则

```
文件 → 识别类型 → 路由到对应渲染器 → 输出统一容器
```

### 4.2 推荐架构

```
┌─────────────────────────────────────────┐
│           PreviewContainer               │
│  （统一容器：卡片+工具栏+批注层）        │
└────────────┬────────────────────────────┘
             │
    ┌────────┼────────┬──────────┬─────────┐
    ▼        ▼        ▼          ▼         ▼
 PDF.js   Mammoth   SheetJS    Monaco   <img>
( PDF )  ( DOCX )  ( Excel )  ( MD )   ( Image )
    │        │        │          │         │
    ▼        ▼        ▼          ▼         ▼
 Canvas   HTML     HTML/      HTML     <img>/
          DOM      AG Grid   渲染        Canvas
```

### 4.3 各格式优先级建议

| 格式 | 渲染方案 | 优先级 | 备注 |
|------|---------|--------|------|
| PNG/JPG/SVG | `<img>` / 内联 | P0 | 基础必需 |
| PDF | PDF.js | P0 | 课件主力格式 |
| MD | marked + highlight | P0 | 笔记/说明 |
| DOCX | mammoth + docx-preview | P1 | 文档展示 |
| Excel | SheetJS + AG Grid | P1 | 数据展示 |
| PPTX | PDF 兜底 + 微软 Embed | P2 | 幻灯片 |

## 五、lingobridge 专项：Canvas、白板、表单

### 5.1 Canvas 应用层设计

**分层架构（推荐）：**
```
┌──────────────────────────────┐
│   React Components（UI层）    │
├──────────────────────────────┤
│   State Management（状态层）  │  ← Excalidraw 的 element 状态机
├──────────────────────────────┤
│   Canvas Renderer（渲染层）   │  ← 绑定 requestAnimationFrame
├──────────────────────────────┤
│   OffscreenCanvas（Worker）  │  ← 重计算移到 Worker
├──────────────────────────────┤
│   WebSocket Provider（同步层）│  ← 实时协作数据通道
└──────────────────────────────┘
```

### 5.2 Excalidraw 集成要点

**接入模式：**
- 作为 React 组件嵌入（`@excalidraw/excalidraw`）
- 通过 `ExcalidrawApp` 包装器获取完整功能
- 自定义主题：`excalidrawTheme` prop，覆盖 `---*` CSS 变量
- 事件监听：`onChange()` 获取实时 JSON 数据

**lingobridge 集成路径：**
1. 嵌入白板作为「课堂画布」功能
2. 教师的板书导出为 `.excalidraw` JSON → 存入数据库
3. 学生的图形作业以相同格式提交 → 教师端查看
4. 导出 PNG/SVG → 插入课件

### 5.3 表单渲染设计

**表单与 Canvas 的关系：**
- 表单输入域（文本框、选择框）在 Canvas 上绘制 → 手绘风格表单
- 交互事件通过坐标映射（hit testing）判断用户点击了哪个域
- 数据结构：`{ fieldId, type, value, bounds: {x,y,w,h} }`

**技术实现路径：**
```
1. 设计工具：Fabric.js（支持 SVG 解析 + Canvas）
2. 表单模板：预定义字段布局，导出为 JSON
3. 填写时：Canvas 坐标 → 检测 field 覆盖 → 弹出虚拟键盘
4. 提交时：JSON 数据 + Canvas 快照（提交凭证）
```

**手绘风格实现：**
- `rough.js`：为任意几何图形添加「手绘抖动」效果
- 独立库，不依赖 Canvas 框架
- Excalidraw 内部使用 rough.js

## 六、核心风险与结论

### 6.1 风险矩阵

| 技术点 | 风险等级 | 主要风险 |
|--------|---------|---------|
| PPTX 渲染 | 🔴 高 | 精准还原难，社区库不成熟 |
| Excel 复杂样式 | 🟡 中 | 样式渲染需 Pro 版或自建 CSS Grid |
| SVG 安全 | 🔴 高 | 需严格 sanitize，用户上传存在 XSS 风险 |
| PDF 大文件 | 🟡 中 | 内存占用高，需懒加载 + 分页 |
| Excalidraw 实时协作 | 🟡 中 | 需自建 WebSocket 或用现成服务（excalidraw.com 官方协作服务） |
| Canvas 性能 | 🟡 中 | 大量元素时需 OffscreenCanvas 优化 |

### 6.2 选型结论

**MVP 阶段推荐：**
- PDF → `pdfjs-dist`（CDN 直接引入，最快集成）
- MD → `marked` + `highlight.js`（零依赖）
- 图片 → 原生 `<img>` + `object-fit`（无需库）
- DOCX → `mammoth.js`（轻量转 HTML）
- Excel → `SheetJS`（数据层）+ AG Grid（可选，简化可不用）
- PPTX → **后端转 PDF 兜底**（LibreOffice 服务，绕过前端难题）
- 白板 → `@excalidraw/excalidraw`（直接 npm 集成）

**二期或高级版：**
- Excel 样式 → SheetJS Pro 或 ExcelJS
- PPTX 交互编辑 → Microsoft 365 Embed
- 表单 + 手绘风格 → Fabric.js + rough.js
- 实时协作 → 基于 Excalidraw 的 CRDT 方案或 WebSocket 自建
