# LingoBridge 后端架构分析

> 版本：v0.1 | 日期：2026-05-14 | 状态：草案
> 依据：LingoBridge MVP PRD v2.0（2026-05-06）、软件架构-层级图.canvas（2026-05-13）、技术方案评审（2026-05-13）
> 原则：平台化设计，业务解耦，可复用至任何垂直业务

---

## 一、总体架构定位

### 1.1 两个版本的架构目标

本分析将后端分为两个独立层次：

| 版本 | 定位 | 核心目标 |
|---|---|---|
| **业务赋能版** | 面向 LingoBridge MVP 的具体业务实现 | 快速交付，跑通"老师上传PPT→学生跟读练习"核心闭环 |
| **通用平台版** | 业务无关的底层能力抽象层 | 与具体业务解耦，可移植到任何垂直教育场景 |

两者关系：

```
┌────────────────────────────────────────────────┐
│           具体业务层（业务赋能版）                   │
│  PPT解析 · 课程管理 · 录音采集 · TTS · 录播        │
└─────────────────────┬──────────────────────────┘
                      │ 依赖（HTTP/gRPC 调用）
┌─────────────────────▼──────────────────────────┐
│           通用平台层（可复用）                     │
│  用户认证 · 文件存储 · 消息通知 · AI接入 · 审计日志   │
└───────────────────────────────────────────────┘
```

### 1.2 技术选型总览

| 组件    | 选型                                  | 理由                                                 |
| ----- | ----------------------------------- | -------------------------------------------------- |
| 后端框架  | **Node.js + Fastify**               | 团队技术栈统一；Fastify 高性能，Schema Validation 内置，适合教育平台    |
| 数据库   | **PostgreSQL**                      | 关系型数据（用户、课程、录音元数据）；JSONB 支持半结构化扩展                  |
| 对象存储  | **MinIO（开发/测试）/ 阿里云OSS（生产）**        | S3 兼容协议，本地 MinIO 无缝切阿里云 OSS；录音/视频/课件存储             |
| 缓存层   | **Redis**                           | 会话存储、接口限流、热点课程页缓存、直播状态                             |
| AI 能力 | **LLM 本地 3B（Ollama）+ 腾讯云 TTS API**  | MVP 阶段 TTS 用腾讯云（免费额度 800万字符/月）；v2.0 扩展本地 Ollama 翻译 |
| 消息队列  | **Redis Pub/Sub（轻量）/ RabbitMQ（扩展）** | MVP 用 Redis Pub/Sub 解耦异步任务；后续扩展录音转存等后台任务           |
| 部署方式  | **Docker Compose（单机）/ K8s（扩展）**     | 一键拉起所有服务，Dev/Prod 环境一致                             |

## 二、业务赋能版：基于 MVP 需求的后端架构

### 2.1 MVP 需求矩阵（Spring 版本对照）

> PRD v2.0 定义了 MVP 核心闭环，以下为对应的后端 API 映射。

| 功能 ID | 功能名称   | 后端 API 路由                                      | 数据模型                | 依赖服务             |
| ----- | ------ | ---------------------------------------------- | ------------------- | ---------------- |
| F01   | PPT 导入 | `POST /api/courses/:id/upload`                 | `course_page`       | MinIO、PPT 解析服务   |
| F02   | 课程练习同步 | `GET /api/courses/:courseId/pages`             | `course_page`       | PostgreSQL       |
| F03   | 中文 TTS | `GET /api/tts?text=...`                        | 无持久化                | 腾讯云 TTS API      |
| F04   | 录音采集   | `POST /api/recordings`                         | `recording`         | MinIO            |
| F05   | 录音存储   | （F04 同一接口）                                     | `recording`         | MinIO            |
| F06   | 录音管理   | `GET/DELETE /api/recordings/:id`               | `recording`         | PostgreSQL、MinIO |
| F07   | 哈语适配   | 前端 i18n，后端提供 `GET /api/i18n/:lang`             | 无                   | 静态资源             |
| F08   | 录播回看   | `POST /api/lectures` + `GET /api/lectures/:id` | `lecture_recording` | MinIO            |

### 2.2 功能拆分点

#### 2.2.1 模块边界划分

```
后端服务（单体模式 → 微服务演进路径）
│
├── auth-service        # 认证授权（SSO、用户注册登录、JWT）
├── course-service      # 课程管理（CRUD、课件上传、页面解析）
├── recording-service   # 录音管理（上传/存储/删除/下载）
├── tts-service         # TTS 调用（腾讯云 API 封装、重试、缓存）
├── lecture-service     # 录播管理（视频上传/转存/回看URL生成）
├── live-service        # 直播信令（WebSocket 翻页同步、权限控制）
├── exercise-service    # 练习作业（Excel 解析、练习题 CRUD）
├── file-service        # 文件上传下载（MinIO/OSS 统一抽象）
└── notification-service # 消息通知（站内信、邮件通知）
```

#### 2.2.2 数据隔离策略

| 策略 | 说明 | 适用场景 |
|---|---|---|
| Schema 隔离 | 不同业务用不同 PostgreSQL Schema | 多租户 SaaS 阶段（v2.0） |
| Row-Level Security | 同一 Schema 内行级数据权限 | MVP 单租户（CZU）足够 |
| Redis 缓存隔离 | 按 `tenant:userId:resourceId` key 命名 | 会话/缓存数据 |

MVP 阶段使用 **Row-Level Security（RLS）** 预留多租户扩展。

### 2.3 服务器配置要求

基于 LingoBridge Canvas 架构图的配置要求：

| 资源类型      | MVP 配置（轻量）               | v1.0 配置（标准）          | v2.0 配置（扩展）     |
| --------- | ------------------------ | -------------------- | --------------- |
| **计算实例**  | 2核 2GB                   | 2核 4GB               | 4核 8GB          |
| **系统盘**   | 40GB SSD                 | 80GB SSD             | 80GB SSD        |
| **内存**    | 2GB                      | 4GB                  | 8GB             |
| **对象存储**  | MinIO 本地（开发）→ 阿里云 OSS HK | 阿里云 OSS HK 500GB/年   | OSS 扩展 + CDN 加速 |
| **数据库**   | PostgreSQL 15（本地 Docker） | PostgreSQL 15（云）     | 主从复制，读写分离       |
| **缓存**    | Redis 7（本地 Docker）       | 阿里云 Redis 1GB        | Redis 集群        |
| **AI 推理** | —                        | Ollama 本地 3B（4核8G 起） | Ollama + GPU 实例 |
| **月成本**   | ~24 元（仅云服务器）             | ~60 元                | ~150 元          |

**配置要求说明：**
- 香港地域（HK Server）：绕过国内 ICP 备案，国内外访问延迟均可接受
- 30Mbps 带宽：满足 100 名学生同时访问 PPT 翻页和 TTS 播放
- MinIO 本地开发，切换生产环境只需改 `endpoint` 配置（S3 协议兼容）

      - "6379:6379"

      - lingo-net

  minio:
    image: minio/minio:latest

      MINIO_ROOT_USER: ${MINIO_USER:-minioadmin}
      MINIO_ROOT_PASSWORD: ${MINIO_PASSWORD:-minioadmin}

      - minio_data:/data

      - "9000:9000"   # API
      - "9001:9001"   # Console
    networks:
      - lingo-net

    image: nginx:alpine
    container_name: lingobridge-nginx
    restart: unless-stopped
    volumes:

      - ./frontend/dist:/usr/share/nginx/html:ro
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - backend
    networks:
      - lingo-net

  # === 业务赋能层 ===
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: lingobridge-backend
    restart: unless-stopped
    environment:
      NODE_ENV: ${NODE_ENV:-production}
      DATABASE_URL: postgres://${DB_USER:-lingo}:${DB_PASSWORD:-changeme}@postgres:5432/lingobridge
      REDIS_URL: redis://redis:6379
      MINIO_ENDPOINT: minio:9000
      MINIO_ACCESS_KEY: ${MINIO_USER:-minioadmin}
      MINIO_SECRET_KEY: ${MINIO_PASSWORD:-minioadmin}
      TENCENT_TTS_SECRET_ID: ${TENCENT_TTS_SECRET_ID}
      TENCENT_TTS_SECRET_KEY: ${TENCENT_TTS_SECRET_KEY}
      JWT_SECRET: ${JWT_SECRET:-changeme}
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock  # 用于录屏上传
    ports:
      - "3000:3000"
    depends_on:
      - postgres
      - redis
      - minio
    networks:
      - lingo-net

networks:
  lingo-net:
    driver: bridge

volumes:
  postgres_data:
  redis_data:
  minio_data:

**Nginx 基础配置（`nginx/nginx.conf`）：**

```nginx
events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    upstream backend {
        server backend:3000;
    }

    server {
        listen 80;
        server_name _;

        # 前端静态文件
        location / {
            root /usr/share/nginx/html;
            try_files $uri $uri/ /index.html;
        }

        # API 代理
        location /api/ {
            proxy_pass http://backend;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

        # WebSocket（直播信令）
        location /api/live/ {
            proxy_pass http://backend;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_read_timeout 86400;
        }

        # MinIO API（对象存储）
        location /minio/ {
            proxy_pass http://minio:9000/;
            proxy_set_header Host $host;
        }

        # 录屏大文件上传（增加超时）
        location /api/recordings {
            client_max_body_size 100M;
            proxy_read_timeout 300s;
            proxy_connect_timeout 75s;
        }
    }
}
```

## 三、通用平台版：业务无关的底层能力架构

### 3.1 通用平台服务矩阵

> 以下服务不包含任何 LingoBridge 特定业务逻辑，可直接复用于任何垂直教育平台。

| 服务名称 | 职责 | 核心 API | 可复用场景 |
|---|---|---|---|
| **auth-service** | 用户注册/登录/JWT/SSO/权限控制 | `POST /auth/*`, `GET /users/me` | 任何需要用户体系的平台 |
| **file-service** | 文件上传/下载/删除（MinIO/OSS 抽象） | `POST /files`, `GET /files/:id`, `DELETE /files/:id` | 任何需要文件存储的平台 |
| **notification-service** | 邮件/站内信/推送通知 | `POST /notifications`, `GET /notifications` | 任何需要通知的平台 |
| **audit-service** | 操作日志、审计追踪（满足合规要求） | `GET /audit/logs`, `POST /audit/events` | 任何需要审计的平台 |
| **ai-gateway** | AI 能力统一接入层（LLM/TTS/ASR/翻译） | `POST /ai/tts`, `POST /ai/translate` | 任何需要 AI 能力的平台 |
| **media-service** | 音视频转码/截图/元数据提取 | `POST /media/transcode`, `GET /media/thumbnail` | 任何有音视频需求的平台 |
| **analytics-service** | 用户行为埋点/数据统计/报表 | `POST /events`, `GET /analytics/*` | 任何需要数据分析的平台 |
| **i18n-service** | 多语言资源管理 | `GET /i18n/:lang` | 任何需要国际化的平台 |

### 3.3 AI 能力接入层（ai-gateway）

```
                    ┌─────────────────────────────────────────┐
                    │            AI Gateway（统一接入层）         │
                    │  ┌──────────┐  ┌──────────┐  ┌───────┐  │
                    │  │ TTS 适配器│  │ 翻译适配器 │  │ LLM  │  │
                    │  └────┬─────┘  └────┬─────┘  └───┬───┘  │
                    └───────┼────────────┼─────────────┼───────┘
                            │            │             │
                    ┌───────▼──────┐  ┌───▼───────┐ ┌───▼──────┐
                    │ 腾讯云 TTS   │  │ DeepSeek  │ │ Ollama 3B│
                    │ （中文 TTS） │  │（翻译 v2.0）│ │（本地翻译）│
                    └─────────────┘  └───────────┘ └──────────┘

ai-gateway 职责：
1. 统一鉴权（Token 限额/计费）
2. 请求路由（根据业务选择 TTS/翻译/LLM）
3. 降级策略（主 API 不可用时切换备选）
4. 缓存 TTS 结果（相同文本不重复调用）
5. 请求日志（AI 费用追踪）
```

### 3.4 插件化架构（业务解耦核心）

```
通用平台层不直接依赖任何具体业务。
通过事件总线（Event Bus）通知业务层：

┌──────────────┐    Event    ┌──────────────────┐
│ 业务赋能层    │ ──────────► │   通用平台层      │
│（具体业务）   │ ◄────────── │  （Event Bus）   │
└──────────────┘   Subscribe  └──────────────────┘

示例事件：
- user.created         → 通知邮件发送
- recording.completed → 触发 AI 质检
- course.published     → 推送课程更新通知
- file.uploaded        → 触发病毒扫描

通用平台层仅消费事件，不感知业务细节。
新业务接入时，只需订阅相关事件，无需修改通用平台代码。
```

### 3.5 扩展路径：从 MVP 到通用平台

```
MVP 阶段（当前）
└── 单体后端，所有业务逻辑在同一服务中
    └── 通用能力抽离为内部模块（尚未独立部署）

    ↓ 未来 2-3 个月

v1.0 阶段
└── 通用服务独立部署（auth/file/ai-gateway/audit）
└── 业务服务继续单体

    ↓ 未来 6 个月

v2.0 阶段（多租户 SaaS）
└── 通用服务标准化（统一 API Gateway、统一 SDK）
└── 业务服务按租户隔离部署
└── 插件市场（各类垂直教育场景接入）
```

## 四、API 设计规范

### 4.1 RESTful 风格约定

| 规范   | 说明                                                                                                        |
| ---- | --------------------------------------------------------------------------------------------------------- |
| 路由格式 | `/api/v1/{resource}`（版本化）                                                                                 |
| 动词   | GET（查询）、POST（创建）、PUT（全量更新）、PATCH（部分更新）、DELETE（删除）                                                         |
| 状态码  | 200 OK、201 Created、400 Bad Request、401 Unauthorized、403 Forbidden、404 Not Found、500 Internal Server Error |
| 响应格式 | `{ "code": 0, "data": {...}, "message": "success" }`                                                      |
| 错误格式 | `{ "code": 40001, "message": "录音文件超过 100MB", "detail": {...} }`                                           |

### 4.2 核心 API 路由

```
认证
POST   /api/v1/auth/register          # 注册
POST   /api/v1/auth/login             # 登录
POST   /api/v1/auth/logout            # 登出
POST   /api/v1/auth/refresh           # 刷新 Token

用户
GET    /api/v1/users/me               # 当前用户信息
PATCH  /api/v1/users/me               # 修改个人信息
GET    /api/v1/users/:id              # 管理员查询用户

课程
GET    /api/v1/courses                # 课程列表
POST   /api/v1/courses                # 创建课程（教师）
GET    /api/v1/courses/:id           # 课程详情
DELETE /api/v1/courses/:id            # 删除课程（教师）
GET    /api/v1/courses/:id/pages      # 课件页面列表

课件
POST   /api/v1/coursewares           # 上传课件（教师）
GET    /api/v1/coursewares/:id        # 课件详情
DELETE /api/v1/coursewares/:id        # 删除课件

录音
POST   /api/v1/recordings             # 上传录音
GET    /api/v1/recordings             # 录音列表（当前用户）
GET    /api/v1/recordings/:id         # 录音详情
DELETE /api/v1/recordings/:id         # 删除录音

练习
GET    /api/v1/exercises?courseId=&page=  # 练习题列表
POST   /api/v1/exercises/:id/submit      # 提交练习答案

录播
POST   /api/v1/lectures               # 创建录播（教师）
GET    /api/v1/lectures               # 录播列表
GET    /api/v1/lectures/:id           # 录播详情（签名 URL）
DELETE /api/v1/lectures/:id           # 删除录播

TTS
GET    /api/v1/tts?text=&lang=        # TTS 合成（返回音频 URL）

AI 能力（v2.0）
POST   /api/v1/ai/translate           # 翻译
POST   /api/v1/ai/asr                # 语音识别
POST   /api/v1/ai/evaluate           # 发音评测
```
