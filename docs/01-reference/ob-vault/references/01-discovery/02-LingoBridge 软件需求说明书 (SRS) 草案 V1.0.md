# **软件需求规格说明书 (SRS) \- LingoBridge 教学辅助平台**

**版本：** V1.0  | **状态：** 需求评审中

## **1\. 行业背景与研发动机 (Industry Context & Motivation)**

当前，俄语区（如哈萨克斯坦、俄罗斯等）的国际中文教育正面临数字化转型的关键节点。据相关研讨会调研指出，传统俄语背景的汉语教学普遍存在教学法体系参差不齐、现代智能技术与传统课堂脱节的痛点\[cite: 1\]。线下教学时效短（如196课时/学期），学生高度依赖静态电子课件预习，缺乏课后沉浸式交互练习工具。LingoBridge 应运而生，旨在通过“母语辅助学习（以俄语学中文）”的创新教学法，结合腾讯会议教育版的开放接口，为C2K等国际教育机构提供一套可解耦、可度量、高互动的全链路教学解决方案。

## **2\. 顶层架构设计 (Top-Down Architecture)**

依据“自顶向下”的设计原则，系统划分为三大核心架构层：

* **资源规划层 (Governance)：** 以“课程教纲”为顶层输入，规范学期课时分配、知识点大纲（如声母、韵脚、音标等）。  
* **业务逻辑执行层 (Execution)：** 教师端作为总控台，实现PPT多媒体转化、课件流式分发、腾讯会议日程自动拉起及教学过程管控。  
* **多模态交互层 (Interaction)：** 学生端支持Web/Mobile/iPad自适应，提供课前动态预习、课中双语直播跟随、课后录音测评与积分排名。

## **3\. 核心功能及技术指标拆解 (Core Features & Benchmarks)**

| 功能模块 (Module)                      | 业务需求说明 (Business Requirement)          | 技术实现方案与指标约束 (Technical Implementation & Specs)                                                                                  |
| :--------------------------------- | :------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------ |
| **实时双语翻译字幕 (Live Subtitles)**      | 在腾讯会议直播期间，为俄语背景学生提供低延迟的中俄双语滚动字幕辅助。     | 调用腾讯会议音频流 API \[cite: 2\]。接入高并发 ASR（语音识别）引擎与 LLM 机器翻译模型。**指标：** 端到端延迟严格控制在 500ms-800ms 内，翻译准确率要求 \>85%，支持专业术语（如“拼音”、“四声”）的语境对齐。 |
| **交互式多媒体课件引擎 (Interactive H5)**    | 突破传统静态 PPT 限制，支持视频、音频、测试题等元素的内嵌与跨端分发。  | 利用 Vite \+ React 架构构建跨端渲染引擎。后端解析原始素材并封装为 JSON 驱动的组件。**架构优势：** 实现组件级状态管理，大幅降低客户端带宽消耗，支持断点续学。                                     |

| **激励型积分排行榜 (Gamification Engine)** | 按“学校-年级-班级”划分域，实时展示学生学习时长与作业质量排名，提升日活。 | 数据库采用 postgresql 维护多租户层级关系。排行榜高频查询业务下沉至 Redis 缓存层利用 ZSET 数据结构实现。**体验指标：** 榜单数据更新延迟 \< 1s，仅班级内可见以保护隐私。                           |

## **4\. 数据结构与实体关系 (Data Structure & ER)**

为支撑多国际教育机构的接入，核心数据表结构需满足 SaaS 级的多租户隔离：  
`# 核心实体关系模型简述`  
`- Institution (机构表): ID, Name, Region (如：哈萨克斯坦), License_Type`  
`- School (学校表): ID, Inst_ID, Name, Contact`  
`- Class (班级表): ID, School_ID, Grade, Teacher_ID`  
`- User (用户表): ID, Role (Student/Teacher/TA), Class_ID, Points, Profile`  
`- Course_Material (教学资源): ID, Class_ID, Type (Video/Audio/H5), Sync_Status`

## **5\. 风险评估与合规性 (Risk & Compliance)**

* **政策与合规风险：** 需对标近期国内关于在线教育及数据出境的合规要求，特别是涉及留学生信息的数据安全传输加密标准。  
* **跨国网络质量：** 针对中亚地区网络基础设施现状，需在客户端部署断线重连、媒体流降级等弱网补偿机制，保障视频连麦与字幕推送的可用性。
