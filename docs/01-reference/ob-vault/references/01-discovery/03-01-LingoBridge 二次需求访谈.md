---

1. Idea 阶段（Layer 1 需求层）
需求分析访谈记录，完成项目初始想法挖掘与用户需求收集。
​
2. Plan + MVP 阶段（Layer 1 规划层）
输出项目计划粗略方案，同步完成最小可行性产品验证调研，明确核心业务范围。（本阶段专注于调研技术可行性和甲方进行交互和专业常识扫盲，安抚甲方情绪，明确sprint开发周期（一般1-2周为一个周期））
​
3. UI/UX 原型阶段（Layer 2 设计层）
完成前端界面设计、交互设计，输出产品原型并开展内部功能测试。
​
4. GitHub + Vercel 模拟上线（Layer 2 研发层）
基于 GitHub 做版本管理、静态版本管控，通过 Vercel 实现内部演示、模拟上线验证。
​
5. ==PRD 产品文档[^1]（Layer 1 需求功能约束层）==
编写产品需求文档，为项目构建唯一约束与开发指导，明确定位/目标/核心规则、需求描述{需求描述}、功能做/不做边界（背景、目标、范围）；用户故事（至少2个场景）； 功能结构（模块拆解）；业务流程（用Mermaid流程图）；交互设计（界面布局 + 交互规则）；数据设计（实体定义）；验收标准（可量化、可测试）。
​
6. Docker‑Compose 一键拉起（Layer 2 后端集成层）
编写  docker‑compose.yml ，整合 API 服务、后端数据库、Redis、MinIO、Nginx、前端分发，完成后端服务集成。
​
7. 测试 + CI/CD 自动化（Layer 3 测试交付层）
接入腾讯云、Lighthouse 完成质量测试；打通 GitHub、Docker Hub 链路，配置 CI/CD 自动化流程；通过 Nginx 解决国内公网访问、代理转发问题。
​
8. 生产环境正式上线（Layer 3 部署上线层）
完成域名注册、ICP 备案、域名绑定，开展合规性校验、线上测试，最终完成产品上线交付。

考虑到几个场景和因素：
1.本项目是国际教育项目，未来需要能够在国内国际都能访问
2.MVP阶段只需要跑通czu单租户即可，即（ppt上传->系统课程同步->进入练习，点击按钮调用中文tts，录制本地录音->录音数据存储本地/云端同步->支持录音删除、下载本地）
3.老师端比赛之前需要系统能够实现：课程录屏（通过本系统进行视频流的存储）、本地摄像头调用（tm虚拟背景）、ppt导入（电脑本地ppt屏幕共享）
4.未来教师端支持实时弹幕和双语字幕需要考虑在云端服务器ollama拉一个3B小模型用于双语字幕翻译；
5.hk的服务器必须满足：国内国外都可以访问；弹性实例计算；对象存储实例；databsee；大模型向量存储；部署小模型翻译（或直接调用模型API按量计费）
