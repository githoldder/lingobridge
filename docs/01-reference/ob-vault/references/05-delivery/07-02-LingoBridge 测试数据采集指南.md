# LingoBridge 测试数据采集指南

> 目的：让老师能组织学生进行真实中文跟读数据采集，并保证数据可用、可检查、可复盘。  
> 适用阶段：MVP 公网试用、小班试采、项目汇报前数据准备。

---

## 一、采集目标

本阶段不是追求 AI 自动评分，而是先建立一批可用的中文学习过程数据：

| 数据类型 | 采集目的 |
|---|---|
| 学生账号与身份 | 区分不同学生的学习记录 |
| 课程课件 | 建立统一练习内容 |
| Excel 练习任务 | 让学生按同一批句子/词汇录音 |
| 学生跟读录音 | 后续可用于教师听评、语音评测、模型测试 |
| 学习完成记录 | 统计学生是否完成指定练习 |
| 问题反馈 | 修复公网访问、录音、上传、界面语言问题 |

---

## 二、推荐采集规模

先小后大，不建议第一次就全班铺开。

| 阶段 | 人数 | 目标 | 时间 |
|---|---:|---|---|
| 试采 1 | 2-3 名学生 | 验证登录、录音、上传是否稳定 | 30 分钟 |
| 试采 2 | 8-10 名学生 | 验证不同设备、不同网络环境 | 1 天 |
| 小班采集 | 20-30 名学生 | 形成第一批可展示数据 | 1 周 |
| 扩展采集 | 50-100 名学生 | 用于项目验收和后续算法评估 | 2-4 周 |

---

## 三、采集前准备

### 1. 老师准备材料

| 材料 | 要求 |
|---|---|
| 课程 PDF/PPTX | 主题明确，建议 5-15 页 |
| Excel 练习表 | 每课 10-30 条词句即可 |
| 采集说明 | 告诉学生录什么、录几条、什么时候完成 |
| 测试账号 | 先用统一测试账号演示，再分配学生账号 |

### 2. 技术准备

| 检查项 | 通过标准 |
|---|---|
| 公网地址可访问 | 手机 4G/5G 和校园网都能打开 |
| 教师账号可登录 | `teacher@test.com` 登录成功 |
| 学生账号可登录 | `student_a@test.com` 登录成功 |
| 麦克风授权可用 | 浏览器能录音 |
| 录音可回放 | 上传后能在列表里播放 |
| 管理后台可查看 | 管理员能看到数据 |

### 3. 学生设备建议

| 设备 | 建议 |
|---|---|
| 手机 | Android/Chrome 优先；iPhone/Safari 需单独测试录音兼容性 |
| 电脑 | Chrome/Edge 优先 |
| 麦克风 | 手机自带麦克风可用，耳机麦克风更稳定 |
| 网络 | 校园网、宿舍网、手机流量都建议各测一遍 |

## 四、Excel 采集模板建议

建议每行对应一个采集任务。

| 字段 | 示例 | 说明 |
|---|---|---|
| `course_code` | `CZU-CHN-001` | 课程编号 |
| `unit` | `1` | 单元 |
| `lesson` | `1` | 课次 |
| `task_id` | `CZU-CHN-001-L01-001` | 每条任务唯一编号 |
| `task_type` | `pronunciation` | 跟读任务 |
| `zh_text` | `大家好，我叫阿合买提。` | 学生需要朗读的中文 |
| `pinyin` | `Dajia hao, wo jiao Ahemaiti.` | 辅助发音 |
| `translation_ru` | `Здравствуйте, меня зовут Ахмет.` | 俄语释义 |
| `translation_kk` | `Сәлеметсіз бе, менің атым Ахмет.` | 哈语释义 |
| `publish_to_homework` | `TRUE` | 是否进入作业 |
| `publish_to_vocab` | `FALSE` | 是否进入词汇 |

任务设计建议：

| 类型 | 数量 | 示例 |
|---|---:|---|
| 问候句 | 5 条 | 你好、大家好、老师好 |
| 自我介绍 | 5-10 条 | 我叫……、我来自…… |
| 课堂指令 | 5-10 条 | 请跟我读、请打开书 |
| 声调重点词 | 10-20 条 | 妈、麻、马、骂 |

## 五、学生采集流程

老师发给学生的简版流程：

1. 打开老师提供的网址。
2. 登录自己的学生账号。
3. 进入指定课程。
4. 点击朗读按钮，听一遍标准中文。
5. 点击录音，允许麦克风权限。
6. 按页面中文完整朗读。
7. 停止录音并等待保存。
8. 播放检查是否清楚。
9. 如果有问题，删除后重新录。
10. 完成老师指定数量后退出登录。

---

## 六、录音质量标准

| 项目 | 合格 | 不合格 |
|---|---|---|
| 声音 | 能清楚听到学生声音 | 没声音、声音太小 |
| 内容 | 按指定中文朗读 | 读错内容或只读一半 |
| 噪音 | 背景安静或轻微噪音 | 音乐、人声干扰明显 |
| 时长 | 5-30 秒 | 过短或长时间空白 |
| 设备 | 同一条录音中设备稳定 | 中途断音、爆音 |

建议老师第一次采集时，不要追求完美发音，先保证“有数据、能听清、能对应任务”。

## 七、数据命名与管理规则

### 1. 账号命名

正式采集建议：

```text
student_001@test.com
student_002@test.com
student_003@test.com
```

如果需要保密，可用编号代替真实姓名。

### 2. 课程命名

```text
2026春-哈萨克斯坦学生中文试采-第01课-自我介绍
```

### 3. Excel 任务编号

```text
KZ-CHN-L01-001
KZ-CHN-L01-002
KZ-CHN-L01-003
```

### 4. 数据导出与备份

MVP 阶段建议每次采集后做三件事：

| 操作 | 负责人 |
|---|---|
| 后台查看录音数量 | 项目负责人 |
| 抽听 5-10 条录音 | 老师 |
| 备份服务器数据目录 | 技术负责人 |

## 八、老师现场组织话术

可直接发给学生：

> 同学们，我们正在测试一个中文学习网站。今天只需要完成一个简单任务：登录网站，进入指定课程，听中文句子朗读，然后自己录音跟读。录音只用于中文学习研究和系统测试，不做公开传播。请尽量在安静环境下录音，录完后自己播放检查一遍，如果没有声音或不清楚，可以删除后重新录。

---

## 九、问题反馈表

每次采集建议记录以下问题：

| 学生编号 | 设备 | 浏览器 | 网络 | 问题描述 | 是否解决 |
|---|---|---|---|---|---|
| student_001 | Android | Chrome | 校园网 | 麦克风无法授权 | 待处理 |
| student_002 | Windows | Edge | 宿舍网 | 上传慢 | 已记录 |

常见问题：

| 问题 | 处理方式 |
|---|---|
| 无法打开网站 | 换手机流量测试；确认服务器是否在线 |
| 登录失败 | 检查账号密码，注意大小写 |
| 麦克风无法使用 | 检查浏览器权限，建议换 Chrome |
| 录音没有声音 | 删除后重新录，确认麦克风输入 |
| 页面卡顿 | 刷新页面或更换网络 |

## 十、阶段性验收口径

小范围试采完成后，可向老师汇报：

| 指标 | 达标口径 |
|---|---|
| 公网访问 | 校园网和手机流量均可打开 |
| 登录 | 教师、学生、管理员账号均可登录 |
| 课件 | 老师能上传，学生能查看 |
| TTS | 学生能听中文朗读 |
| 录音 | 学生能录制、上传、播放、删除 |
| 数据 | 后台能看到录音和学习进度 |
| 反馈 | 已记录设备兼容和使用问题 |
