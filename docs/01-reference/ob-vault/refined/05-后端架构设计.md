# LingoBridge 后端架构设计规范

## 1. 总体架构与选型

基于“PPT课件驱动中文跟读练习工具”的 MVP 定位，后端整体服务架构采用 **Node.js + Fastify** 极简高性能单体架构。

### 技术选型矩阵
- **Web 服务框架**：Node.js + Fastify（相较于 Express，拥有内置高性能 Schema 检验与极速 JSON 序列化能力）。
- **关系型数据库**：PostgreSQL 15（存储用户、课程、页码和录音元数据，利用行级安全策略 RLS 支撑单租户扩展）。
- **对象存储**：MinIO（本地开发与测试环境）/ 阿里云 OSS 香港地域（生产环境，S3 协议高度兼容，CN2 优质网络加速）。
- **缓存与状态层**：Redis 7（提供用户 Session 校验、高频课程页热点数据缓存、接口限流等）。
- **容器与部署**：Docker Compose 实现一键全容器化服务编排，Nginx 充当反向代理与 HTTPS SSL 入口。

---

## 2. 数据隔离策略 (RLS)

MVP 阶段只跑通单学校单租户（CZU）闭环。为了对未来 V2.0 阶段的 SaaS 多学校入驻保留优雅的可扩展性，我们在数据库表结构设计上统一引入 `tenant_id` 与 `school_id` 属性，在元数据层面通过 PostgreSQL 的 **Row-Level Security (RLS，行级安全策略)** 进行逻辑隔离。

- **行级安全约束**：系统通过查询上下文中的当前用户 Session 的 `tenant_id`，在数据库引擎层面自动对 `course`、`recording`、`user` 的查询和插入进行 `WHERE tenant_id = CURRENT_TENANT` 限制，无需在应用层写大量冗余过滤逻辑。

---

## 3. AI Gateway 统一接入层 (AI 调度引擎)

为妥善应对 AI 频繁调用导致的额度受限问题（特别是 TTS 与 ASR 额度），我们在通用层封装了 **AI Gateway (AI 调度引擎)**。

### 3.1 调度机制
- **流控与重试**：AI Gateway 对第三方接口调用（腾讯云/百度/Azure）进行高频流控限制，当发生超频限流错误时，系统以指数退避算法进行 3 次重试。
- **多源平滑备选**：默认调度腾讯云基础合成音色（每月享有 800 万字符免费额度），当监控到腾讯云接口出现 5xx 故障或资源包超限时，网关瞬间且无感地平滑切换为百度语音或阿里云 TTS 基础版。
- **OSS 音频缓存（避雷针机制）**：学生端高频点击同一课件的同一句话时，AI Gateway 拦截该请求，首先检测 OSS 中是否已缓存对应的 MD5 文本哈希命名的音频文件。如果存在，则直接返回 OSS 的 CDN 加速静态链接，**彻底阻断三方 API 重复调用与额度扣减**。

## 4. 核心 API 路由与 Schema 规范 (30条)

### 4.1 通用约定
- 路由前缀：`/api/v1`
- 请求体及响应格式：`application/json`
- 成功响应通用结构：
  ```json
  {
    "code": 0,
    "data": {},
    "message": "success"
  }
  ```

---

### 4.2 API 路由列表 (分类说明)

#### 4.2.1 认证模块 (Auth)
1. **`POST /api/v1/auth/register`**：用户注册。
   - 请求参数：`{ username, password, display_name, role }`
   - 响应数据：`{ id, username, role }`
2. **`POST /api/v1/auth/login`**：用户登录。
   - 请求参数：`{ username, password }`
   - 响应数据：`{ token, user: { id, username, display_name, role } }`
3. **`POST /api/v1/auth/logout`**：用户登出，销毁 Session 缓存。
4. **`POST /api/v1/auth/refresh`**：通过 Refresh Token 刷新 JWT 有效期。

#### 4.2.2 用户模块 (Users)
5. **`GET /api/v1/users/me`**：获取当前登录用户信息。
6. **`PATCH /api/v1/users/me`**：更新用户个人偏好设置（如 `language_pref` 界面切换偏好）。
7. **`GET /api/v1/users/:id`**：管理员或教师查询特定学生档案与统计信息。
8. **`DELETE /api/v1/users/:id`**：管理员注销该用户。

#### 4.2.3 课程模块 (Courses)
9. **`GET /api/v1/courses`**：获取课程列表，学生仅能获取所属班级的课程。
10. **`POST /api/v1/courses`**：创建新课程（限教师）。
    - 请求参数：`{ title, description }`
11. **`GET /api/v1/courses/:id`**：获取特定课程大纲详情。
12. **`DELETE /api/v1/courses/:id`**：教师删除特定课程（级联删除课件页与录音记录）。
13. **`GET /api/v1/courses/:id/pages`**：获取特定课程解析后的全部课件图片与朗读文本页面列表。

#### 4.2.4 课件处理模块 (Coursewares)
14. **`POST /api/v1/courses/:id/upload`**：上传并关联课件文件（限教师，最大50MB）。
    - 请求：`multipart/form-data`（字段 `file`）
    - 响应：`{ id, filename, sync_status: 'processing' }`
15. **`GET /api/v1/coursewares/:id`**：获取解析状态，包括后端 LibreOffice 转换 PDF 的进度反馈。
16. **`DELETE /api/v1/coursewares/:id`**：教师下架、清除该课件。

#### 4.2.5 录音管理模块 (Recordings)
17. **`POST /api/v1/recordings`**：学生上传跟读录音文件。
    - 请求：`multipart/form-data`（字段 `file`, `course_id`, `page_number`, `duration_sec`）
    - 响应：`{ id, audio_url, points_added: 10 }`
18. **`GET /api/v1/recordings`**：学生拉取个人的所有录音跟读记录。
19. **`GET /api/v1/recordings/:id`**：拉取特定录音文件的播放详情。
20. **`DELETE /api/v1/recordings/:id`**：学生删除个人特定录音。
21. **`GET /api/v1/recordings/teacher/view`**：教师拉取全班学生提交的录音，支持按课程、学生或页码筛选。

#### 4.2.6 练习作业模块 (Exercises)
22. **`GET /api/v1/exercises`**：获取当前页面由 Excel 大纲导入并自动绑定的练习题作业。
23. **`POST /api/v1/exercises/:id/submit`**：学生在线提交练习答案，自动记录得分。

#### 4.2.7 课程录播模块 (Lectures)
24. **`POST /api/v1/lectures`**：教师上传录屏后的 WebM 课程录像大文件（最大100MB）。
25. **`GET /api/v1/lectures`**：学生拉取历史录像列表。
26. **`GET /api/v1/lectures/:id`**：获取特定录像的高保真 OSS 签名链接。
27. **`DELETE /api/v1/lectures/:id`**：教师下架已录制的上课视频。

#### 4.2.8 AI 能力调度 (AI)
28. **`GET /api/v1/tts`**：获取当页绑定文本的普通话标准音频文件。
    - 参数：`text=中文朗读文本&lang=zh-CN`
    - 响应：`{ audio_url }`（先查 OSS 缓存，未命中则调用腾讯云 API 写入 OSS 后返回）
29. **`POST /api/v1/ai/translate`**（V2.0）：中哈/中俄文本流式翻译。
30. **`POST /api/v1/ai/evaluate`**（V1.0）：调用 ASR 进行中文发音自动评测打分。

## 5. 部署配置梯度与环境实现

### 5.1 Dev/Prod 环境配置梯度
- **开发/测试环境 (Dev)**：所有基础设施跑在开发者本地电脑的 Docker 容器中。对象存储完全采用本地 **MinIO**（9000端口 API，9001端口 Console），提供 S3 高度兼容性。数据库为单实例 PostgreSQL，Redis 作为本地缓存。
- **生产环境 (Prod)**：部署在阿里云香港地域（HK Server）。对象存储无缝切换为**阿里云 OSS（香港地域）**。通过 Nginx 配置 HTTPS Let's Encrypt 证书并对外发布，大文件上传超时参数调整。

---

### 5.2 生产环境容器编排 (`docker-compose.yml`)

```yaml
version: "3.9"

services:
  # 1. 结构化数据库 - PostgreSQL 15
  postgres:
    image: postgres:15-alpine
    container_name: lingobridge-db
    restart: unless-stopped
    environment:
      POSTGRES_DB: lingobridge
      POSTGRES_USER: ${DB_USER:-lingo}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    networks:
      - lingo-net

  # 2. 缓存及会话存储 - Redis 7
  redis:
    image: redis:7-alpine
    container_name: lingobridge-redis
    restart: unless-stopped
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"
    networks:
      - lingo-net

  # 3. 反向代理与 HTTPS 服务器 - Nginx
  nginx:
    image: nginx:alpine
    container_name: lingobridge-nginx
    restart: unless-stopped
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./frontend/dist:/usr/share/nginx/html:ro
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - backend
    networks:
      - lingo-net

  # 4. 后端 Fastify API 服务
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: lingobridge-backend
    restart: unless-stopped
    environment:
      NODE_ENV: production
      DATABASE_URL: postgres://${DB_USER:-lingo}:${DB_PASSWORD}@postgres:5432/lingobridge
      REDIS_URL: redis://redis:6379
      OSS_ENDPOINT: oss-cn-hongkong.aliyuncs.com
      OSS_ACCESS_KEY_ID: ${OSS_KEY_ID}
      OSS_ACCESS_KEY_SECRET: ${OSS_KEY_SECRET}
      OSS_BUCKET: lingobridge-recordings
      TENCENT_TTS_SECRET_ID: ${TENCENT_TTS_SECRET_ID}
      TENCENT_TTS_SECRET_KEY: ${TENCENT_TTS_SECRET_KEY}
      JWT_SECRET: ${JWT_SECRET}
    ports:
      - "3000:3000"
    depends_on:
      - postgres
      - redis
    networks:
      - lingo-net

networks:
  lingo-net:
    driver: bridge

volumes:
  postgres_data:
  redis_data:
```

### 5.3 生产环境反向代理配置 (`nginx/nginx.conf`)

```nginx
events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    upstream backend_api {
        server backend:3000;
    }

    server {
        listen 80;
        server_name _;
        # 强制跳转 HTTPS
        return 301 https://$host$request_uri;
    }

    server {
        listen 443 ssl;
        server_name _;

        ssl_certificate /etc/nginx/ssl/live.crt;
        ssl_certificate_key /etc/nginx/ssl/live.key;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;

        # 1. 前端 React 构建包静态直接服务
        location / {
            root /usr/share/nginx/html;
            try_files $uri $uri/ /index.html;
            gzip_static on;
        }

        # 2. 后端 Fastify API 接口反向代理
        location /api/ {
            proxy_pass http://backend_api;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

        # 3. HTML5 录音录屏大文件上传配置（解除默认限制并增加超时）
        location /api/v1/recordings {
            client_max_body_size 120M;
            proxy_read_timeout 300s;
            proxy_connect_timeout 90s;
            proxy_pass http://backend_api;
        }

        location /api/v1/lectures {
            client_max_body_size 150M;
            proxy_read_timeout 400s;
            proxy_connect_timeout 90s;
            proxy_pass http://backend_api;
        }
    }
}
```
