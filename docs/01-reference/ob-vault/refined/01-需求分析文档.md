# LingoBridge 需求分析文档

## 1. 行业背景与研发动机

俄语区（如哈萨克斯坦、俄罗斯等）的国际中文教育面临数字化转型的关键节点。据中国俄语教学研究研讨会指出，传统俄语背景的汉语教学中现代智能技术与传统课堂脱节。线下教学时效短（196课时/学期，每周上一次课），学生缺乏交互练习工具。

LingoBridge 采用“母语辅助学习（以俄语学中文）”教学法，为C2K等国际教育机构提供可解耦、高互动的全链路教学解决方案，与腾讯会议等工具形成合围之势。

## 2. 用户与角色

### 2.1 用户体系与用户画像

| 角色 | 职责 | 客户端类型 |
| :--- | :--- | :--- |
| **老师** | 发布课程、上传及录制PPT课件、批改作业、管理学生学籍与数据 | Web端 |
| **学生** | 课前动态预习、课后发音练习、进行音标/声母/韵脚训练、发音测评、查看班级积分排名 | Web端（响应式自适应手机、平板与PC） |
| **管理员** | 平台整体系统管控、机构入驻审核、学校认证管理 | Web端 |

#### 典型用户画像（学生）
- **主要群体**：哈萨克斯坦高校在读留学生（年龄18-22岁为主），以及具有俄语背景的中文初学者。
- **学制背景**：1学期4.5个月，总课时196学时。由于每周仅上1-2次课，课外存在大段时间间隔，极度缺乏日常高频的“拼音-声母-韵脚”辅助发音校准工具。
- **课堂场景**：1名中文教师对20名留学生的在线网课教学。

---

### 2.2 核心优势
- **师资力量**：任课老师均是对外交流使者，具备丰富的跨文化教学经验。
- **学生资源**：依托高校资源，能直接对接哈萨克斯坦的大量中文留学生。
- **机构合作**：作为 C2K 国际教育机构成员，具备顺畅的跨境B端沟通与试点渠道。
- **模式跑通**：已在疫情时期跑通线上线下结合的“网课直播教学”模式。

## 3. 需求演进与分期规划

### 3.1 两次需求访谈的演进轨迹
在项目初期，甲方提出了涵盖“实时双语直播翻译、云端视频自动沉淀、B站式双语字幕、解耦收费、班级积分”等庞大的系统预想。经过两次深入访谈，项目逐步由“全量平台”向“实用性工具网站”收敛，最终确定了以 **最小可行性产品 (MVP)** 快速迭代的路线。

- **V1.0 MVP 核心收敛**：放弃初期繁琐的教务对接和高成本的实时直播字幕，聚焦于“PPT导入 -> 系统课程同步 -> 音标/发音跟读练习”闭环。利用本地录音调用中文 TTS 进行发音校准，并对录音数据进行本地或云端低成本存储管理。
- **V2.0 阶段**：引入实时弹幕和双语字幕，在云端服务器部署 3B 规模翻译小模型，并加入测评系统和学校认证。

### 3.3 分期交付规划
1. **V1.0 MVP 阶段**：单租户架构。跑通“PPT课件共享导入 -> 拼音发音跟读 -> 录音调用 TTS 校准 -> 录音本地与云端管理 -> 班级积分排名”闭环。
2. **V1.1 阶段**：支持多租户隔离，上线课前动态预习系统，腾讯会议日程一键预约。
3. **V2.0 阶段**：上线电子试卷测评系统、学校认证，在云端部署 3B 翻译模型提供高并发低延迟的实时双语字幕。
4. **V3.0 阶段**：集成国际付费（C端付费与B端抽成解耦）、线上互动社区。

## 5. 关键待澄清项与最终结论

1. **收费模式**
   - **结论**：MVP阶段为0元试用与低成本试点方案，后续 V3.0 解耦收费支持B端抽成与C端按课程包付费两种模式。
2. **视频与录音存储方案**
   - **结论**：初期使用腾讯云临时过度，正式上线统一采用阿里云香港节点的对象存储（OSS），满足跨境访问低延迟。
3. **是否需要对接哈萨克斯坦高校教务系统**
   - **结论**：不直接对接。系统独立运行，通过“学校-年级-班级”的 SaaS 结构提供独立的企业级/学校级分发管理。
4. **实时双语字幕技术选型**
   - **结论**：V1.0 MVP阶段暂不上线。V2.0 阶段在云端部署 Ollama 加载 3B 轻量翻译大模型，或者按量计费调用大厂 API，端到端延迟控制在 800ms 内。
