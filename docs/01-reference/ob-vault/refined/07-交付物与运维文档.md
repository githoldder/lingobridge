## 📊 四、 测试数据采集指南

### 4.1 采集目标与规模规划
本阶段旨在建立一套高保真、可量化、可用于模型训练的留学生汉语学习录音资产库：
- **试采阶段 1**（2-3名学生）：验证公网访问下录音上传稳定性，测试网络延时。
- **试采阶段 2**（8-10名学生）：覆盖哈萨克斯坦等主流生源国，采用不同设备进行测试（Chrome/Safari等）。
- **小班数据采集**（20-30名学生）：形成第一批完整的班级录音数据集。

### 4.2 Excel 题目导入大纲标准模版

| unit | lesson | task_id | task_type | zh_text | pinyin | translation_ru | translation_kk |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | 1 | L01-001 | pronunciation | 大家好，我叫阿合买提。 | Dajia hao, wo jiao Ahemaiti. | Здравствуйте, меня зовут Ахмет. | Сәлеметсіз бе, менің атым Ахмет. |

### 4.3 录音质量准入与验收标准
- **环境底噪**：录音环境要求安静，音频分贝背景底噪控制在 -40dB 以下。
- **音频封装格式**：强制采用轻量级 `WebM/Opus` 封装，确保采样率不低于 16kHz。
- **单条音频长度**：跟读时长限制在 3秒 - 30秒 之间，严禁提交长时间空白的无效数据。

## 🏆 六、 商赛答辩策略与毒舌评委问答

### 6.1 答辩原则 (黄金法则)
- **先接纳后转折**：承认当前是 MVP 产品，以极高的教学同理心认可评委的顾虑。先抛出可落地的结论，再补足行研与试点数据，不与评委争输赢。

### 6.2 18道高频毒舌评委模拟问答

#### Q1：这不就是一个换壳套皮的在线网课平台吗？有什么创新？
- **答**：您说得很专业。如果只看课件浏览，确实很像网课平台。但 LingoBridge 的创新不在于大而全的网课，而在于聚焦哈萨克斯坦等留学生课后拼音、声母韵脚的“练习流程闭环”。我们将课件、TTS 标准音、防挫败 3 卡槽录音和教师“免下载”回听整合在同一条高内聚的教学链路里，目前市场上几乎没有能如此轻量解耦的垂直自学工具。

#### Q2：你们凭什么断定留学生真的有这个需求？
- **答**：这个痛点来自于一线教师与留学生的真实教学反馈。留学生一学期虽然有 196 课时，但每周只上一节课，平时缺乏普通话发音练习环境。老师在课后只能通过微信群收录音，管理极其混乱。课后跟读自测和录音的结构化留痕，是极高频的真实教务刚需。

#### Q3：市场这么小，只做俄语学中文，有什么商用意义？
- **答**：在“一带一路”支持热点下，来华留学生规模不断扩大。在小市场中切入，我们能以接近为 0 的极低成本跑通商业验证。并且我们的底层架构设计是**通用平台版（业务无关）**，这套“课件驱动+录音自测”的模式能秒级解耦复用至任何外语垂直场景（如中亚学英语、学韩语等），未来的边际效应极大。

#### Q4：已经有学习通、腾讯会议、雨课堂了，你们怎么活？
- **答**：雨课堂等主攻通用大班课堂管理，腾讯会议主攻在线会议。LingoBridge 的定位不是替代它们，而是形成“合围之势”。例如，老师上完腾讯会议后，LingoBridge 充当课后课件浏览、音频录制资产沉淀及作业跟读自测的细分流程平台，是极佳的流程互补。

#### Q5：老师会不会觉得多用一个系统很麻烦，不愿意用？
- **答**：这是最核心的教学采用风险。所以 LingoBridge 在设计上极为节制，老师只需做三件事：上传现有课件、下发 Excel 大纲、在线播放学生录音。我们绝对不做大而全的系统强迫老师改变教学习惯，而是顺应其已有工作流，提供极佳的减负体验。

#### Q6：学生为什么不用微信直接给老师发录音，还要用你们的网站？
- **答**：单个学生确实用微信发更方便，但当面对 20 人网课班级、上百个留学生时，老师的微信很快就会被淹没，录音也无法与课时的具体课件页码进行对应。系统最大的价值不仅是录音，而是**结构化的留痕管理**，使录音、页码、学生档案精准匹配。

#### Q7：你们为什么不做 AI 自动语音评分纠错？
- **答**：对于俄语口音的留学生而言，现有的 ASR 识别打分极易因为轻微的发音偏离而给出“不合格”的低分，这会严重挫败留学生的学习自信。我们认为，在 MVP 阶段，提供高保真的 3 卡槽自测对比，让有温度的老师能“免下载”回听指导，是比冷冰冰的 AI 误判打分在教学上更为负责任的第一步。

#### Q8：你们的 3 卡槽录音自测有什么技术价值？
- **答**：它完美体现了“精益 (Lean) 教学”的设计理念。留学生在提交前有 3 次独立机会自测，极大降低了由于外部杂音、发音不准导致的“废片率”，极大减少了服务器带宽和对象存储的计算与存储负担，避免将废数据传入后台。

#### Q9：现场网络如果崩了，你们如何自救？
- **答**：我们准备了**三层兜底策略**：第一层，展示部署在 `101.34.72.227` 腾讯云公网环境的线上版本；第二层，若跨国或现场网络异常，立刻切换至本地 `localhost` 的完整容器版本演示；第三层，若完全断网，我们备齐了全套高清实机截图与录屏，保证答辩叙事 100% 不被打断。

## 🛠️ 七、 运维部署与数据备份指南

### 7.1 服务日常重启与监控
- **一键启动/重启**：
  ```bash
  cd /opt/lingobridge
  docker-compose down
  docker-compose up -d --build
  ```
- **查看容器日志**：
  ```bash
  docker-compose logs -f backend
  ```

### 7.2 每日数据库自动备份脚本 (`/opt/lingobridge/backup.sh`)

```bash
#!/bin/bash
# LingoBridge 数据库每日备份脚本
BACKUP_DIR="/opt/lingobridge/backups"
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
DB_CONTAINER="lingobridge-db"
DB_USER="lingo"
DB_NAME="lingobridge"

mkdir -p $BACKUP_DIR

# 执行 pg_dump
docker exec -t $DB_CONTAINER pg_dump -U $DB_USER $DB_NAME > $BACKUP_DIR/backup_$TIMESTAMP.sql

# 仅保留最近 7 天的备份，防止磁盘溢出
find $BACKUP_DIR -type f -name "*.sql" -mtime +7 -exec rm {} \;

echo "LingoBridge DB Backup completed successfully at $TIMESTAMP."
```
