# 2026-06-02 LaTeX 大作业报告注入审计

## 结论

上一版 agent 的交付不合格，不能直接作为课程大作业报告生产管线使用。

主要原因：

1. 将 01-FA 至 07-UM 的软件工程生命周期文档整块拼接进课程报告，导致报告从 27 页模板膨胀到 77 页，偏离人工智能课程大作业的“算法、数据、实验、结果分析”重点。
2. 虽然保留了 5 个一级章名，但二级标题被大量替换为“可行性、需求规格、接口设计、部署验证”等软件工程文档标题，破坏了 `第17组.pdf` 的真实模板骨架。
3. 所谓 `standard_illustrations` 没有来源、页码、URL 或标准号截图依据，且文件元数据表现为 JPEG 数据伪装成 `.png`，疑似 AI 生成图，不可作为国标/行标/UML 标准对照图。
4. 注入脚本使用硬编码绝对路径，并直接覆盖目标章节文件，缺少模板骨架校验、dry-run 预览和回滚护栏。

## 模板真实骨架

真实模板来源：`docs/05-lecture-delivery/报告/第17组.pdf` 与 `data/chap01.tex.bak` 至 `data/chap05.tex.bak`。

报告骨架：

- 第 1 章 绪论：国内研究背景、国外研究背景、研究目的
- 第 2 章 系统设计与方法：方法设计、算法建模、模型训练、多人/多场景策略
- 第 3 章 系统具体实现：开发环境、端侧/服务端实现、算法工程化、UI 展示
- 第 4 章 实验：实验环境、数据集、准确率/鲁棒性/消融/延迟等评估
- 第 5 章 总结与展望：全文总结、运行效果展示、研究不足与未来展望

后续注入必须围绕这个骨架写“基于机器学习的国际中文学习行为分析与个性化练习推荐系统”，不得再把软件工程文档目录当作课程报告目录。

## 标准来源策略

可用来源分级：

- S 级：国家标准公开系统、OMG 官方规范 PDF、学校图书馆/数据库、课程教师指定教材。
- A 级：标准机构或出版社页面，可用于核验标准号、名称、状态。
- B/C 级：论坛、下载站、博客、二次整理页面，只能作为检索线索，不能作为仓库内标准截图来源。

已确认的可信线索：

- 国家标准公开系统 GB/T 9385-2008 页面：`https://openstd.samr.gov.cn/bzgk/gb/newGbInfo?hcno=2790825C43AD0B69E3C38C140BFFCFE6`
- OMG UML 2.5.1 官方入口：`https://www.omg.org/spec/uml/`
- GB/T 1526-1989 可用于核验数据流图/流程图符号，但全文截图应优先通过学校数据库、官方授权 PDF 或人工手动截图完成。
