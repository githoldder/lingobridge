## 常见错误

1. **把实现细节写进 SRS**：如写了具体的数据库表名、类名、方法名 → 应只描述功能行为
2. **需求不可测试**：如"系统要快" → 应改为"响应时间≤500ms"
3. **遗漏异常流程**：只写了正常流程 → 应补充异常和边界条件
4. **需求描述模糊**：使用"大约""可能""也许" → 应改为精确描述
