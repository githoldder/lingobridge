#### 2.1.2 类设计

**UserService 类**

| 属性 | 说明 |
|---|---|
| 包/命名空间 | services.user |
| 职责 | 用户相关业务逻辑 |
| 主要方法 | createUser, getUserById, updateUser |

```typescript
class UserService {
    // 创建用户
    async createUser(data: CreateUserDTO): Promise<User>
    
    // 根据ID获取用户
    async getUserById(id: string): Promise<User | null>
    
    // 更新用户信息
    async updateUser(id: string, data: UpdateUserDTO): Promise<Boolean>
    
    // 用户登录
    async login(phone: string, code: string): Promise<LoginResult>
}
```

#### 2.1.3 方法详细设计

**createUser 方法**

| 属性 | 内容 |
|---|---|
| 方法签名 | `async createUser(data: CreateUserDTO): Promise<User>` |
| 功能描述 | 创建新用户，验证手机号唯一性 |
| 输入参数 | `data: CreateUserDTO` |
| 返回值 | `Promise<User>` |
| 异常 | `UserAlreadyExistsError` - 手机号已注册 |

```
处理逻辑：
1. 验证 data.phone 格式
2. 检查手机号是否已注册（查询 users 集合）
3. 若已注册，抛出 UserAlreadyExistsError
4. 生成 _id（ObjectId）
5. 插入 users 集合
6. 返回新用户对象
```

#### 2.1.4 数据库操作

| 操作类型 | 集合 | 查询条件 | 更新内容 |
|---|---|---|---|
| INSERT | users | - | nickName, phone, createdAt |
| SELECT | users | { _id: id } | - |
| UPDATE | users | { _id: id } | { nickName, updatedAt } |

## 3. 接口详细设计

**基本信息**

| 属性 | 内容 |
|---|---|
| 接口路径 | /api/xxx/yyy |
| HTTP方法 | POST |
| 认证要求 | 是/否 |

**请求规格**

```typescript
// 请求参数
interface XXXRequest {
    field1: string    // 必填，字段1说明
    field2?: number   // 可选，字段2说明
}
```

**响应规格**

```typescript
// 成功响应 (200)
interface XXXSuccessResponse {
    code: 0
    message: 'success'
    data: {
        // 响应数据
    }
}

// 错误响应
interface XXXErrorResponse {
    code: number      // 错误码
    message: string   // 错误信息
    data: null
}
```

**错误码**

| 错误码 | 说明 | 处理方式 |
|---|---|---|
| 40001 | 参数错误 | 返回错误字段信息 |
| 40101 | 未授权 | 引导用户登录 |
| 40401 | 资源不存在 | 返回友好提示 |
| 50001 | 服务器错误 | 记录日志，返回通用错误 |
