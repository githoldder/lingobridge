# 文档质量审查清单

> **版本：** v1.0  
> **适用场景：** 重构后的文档管理 Skill 全流程质量控制  
> **使用说明：** 本清单覆盖文档从创建到归档的全生命周期，支持 AI 辅助审查与人工审查两种模式  

---

## 一、文档质量评分模型（10分制）

### 1.1 综合评分计算公式

```
综合得分 = Σ(维度得分 × 权重) - 否决项扣分

最终等级：
  9.0 – 10.0  ★★★★★  卓越（Excellent）
  8.0 – 8.9   ★★★★   良好（Good）
  7.0 – 7.9   ★★★    合格（Adequate）
  6.0 – 6.9   ★★     欠佳（Needs Improvement）
  < 6.0       ★      不合格（Unacceptable）
```

### 1.2 九大维度权重分配

| 序号 | 维度 | 权重 | 满分 |
|------|------|------|------|
| 1 | 结构完整性 | 15% | 1.5 |
| 2 | 内容准确性 | 20% | 2.0 |
| 3 | 逻辑连贯性 | 12% | 1.2 |
| 4 | 标准符合性 | 10% | 1.0 |
| 5 | 证据充分性 | 8% | 0.8 |
| 6 | 可执行性 | 12% | 1.2 |
| 7 | 可维护性 | 8% | 0.8 |
| 8 | 可读性 | 8% | 0.8 |
| 9 | 版本可追踪性 | 7% | 0.7 |
| **合计** | | **100%** | **10.0** |

## 二、九大评分维度及评分细则

### 维度 1：结构完整性（满分 1.5 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 1.5 | 包含全部必要章节：标题、目录（如适用）、正文、结论、参考文献、附录；层次结构清晰（≤3级标题）；信息无缺失 |
| 良好 | 1.2 | 主要章节完整，层次结构清晰，有轻微遗漏（如缺少附录说明） |
| 合格 | 0.9 | 核心章节存在，结构基本可读，但层次混乱或部分章节残缺 |
| 欠佳 | 0.5 | 主要章节严重残缺，结构混乱，难以理解整体框架 |
| 不合格 | 0 | 无明显结构，或仅有零散段落堆砌 |

**扣分细则：**
- 缺少摘要/概述：-0.3
- 缺少结论/总结：-0.3
- 缺少版本信息/作者/日期：-0.2
- 标题层级跳级（例：# 直接跳到 ###）：-0.2
- 目录与正文标题不一致：-0.2

---

### 维度 2：内容准确性（满分 2.0 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 2.0 | 所有事实、数据、引用均准确无误；无概念混淆；与权威来源一致 |
| 良好 | 1.6 | 主体内容准确，仅有1-2处微小错误（笔误、数值偏差） |
| 合格 | 1.2 | 核心内容准确，但存在少量事实性错误或过时信息 |
| 欠佳 | 0.8 | 存在多处事实性错误，部分数据不可靠 |
| 不合格 | 0 | 大量关键事实错误，或核心概念定义错误 |

**扣分细则：**
- 每处事实性错误（经核实）：-0.3
- 过时信息未标注（如废弃标准、已变更法规）：-0.3
- 引用来源不可查证：-0.3
- 数据与图表不一致：-0.3
- 专业术语使用错误：-0.2/处

---

### 维度 3：逻辑连贯性（满分 1.2 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 1.2 | 论点→论据→结论链条完整；段落之间衔接自然；无逻辑跳跃或自相矛盾 |
| 良好 | 1.0 | 整体逻辑清晰，存在1-2处过渡不流畅 |
| 合格 | 0.7 | 逻辑基本通顺，但因果关系或递进关系不明确 |
| 欠佳 | 0.4 | 前后矛盾或逻辑混乱，部分段落与主题无关 |
| 不合格 | 0 | 完全缺乏逻辑组织，论点散乱 |

**扣分细则：**
- 段落间缺乏过渡语句：-0.2
- 论证链断裂（提出观点后无支撑直接跳转）：-0.3
- 前后矛盾（前后描述不一致）：-0.4
- 离题段落未删除或说明：-0.2

---

### 维度 4：标准符合性（满分 1.0 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 1.0 | 完全符合文档分类对应的标准规范（如国标格式、团队模板、行业规范） |
| 良好 | 0.8 | 基本符合，有1-2处轻微偏差 |
| 合格 | 0.6 | 符合主要标准，关键格式规范到位 |
| 欠佳 | 0.3 | 存在多处标准偏差 |
| 不合格 | 0 | 完全不符合既定标准规范 |

**扣分细则：**
- 命名规范不符合要求：-0.2
- 元数据缺失或填写错误：-0.2
- 格式（字体、字号、缩进）不统一：-0.2
- 缺少必要的标准声明或版本标记：-0.2

---

### 维度 5：证据充分性（满分 0.8 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 0.8 | 所有关键论点均有可靠证据支撑；引用来源权威且可查证；数据和案例来源明确 |
| 良好 | 0.6 | 主要论点有证据支撑，少量论点缺乏直接证据 |
| 合格 | 0.5 | 核心论点有证据，但引用质量参差 |
| 欠佳 | 0.3 | 大量论点缺乏证据支撑，依赖主观判断 |
| 不合格 | 0 | 几乎无证据，纯个人意见堆砌 |

**扣分细则：**
- 关键论点无任何引用：-0.3
- 引用来源权威性不足（如非正式来源）：-0.2
- 数据来源未标注：-0.2
- 案例缺乏可验证性：-0.2

---

### 维度 6：可执行性（满分 1.2 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 1.2 | 操作步骤具体、明确、可直接执行；前置条件清晰；预期结果可验证 |
| 良好 | 1.0 | 步骤基本可执行，存在1-2处模糊描述 |
| 合格 | 0.8 | 主体流程可执行，但部分步骤需补充才能操作 |
| 欠佳 | 0.5 | 步骤描述笼统，需要大量补充说明才能执行 |
| 不合格 | 0 | 无法指导实际操作 |

**扣分细则：**
- 步骤缺少操作对象（未说明对什么操作）：-0.3
- 缺少前置条件说明：-0.2
- 缺少预期结果/验收标准：-0.2
- 命令或代码不可执行（语法错误、参数缺失）：-0.4
- 超时时间、阈值等关键参数未定义：-0.2

---

### 维度 7：可维护性（满分 0.8 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 0.8 | 模块化结构，变更影响范围明确；使用变量/占位符避免硬编码；维护责任人和周期清晰 |
| 良好 | 0.6 | 结构较清晰，主要依赖关系可追踪 |
| 合格 | 0.5 | 基本可维护，但存在少量耦合或硬编码 |
| 欠佳 | 0.3 | 高度耦合，变更影响不可预测 |
| 不合格 | 0 | 无结构设计，变更即破坏 |

**扣分细则：**
- 关键参数硬编码未提取：-0.2
- 缺少变更记录区（changelog）：-0.2
- 缺少维护责任人信息：-0.2
- 文档间交叉引用失效或缺失：-0.2

---

### 维度 8：可读性（满分 0.8 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 0.8 | 语言简洁精准；图表丰富且清晰；术语一致；目标读者匹配 |
| 良好 | 0.6 | 语言通顺，偶有冗余或晦涩表述 |
| 合格 | 0.5 | 可读性基本达标，存在部分难懂段落 |
| 欠佳 | 0.3 | 语言晦涩，大量冗余信息干扰阅读 |
| 不合格 | 0 | 几乎无法正常阅读理解 |

**扣分细则：**
- 同一术语有多种表述（不一致）：-0.2
- 大量长难句未拆分（>3行无句号）：-0.2
- 缺少必要图表说明（图表标题、单位、图例）：-0.2
- 错别字每处：-0.1（最高扣0.4）
- 标点符号错误：-0.1

---

### 维度 9：版本可追踪性（满分 0.7 分）

**评分标准：**

| 等级 | 得分 | 描述 |
|------|------|------|
| 优秀 | 0.7 | 完整的版本历史；每次变更均有记录（日期、变更内容、变更人）；可追溯到任意历史版本 |
| 良好 | 0.55 | 版本历史基本完整，主要变更有记录 |
| 合格 | 0.4 | 有版本号，但变更记录不完整 |
| 欠佳 | 0.25 | 版本混乱或几乎无版本管理 |
| 不合格 | 0 | 无任何版本信息 |

**扣分细则：**
- 无版本号：-0.3
- 无变更日志（changelog）：-0.3
- 版本号命名混乱不符合规范：-0.2

## 三、一票否决项

> 以下任意一项触发，文档立即判定为**不合格（★）**，需整改后方可流转。

| # | 否决项 | 说明 |
|---|--------|------|
| 1 | **事实性错误** | 核心内容存在已被证伪的事实，或关键数据与权威来源严重不符 |
| 2 | **安全/合规风险** | 包含违规内容（泄密信息、侵权内容、违反法规的政策解读等） |
| 3 | **操作性危害** | 步骤/命令执行后可能造成数据丢失、系统故障、人身伤害等不可逆后果 |
| 4 | **严重过时** | 引用标准/法规/版本已废止/作废，且无任何提示，可能导致读者误用 |
| 5 | **版权侵权** | 直接复制未授权内容且无合理使用声明 |
| 6 | **关键信息缺失** | 缺失必要元数据（作者、日期、版本号）导致文档无法被正确归档和使用 |
| 7 | **逻辑完全断裂** | 文档结构完全混乱，任何人无法从中提取有效信息 |
| 8 | **交叉引用完全失效** | 多个关键内部引用链接失效，且无替代说明 |
| 9 | **正式 PDF 页面大面积异常留白** | 因表格、图片、公式或浮动体过大导致某页主体内容被挤到下一页，且非封面、目录、章节首页、参考文献起始页等模板性例外 |
| 10 | **参考文献不可验证或正文未引用** | 文后条目无法通过 DOI、URL、ISBN、标准号或权威数据库独立检索，或没有在正文中被实际引用 |

## 四、常见问题库（20个常见问题 + 解决方案）

| # | 问题类型 | 问题描述 | 解决方案 |
|---|----------|----------|----------|
| 1 | 结构问题 | 文档缺少目录，长文档阅读时难以定位 | 添加自动目录或手动目录；长文档（>2000字）必须提供目录 |
| 2 | 结构问题 | 章节编号不连续或重复（如 1.2 → 1.2.1 → 1.3） | 统一编号体系，使用规范的多级编号 |
| 3 | 内容问题 | 同一术语在不同章节表述不一致 | 建立术语表，全文统一替换为标准术语 |
| 4 | 内容问题 | 引用来源过时（标准已更新） | 标注原引用时间，主动提示"请以最新版本为准" |
| 5 | 内容问题 | 数据和图表数据不对应 | 逐一核对图表与正文数据，交叉验证 |
| 6 | 逻辑问题 | 段落之间缺乏过渡，跳跃式叙述 | 添加过渡句或承上启下段落 |
| 7 | 逻辑问题 | 论点与论据不匹配 | 重新审视论证结构，确保每条论点都有直接证据支撑 |
| 8 | 逻辑问题 | 结论与正文内容不符 | 核查结论是否忠实反映正文分析结果，修正结论 |
| 9 | 标准问题 | 命名不符合规范 | 对照团队/国标命名规范重命名，批量替换 |
| 10 | 标准问题 | 元数据填写不完整 | 补充完整元数据（作者、创建日期、版本、标签等） |
| 11 | 证据问题 | 关键观点缺乏引用 | 补充权威来源引用，或标注"经验判断"并说明依据 |
| 12 | 证据问题 | 引用格式不统一 | 按统一引用规范（GB/T 7714等）格式化所有引用 |
| 13 | 可执行问题 | 命令缺少参数说明 | 为每条命令补充参数说明、示例输出和错误处理 |
| 14 | 可执行问题 | 缺少错误处理说明 | 添加常见错误场景、错误信息和解决路径 |
| 15 | 可维护问题 | 硬编码阈值/路径未提取为变量 | 重构为配置项或变量，文档中引用变量名 |
| 16 | 可维护问题 | 变更未记录 | 在 changelog 区域补录历史变更，按时间倒序排列 |
| 17 | 可读性问题 | 大量长句未拆分 | 将超过3行的句子拆分为短句，每句表达一个意思 |
| 18 | 可读性问题 | 图表缺少标题和说明 | 为每个图表添加编号、标题、数据来源和必要注释 |
| 19 | 版本问题 | 版本号混乱无法判断新旧 | 采用语义化版本（v1.0.0），建立版本更新规则 |
| 20 | 综合问题 | 文档已归档但内容仍在更新 | 建立归档→激活流程，归档文档更新需走变更流程 |

## 五、AI 辅助审查 Prompt

> **使用方式：** 将以下 Prompt 完整复制到 AI 对话中，并将待审查文档内容替换到 `{文档内容}` 部分。

```
## 角色设定
你是一位资深文档质量审查专家，擅长依据标准化审查清单对技术文档进行系统性评估。

## 任务
请对以下文档进行完整的质量审查，输出结构化审查报告。

## 文档内容
```
{文档内容}
```

## 审查维度（共9个）
1. 结构完整性（权重15%）
2. 内容准确性（权重20%）
3. 逻辑连贯性（权重12%）
4. 标准符合性（权重10%）
5. 证据充分性（权重8%）
6. 可执行性（权重12%）
7. 可维护性（权重8%）
8. 可读性（权重8%）
9. 版本可追踪性（权重7%）

## 审查要求

### 第一步：逐维度评分
针对每个维度，输出：
- 该维度得分（满分10分制）
- 评分依据（具体引用文档内容说明扣分原因）
- 该维度的主要问题列表

### 第二步：综合评分
按以下公式计算综合得分（满分10分）：
综合得分 = Σ(维度得分 × 权重)
并给出等级（★★★★★卓越 / ★★★★良好 / ★★★合格 / ★★欠佳 / ★不合格）

### 第三步：一票否决项检查
逐项检查以下8项否决条件，如有任何一项触发，明确指出：
1. 存在已被证伪的事实性错误
2. 包含安全/合规风险内容
3. 存在可能导致不可逆后果的操作步骤
4. 引用已废止的标准/法规/版本
5. 存在版权侵权内容
6. 关键元数据（作者/日期/版本号）缺失
7. 文档结构完全混乱
8. 多个关键内部引用失效

### 第四步：问题汇总
列出文档中所有发现的问题，按严重程度分为：
- 🔴 严重问题（影响可用性/准确性）
- 🟡 中等问题（影响规范性/可读性）
- 🟢 轻微问题（优化建议）

### 第五步：改进建议
针对每个严重和中等问题，提供具体可操作的修改建议。

### 输出格式要求
使用 Markdown 格式输出，结构清晰，便于直接引用。
```
