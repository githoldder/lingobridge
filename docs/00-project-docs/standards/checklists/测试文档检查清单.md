# 测试文档检查清单

> 本清单用于测试计划、测试用例、测试报告的审查。

---

## A. 测试计划检查清单

### A.1 测试范围
- [ ] 测试范围明确（测什么）
- [ ] 不测试范围明确（不测什么）
- [ ] 测试对象定义清晰（功能模块/系统）

### A.2 测试策略
- [ ] 测试类型完整（单元/集成/系统/验收）
- [ ] 每种测试类型有明确目标
- [ ] 测试方法选择有依据

### A.3 测试环境
- [ ] 硬件环境明确
- [ ] 软件环境（操作系统/中间件/依赖）明确
- [ ] 测试数据准备方案明确

### A.4 测试资源
- [ ] 人员分配明确
- [ ] 进度安排合理（含里程碑）
- [ ] 风险与对策完整

---

## B. 测试用例检查清单

### B.1 基本信息
- [ ] 每个用例有唯一ID（如 TC_ORDER_001）
- [ ] 有用例标题（简洁描述）
- [ ] 有优先级（P0/P1/P2）
- [ ] 有前置条件

### B.2 用例设计
- [ ] 用例覆盖所有功能点
- [ ] 包含正向测试（正常流程）
- [ ] 包含逆向测试（异常/边界/错误输入）
- [ ] 包含边界值测试（最大/最小/边界值）
- [ ] 包含等价类划分（代表性输入）
- [ ] 测试步骤描述清晰、可执行

### B.3 预期结果
- [ ] 每个步骤有明确的预期结果
- [ ] 预期结果可验证（有明确的判断标准）
- [ ] 有性能预期的量化值（适用时）

### B.4 可执行性
- [ ] 测试用例可重复执行
- [ ] 无歧义表达（不同人执行结果一致）
- [ ] 测试数据具体（而非泛指）

---

## C. 测试报告检查清单

### C.1 执行情况
- [ ] 测试用例总数、已执行数、通过数、失败数明确
- [ ] 测试覆盖率说明
- [ ] 测试环境与计划一致

### C.2 缺陷分析
- [ ] 缺陷按严重程度分类（P0致命/P1严重/P2一般/P3轻微）
- [ ] 缺陷按模块分布统计
- [ ] 缺陷趋势分析（新增/修复/遗留）

### C.3 测试结论
- [ ] 有明确的测试结论（通过/不通过/有条件通过）
- [ ] 有遗留问题说明及风险评估
- [ ] 有验收建议

---

## D. 测试文档质量自检

- [ ] 测试用例与需求规格（SRS）一一对应
- [ ] 可建立需求追踪矩阵（需求→用例→结果）
- [ ] 测试数据真实可信（模拟真实场景）
- [ ] 测试报告数据与实际执行结果一致
- [ ] 缺陷单描述清晰（可复现）
- [ ] 缺陷单可重现率≥ 90%
