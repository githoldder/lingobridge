# 软件设计文档检查清单

> 本清单用于概要设计（HLD）和详细设计（LLD）文档的审查。依据 GB/T 8567—2006《计算机软件文档编制规范》。

---

## A. 概要设计（HLD）检查清单

### A.1 引言
- [ ] 明确设计目标和范围
- [ ] 明确与需求的对应关系
- [ ] 定义术语和缩略语

### A.2 总体设计
- [ ] 有系统架构图
- [ ] 有模块划分及职责说明
- [ ] 有模块间接口概览
- [ ] 技术选型有依据（性能/成本/团队能力）

### A.3 接口设计
- [ ] 有模块间接口概览
- [ ] 接口参数定义清晰（名称、类型、含义、约束）
- [ ] 错误码定义完整
- [ ] 有异常处理策略

### A.4 数据设计
- [ ] 有数据库选型说明
- [ ] 有整体 ER 图或数据模型
- [ ] 有数据分布策略（如分库分表）
- [ ] 有数据备份策略

### A.5 运行设计
- [ ] 有系统部署架构图
- [ ] 有业务流程图（关键业务流程）
- [ ] 有故障切换策略

### A.6 安全设计
- [ ] 有认证机制说明
- [ ] 有授权机制说明（RBAC/ABAC）
- [ ] 有数据加密策略
- [ ] 有安全审计机制

### A.7 关键技术问题
- [ ] 识别了关键技术难点
- [ ] 每个难点有解决方案或备选方案
- [ ] 风险评估完整

---

## B. 详细设计（LLD）检查清单

### B.1 模块详细设计
对每个模块检查：
- [ ] 模块职责说明清晰
- [ ] 类/函数设计有代码级描述
- [ ] 输入/输出参数完整（类型、含义、约束）
- [ ] 处理逻辑描述清晰（伪代码/流程图）
- [ ] 异常处理完整（含边界条件和异常分支）
- [ ] 调用关系正确（调用了不存在的接口/类？）
- [ ] 依赖关系正确（循环依赖？）

### B.2 数据库详细设计
对每张表检查：
- [ ] 字段命名规范
- [ ] 字段类型和长度合理
- [ ] 主键/外键定义正确
- [ ] 索引设计有依据（频繁查询字段有索引）
- [ ] 约束条件完整（NOT NULL/UNIQUE/CHECK）
- [ ] 注释完整（字段用途说明）

### B.3 接口详细设计
对每个接口检查：
- [ ] 请求参数完整（必填/可选标注）
- [ ] 请求参数类型、格式、取值范围明确
- [ ] 响应格式定义完整
- [ ] 错误码覆盖所有异常情况
- [ ] 有鉴权说明
- [ ] 有请求频率限制说明

---

## C. 设计评审自检

- [ ] 设计支持所有 SRS 需求
- [ ] 每条需求可在设计中找到对应实现
- [ ] 无过度设计（简单场景不用复杂架构）
- [ ] 无设计漏洞（遗漏了正常或异常流程）
- [ ] 模块划分合理（单一职责、内聚性高、耦合性低）
- [ ] 可直接指导编码（无模糊描述）
- [ ] 技术选型与团队能力匹配

---

## D. 常见设计问题

- [ ] 无循环依赖（A→B→C→A）
- [ ] 接口设计无版本管理（v1/v2）
- [ ] 敏感数据未加密存储
- [ ] 大数据量场景未考虑分页/分区
- [ ] 无幂等性设计（重复请求会产生错误结果）
- [ ] 无事务边界设计（跨表/跨库操作）
- [ ] 无缓存策略设计（频繁查询的热点数据）
