# 产品与项目管理文档规范

> 本模块为产品需求文档（PRD/MRD/BRD）和项目管理文档建立规范体系。

---

## 一、产品文档分类体系

| 文档类型 | 目的 | 目标读者 | 生命周期 |
|---|---|---|---|
| **BRD** | 说明业务价值，争取资源 | 高管/决策层 | 项目立项前 |
| **MRD** | 说明市场需求，指导产品方向 | 产品/管理层 | 产品规划阶段 |
| **PRD** | 精确说明产品需求，指导开发 | 开发/测试/设计 | 开发阶段 |
| **用户故事** | 从用户视角描述功能 | 开发/测试 | 迭代开发 |
| **竞品分析** | 分析竞争对手产品 | 产品团队 | 产品规划 |
| **产品路线图** | 规划产品发展方向 | 全部相关方 | 产品全周期 |

## 三、用户故事规范

### 用户故事格式

```
作为 [角色]，
我希望 [功能]，
以便 [收益/价值]。

验收标准：
- Given [前置条件] When [操作] Then [预期结果]
- Given [前置条件] When [操作] Then [预期结果]
```

### 用户故事示例

```
故事 #US-001
作为 已注册用户，
我希望 将商品加入购物车，
以便 汇总多个商品一起结算。

验收标准：
- Given 用户已登录 AND 商品有库存
  When 用户点击"加入购物车"
  Then 商品出现在购物车列表中 AND 购物车数量+1
  
- Given 用户未登录
  When 用户点击"加入购物车"
  Then 弹出登录对话框
```

## 五、文档检查清单

### PRD 检查清单
- [ ] 目标用户画像清晰
- [ ] 用户痛点有数据/调研支撑
- [ ] 每个功能有用户故事
- [ ] 每个用户故事有验收标准（Given-When-Then）
- [ ] 优先级合理（P0/P1/P2/P3）
- [ ] 非功能需求可量化
- [ ] UI/UX 有原型链接
- [ ] 风险有识别和对策
- [ ] 团队已评审并签字

### 项目计划检查清单
- [ ] 项目目标 SMART 化
- [ ] WBS 分解到可执行的任务
- [ ] 里程碑合理
- [ ] 资源计划覆盖全周期
- [ ] 风险识别完整
- [ ] 沟通计划明确
