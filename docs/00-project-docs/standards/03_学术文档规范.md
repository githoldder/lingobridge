# 学术文档规范

> 本模块为课程论文、实验报告、文献综述、开题报告、毕业论文、调研报告建立标准驱动的写作规范。
>
> **核心依据：**
> - GB/T 7713.1—2025《信息与文献 编写规则 第1部分：学位论文》
> - GB/T 7713.2—2022《学术论文编写规则》
> - GB/T 7714—2015《信息与文献 参考文献著录规则》
> - GB/T 15834—2011《标点符号用法》
> - GB 3100/3101/3102—93《量和单位》系列国家标准

## 一、学术文档分类体系

### 1.1 文档类型矩阵

| 一级分类 | 二级分类 | 参考标准 | 适用场景 | 篇幅参考 |
|---|---|---|---|---|
| 课程论文 | 课程小论文、课程报告 | GB/T 7713.2—2022 | 大学课程作业 | 3,000-8,000字 |
| 实验报告 | 实验记录、实验总结 | 课程模板 + GB/T 7713.2 | 理工科实验 | 3,000-10,000字 |
| 调研报告 | 社会调研、行业调研 | GB/T 7713.2—2022 | 课程/毕设前期调研 | 5,000-15,000字 |
| 文献综述 | 国内外研究现状 | GB/T 7713.2 + GB/T 7714 | 毕设第二章/独立成文 | 5,000-20,000字 |
| 开题报告 | 选题背景、研究方案 | 高校模板 + GB/T 7713.1 | 毕业设计开题 | 5,000-10,000字 |
| 毕业论文 | 本科学位论文 | GB/T 7713.1—2025 + GB/T 7714 | 本科毕业设计 | 15,000-30,000字 |
| 学术论文 | 期刊/会议论文 | GB/T 7713.2—2022 + GB/T 7714 | 投稿发表 | 5,000-15,000字 |

### 1.2 文档选用决策树

```
是否有原始实验/调研数据？
  ├── 是 → 需要"实验报告"或"调研报告"结构
  └── 否 → 是否为课程作业？
            ├── 是 → "课程论文"结构
            └── 否 → 是否为毕业论文？
                      ├── 是 → "学位论文"结构（完整版）
                      └── 否 → 是否为投稿？
                                ├── 是 → "学术论文"结构
                                └── 否 → "文献综述"结构
```

## 二、学位论文标准结构（GB/T 7713.1—2025）

### 2.1 完整章节层次

```
封面                  ← 封面（学校统一模板，不进入正文页码）
扉页                  ← 扉页（学校统一模板，不进入正文页码）
学位论文学位声明        ← 声明页（不进入正文页码）

摘要                  ← 页码从正文页码开始（罗马数字 i）
Abstract
目录                  ← 罗马数字页码
图目录                ← 如有
表目录                ← 如有
主要符号表              ← 如有

第1章 绪论            ← 正文第1章，阿拉伯数字页码
  1.1 研究背景
  1.2 国内外研究现状
  1.3 研究目的与意义
  1.4 主要研究内容
  1.5 论文组织结构

第2章 相关技术与理论基础
  2.1 [技术A]
  2.2 [技术B]
  2.3 ...

第3章 [核心研究章节A]
  3.1 问题定义
  3.2 方法设计
  3.3 实现与验证

第4章 [核心研究章节B]
  4.1 ...
  4.2 ...

第5章 总结与展望
  5.1 主要工作总结
  5.2 创新点
  5.3 不足与展望

参考文献
附录A [补充材料]        ← 可选
附录B [核心代码]        ← 可选（如算法实现）
致谢
作者简历              ← 部分高校要求
```

### 2.2 各章节核心要求

| 章节 | 核心问题 | 常见问题 | 字数参考 |
|---|---|---|---|
| 摘要 | 研究了什么？怎么做？结果如何？ | 含图表、参考文献、"本文" | 300-500字 |
| 第1章 绪论 | 为什么要做？ | 背景过大、研究问题不聚焦 | 2,000-4,000字 |
| 第2章 理论基础 | 用什么理论/技术做？ | 写成技术说明书，缺乏针对性 | 2,000-5,000字 |
| 第3-4章 核心章节 | 怎么做的？做出了什么？ | 内容薄弱、缺乏实验验证 | 各4,000-8,000字 |
| 第5章 总结 | 做得怎样？创新点在哪？ | 创新点夸大、缺乏数据支撑 | 1,000-2,000字 |

### 2.3 页码规范（GB/T 7713.1—2025）

| 部分 | 页码格式 | 说明 |
|---|---|---|
| 前置部分（摘要至目录） | 罗马数字（i, ii, iii…） | 不计入正文总页数 |
| 正文部分（第1章起） | 阿拉伯数字（1, 2, 3…） | 连续编码 |
| 参考文献 | 阿拉伯数字 | 与正文连续 |
| 附录 | 阿拉伯数字 | 与正文连续 |
| 致谢/简历 | 阿拉伯数字 | 与正文连续 |

## 三、学术论文标准结构（GB/T 7713.2—2022）

> 用于期刊投稿和会议论文。篇幅通常为5,000-15,000字。

```
标题
  └─ 准确、简洁、清晰；不超过25个汉字；可加副标题

作者信息
  └─ 姓名、单位（高校到院/系一级）、邮编；通信作者标注†或*

摘要
  └─ 目的、方法、结果、结论（四要素）；200-300字

Abstract
  └─ 与中文摘要内容完全对应

关键词 / Keywords
  └─ 3-8个；中英文对应；尽量用规范术语

1 引言（或"前言"）
  └─ 研究背景、研究现状、研究空白、本文贡献

2 [正文部分—视期刊要求灵活组织]
  └─ 常见结构：
     - 研究方法 + 结果分析（理科）
     - 理论框架 + 案例分析（文科）
     - 系统架构 + 实验验证（工科）

3 结论

参考文献

附录（必要时）
```

## 四、文献综述结构

> 独立成文时为"文献综述"；作为毕业论文第2章时内容可精简。

### 4.1 完整结构

```
1 引言
  1.1 研究背景
  1.2 研究意义
  1.3 综述范围与界定

2 研究现状
  2.1 [主题A] 研究
    2.1.1 国外研究进展
    2.1.2 国内研究进展
  2.2 [主题B] 研究
    2.2.1 国外研究进展
    2.2.2 国内研究进展
  2.3 [主题C] 研究
  ……

3 研究述评
  3.1 现有研究的主要贡献
  3.2 现有研究的不足（空白/争议/方法局限）
  3.3 未来研究发展趋势

4 结论
  └─ 总结本文综述了哪些主题、梳理了哪些脉络、发现了什么 gap

参考文献
```

### 4.2 写作规范

| 规范项 | 要求 |
|---|---|
| 文献数量 | 硕士≥50篇，博士≥100篇；核心期刊/高水平文献≥60% |
| 时效性 | 近5年文献≥40%；经典文献可不受时效限制 |
| 组织方式 | **按主题（Topic）组织**，不按国别或年份平铺 |
| 每篇文献写法 | 作者（年份）。核心观点。贡献/不足（你的评价） |
| 禁止 | 直接复制文献摘要；仅罗列"XX研究了XX"而无分析 |
| 引用格式 | GB/T 7714—2015；与毕业论文正文引用统一 |

### 4.3 文献综述质量评估表

| 评估维度 | 合格 | 良好 | 优秀 |
|---|---|---|---|
| 覆盖广度 | 基本覆盖主要研究 | 覆盖全面，有遗漏 | 全面系统，无重大遗漏 |
| 分类逻辑 | 有分类但逻辑较乱 | 分类清晰，逻辑通顺 | 分类创新，逻辑严密 |
| 述评深度 | 简单罗列 | 有评述但深度不足 | 评述深刻，能发现gap |
| 时效性 | 近5年<30% | 近5年40%-60% | 近5年>60% |
| 格式规范 | 有格式错误 | 基本规范 | 完全规范 |

## 五、开题报告结构

> 各高校模板略有差异，以下为通用结构，标注★的为核心章节。

```
1 选题背景与依据 ★
  1.1 课题来源（纵向/横向/自选）
  1.2 研究背景
  1.3 选题依据

2 国内外研究现状 ★
  └─ 即文献综述（精练版，2000-4000字）

3 研究目标与内容 ★
  3.1 研究目标 ★
  3.2 主要研究内容 ★

4 研究方案与技术路线 ★
  4.1 研究方法 ★
  4.2 技术路线 ★（建议附流程图）
  4.3 拟解决的关键科学/技术问题 ★

5 预期成果与创新点 ★
  5.1 预期成果
  5.2 创新点（2-3条，每条1-2句话）

6 研究进度安排 ★
  └─ 甘特图或时间节点表

7 研究条件与可行性分析
  7.1 已有研究基础
  7.2 实验条件/数据来源
  7.3 可行性分析

8 参考文献
```

### 创新点撰写规范

**禁止写法（套话）：**
- "国内首次系统研究……"
- "填补了国内外空白"
- "具有重大理论意义和现实意义"

**合格写法（具体+可验证）：**
- "针对现有XX方法在XX场景下存在的XX问题，提出了一种XX方法（技术名称），通过XX机制，实现了XX指标的提升，较同类方法准确率提高X%。"
- "设计并实现了一个基于XX架构的XX系统，在XX数据集上验证了可行性，为XX领域提供了新的研究工具。"

## 六、实验报告结构

### 6.1 标准模板

```
1 实验目的
  └─ 本实验要验证什么假设/掌握什么技能/学习什么方法

2 实验环境
  2.1 硬件环境（设备型号、规格参数）
  2.2 软件环境（操作系统版本、软件版本、依赖库版本）

3 实验原理
  └─ 简述实验所依据的理论基础（公式/定律/算法框架）

4 实验步骤
  └─ 可编号分步，每步1-2句话；可用流程图辅助说明

5 实验结果与分析
  5.1 结果展示
    └─ 图表呈现（原始数据→图表→标注）
  5.2 结果分析
    └─ 与预期对比、与理论值对比、误差分析

6 讨论
  6.1 结果的合理性分析
  6.2 与已有研究/方法的对标
  6.3 局限性说明

7 结论
  └─ 用1-2段话概括实验验证的结论

8 参考文献
```

### 6.2 实验报告特殊规范

| 项目 | 规范 |
|---|---|
| 可重复性 | 实验条件须完整记录，确保他人可复现 |
| 数据呈现 | 原始数据→处理后数据→图表；保留小数位数一致 |
| 误差分析 | 须说明误差来源（仪器精度/环境/方法）和估算方式 |
| 图注 | 每张图须注明坐标系、单位、图例 |
| 表注 | 每张表须注明数据来源、测量条件 |

## 七、调研报告结构

```
1 调研背景与目的
  1.1 调研背景
  1.2 调研目的
  1.3 调研意义

2 调研设计
  2.1 调研对象
  2.2 调研方式（问卷/访谈/文献调研/实地观察）
  2.3 抽样方法与样本量
  2.4 调研实施情况（时间、地点、样本回收率）

3 调研结果
  3.1 描述性统计（频率、均值、标准差）
  3.2 交叉分析（按维度分组对比）
  3.3 典型案例/访谈摘录（匿名处理）

4 讨论
  4.1 结果解读
  4.2 与已有研究的比较
  4.3 调研局限性

5 结论与建议
  5.1 主要结论
  5.2 对策建议（分条列出，注明针对对象）

6 参考文献

7 附录
  7.1 问卷原文
  7.2 访谈提纲
  7.3 原始数据（可选）
```

### 调研报告写作要求

- **样本量说明**：须报告有效样本数和回收率
- **匿名处理**：访谈对象使用化名，注明已获得知情同意
- **图表规范**：每张图表有来源说明（"基于XX年XX调研"）
- **建议可操作**：建议须有针对性，避免"加强宣传""完善机制"等空话

## 八、格式通用规范

### 8.1 摘要规范

| 项目 | 中文摘要 | 英文摘要（Abstract） |
|---|---|---|
| 字数 | 300-500字（本科）/ 500-1000字（硕士） | 与中文对应，约300-500词 |
| 人称 | 第三人称；不得出现"本文""作者""我" | Third person; "this study" / passive voice |
| 时态 | 一般现在时/现在完成时 | Present perfect or simple past |
| 主动句 | 避免；多用"研究了""提出了""验证了" | Use passive voice or "this study..." |
| 结构 | 研究背景（1句）→研究目的（1句）→研究方法（1-2句）→研究结果（2-3句）→结论（1-2句） | Same five-part structure |
| 图表 | 不含图表 | No figures or tables |
| 引用 | 不含参考文献 | No citations |
| 数学符号 | 不使用复杂公式，必要时用文字描述 | Same |

**摘要禁止出现：**
- ❌ "本文……"（改："本研究……"或"XX方法……"）
- ❌ "我认为……"（学术写作无第一人称评价）
- ❌ "可见……"（口语化，改："结果表明……"）
- ❌ 图表、公式、参考文献
- ❌ 超出摘要范围的内容（正文才展开的内容）

**摘要参考模板（五段式）：**
```
[背景] XX领域在YY方面面临ZZ问题。
[目的] 为了解决该问题，本文提出/研究了XX方法/理论。
[方法] 该方法基于XX原理，通过XX步骤实现YY功能。
[结果] 在XX数据集上的实验结果表明，XX指标达到XX（与XX方法相比准确率提升XX%）。
[结论] 本研究验证了XX方法的可行性，为YY领域提供了参考。
```

### 8.2 关键词规范

| 规范项 | 要求 |
|---|---|
| 数量 | 3-8个（期刊）/ 5-8个（学位论文） |
| 来源 | 从标题、摘要、正文中抽取的核心术语 |
| 选择标准 | 优先使用专业词汇（Subject Headings）、避免口语化 |
| 格式 | 用分号（；）分隔，末尾无标点 |
| 英文对应 | 须与中文一一对应；可参考《汉语主题词表》或各学科规范 |

**正确示例：**
```
关键词：深度学习；图像识别；目标检测；卷积神经网络；迁移学习
Keywords: deep learning; image recognition; object detection; convolutional neural network; transfer learning
```

**错误示例：**
- ❌ 关键词之间用逗号分隔 → 中文用分号
- ❌ 关键词中出现"研究""方法"等泛化词
- ❌ 中英文关键词不对应

### 8.3 目录规范

| 项目 | 规范 |
|---|---|
| 层级 | 一般到三级标题（1.x.x），不超过四级 |
| 页码 | 右对齐，前导点（........）连接标题和页码 |
| 字体 | 与正文一致；章标题加粗 |
| 格式 | 学校模板有规定时按模板；无模板时"第X章"加粗 |
| 生成方式 | 用 Word/Zotero/LaTeX 自动生成后人工核对 |

### 8.4 图表规范（GB/T 7713.1—2025、GB/T 7713.2—2022）

#### 图表编号规则

| 类型 | 编号格式 | 示例 |
|---|---|---|
| 图（Figure） | 图X-X（第X章第X图） | 图1-1 系统架构图 |
| 子图 | 图1-1(a)、图1-1(b) 或 (a)(b) |  |
| 表（Table） | 表X-X（第X章第X表） | 表2-1 实验参数设置 |
| 跨页表 | 表头在首页，续表在左上角注明"表X-X（续）" |  |

#### 图（Figure）格式要求

```
┌──────────────────────────────────┐
│                                  │
│         [图的内容]               │
│                                  │
└──────────────────────────────────┘
           图1-1 系统架构图
```

- 居中排列，宽度不超过正文宽度的2/3（复杂架构图可通栏）
- 图题置于图下方，居中，格式：**图1-1 ××××（或Figure 1-1 ×××）**
- 引用时："如图1-1所示" / "see Figure 1-1"
- 图片清晰度：矢量图（.svg/.emf）> .png/.tiff ≥ 300dpi
- 引用来源：若为引用他人图，须在图题后加注"（来源：XXX, 2023）"

#### 表（Table）格式要求

```
表1-1 实验参数设置
┌────────────┬────────────┬────────────┐
│    参数    │    取值    │   说明     │
├────────────┼────────────┼────────────┤
│   学习率   │    0.001   │  经验选取  │
├────────────┼────────────┼────────────┤
│  批次大小  │     32     │  GPU显存限制│
└────────────┴────────────┴────────────┘
```

- 表题置于表上方，居左，格式：**表1-1 ××××（或Table 1-1 ×××）**
- 推荐使用三线表（顶线、底线、数据行间无线）
- 列标题居中加粗，数据居中或居左（文字居左，数字居右）
- 表注置于表下方，解释数据来源、缩写、未尽事宜
- 引用时："见表1-1" / "see Table 1-1"

### 8.5 公式规范

| 项目 | 规范 |
|---|---|
| 编号位置 | 公式右侧，括号括起，与右端对齐 |
| 编号格式 | (X-Y)，X为章节号，Y为该章内序号 |
| 编号内容 | 阿拉伯数字，流水编号 |
| 公式格式 | 居中，公式编号右对齐 |
| 变量说明 | 首次出现时须在正文或公式下方说明每个符号的含义 |
| 字体 | 变量用斜体（Italic），数字和运算符用正体 |
| 引用 | "如式(3-1)所示" / "as shown in Eq. (3-1)" |

**示例：**
```

其中，y为输出值，w为权重，x为输入值，b为偏置项。
```

## 十、量和单位规范（GB 3100/3101/3102—93）

### 10.1 基本规则

| 规则 | 示例 |
|---|---|
| 使用国际单位制（SI） | m（米）、kg（千克）、s（秒）、A（安培） |
| 单位符号用正体 | 5 m（不是5 m斜体m） |

| 公分 | cm（厘米） |
| hr | h（小时） |
| mole | mol（摩尔） |

### 10.3 图表中量的表示

- 坐标系须标注量符号和单位（如 "时间 t / s" 或 "t (s)"）

## 十五、完整质量检查清单

> 分阶段使用：开题阶段 → 写作阶段 → 提交前 → 答辩后终稿

### 15.1 开题阶段检查

- [ ] 研究问题具体、边界清晰（不是泛泛的"人工智能研究"）
- [ ] 国内外研究现状覆盖主要流派，有评述
- [ ] 研究目标可量化或可验证
- [ ] 技术路线图清晰，各阶段工作明确
- [ ] 创新点有依据，不是空话
- [ ] 进度安排合理，与学制匹配
- [ ] 参考文献≥30篇，近5年≥40%

### 15.2 写作阶段检查

- [ ] 每章开头有背景过渡（说明本章与前后章节的关系）
- [ ] 每章结尾有小结（总结本章做了什么，为下一章做什么铺垫）
- [ ] 所有图表在正文中都有引用说明
- [ ] 所有公式在正文中都有解释（首次出现时）
- [ ] 所有变量符号全文使用一致
- [ ] 所有引用与文后文献一一对应
- [ ] 每周写作内容备份，防止数据丢失

### 15.3 提交前终检（最高优先级）

#### 格式部分

- [ ] 全文字体、字号、行距统一（封面→目录→正文→参考文献）
- [ ] 页眉页脚正确（前置部分罗马数字，正文阿拉伯数字）
- [ ] 目录页码与正文一致（手动或自动生成后逐一核对）
- [ ] 中英文封面信息完整（学校、学院、专业、姓名、学号、导师）
- [ ] 声明页/原创性声明页签字完整

#### 摘要部分

- [ ] 中文摘要300-500字，无图表、无参考文献
- [ ] 英文Abstract与中文内容完全对应
- [ ] 中英文关键词3-8个，一一对应

#### 图表公式

- [ ] 每张图有编号（ 图X-X）、图题（在图下方）、引用说明
- [ ] 每张表有编号（ 表X-X）、表题（在表上方）、表注（如有）
- [ ] 每张图/表质量≥300dpi（矢量图最佳）
- [ ] 每个公式有编号（右对齐括号内）、变量说明
- [ ] 量和单位使用SI标准

#### 参考文献

- [ ] 全文无重复编号（Zotero/EndNote辅助核查）
- [ ] 正文引用与文献列表100%对应
- [ ] 所有文献条目格式完全统一（GB/T 7714—2015）
- [ ] 无超过10年且非经典的低相关度引用
- [ ] 外文文献翻译/缩写名检查

#### 内容部分

- [ ] 全文无错别字（朗读检查 + AI辅助）
- [ ] 标点符号全部为中文全角（参考文献除外）
- [ ] 创新点描述有实验/数据支撑
- [ ] 致谢内容已确认（答辩通过后不需修改）
- [ ] 全文逻辑连贯，无章节跳跃

#### 学术诚信

- [ ] 所有引用标注了来源
- [ ] 无大段未标注的直接引用（>50字须引号+脚注）
- [ ] AI辅助内容未作为事实来源
- [ ] 查重率低于学校要求（通常≤20-30%）
- [ ] 自我剽窃检查（不得将本人已发表内容未加引用直接放入）

### 15.4 答辩后终稿检查

- [ ] 根据答辩意见修改后页码重新更新
- [ ] 致谢确认（如答辩通过后补写）
