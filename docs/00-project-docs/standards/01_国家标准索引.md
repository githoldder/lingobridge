# 国家标准索引库

> **索引建立说明**：本索引为「国家标准驱动的文档管理与工程化写作规范」Skill 的核心知识库基础。所有 Skill 规则、模板、检测逻辑均应追溯至本清单所列标准。本文档基于训练知识库编制，如有疑义请以官方正式出版标准为准。

## 一、学位论文与学术写作

---

### GB/T 7713.1—2025 信息与文献 编写规则 第1部分：学位论文

- **类型：** 国家标准（GB/T）
- **发布日期：** 2025年
- **实施日期：** 2025年
- **当前状态：** 现行
- **主管部门：** 全国信息与文献标准化技术委员会（TC 4）
- **适用范围：** 规范学位论文的编写格式、结构要素、技术内容要求。包括：前置部分（封面、声明、摘要、关键词）、正文部分（引言、正文、结论、参考文献）、附录部分、结尾部分的编写要求。适用于学士、硕士、博士学位论文。
- **对 Skill 的用途：** 作为学位论文模板的格式基准；规范摘要、目录、参考文献的自动生成规则；检测论文章节结构的完整性。
- **是否必须纳入：** 必须
- **备注：** 2025年修订版，替代 GB/T 7713.1—1987《科学技术报告、学位论文和学术论文的编写格式》。修订后更名为"学位论文"，范围更聚焦。

---

### GB/T 7713.2—2022 学术论文编写规则

- **类型：** 国家标准（GB/T）
- **发布日期：** 2022年
- **实施日期：** 2023年
- **当前状态：** 现行
- **主管部门：** 全国信息与文献标准化技术委员会（TC 4）
- **适用范围：** 规范学术论文（含期刊论文、会议论文等）的编写规则，涵盖题名、摘要、引言、正文、结论、参考文献、附录等部分的编写要求。
- **对 Skill 的用途：** 作为通用学术写作模板的格式依据；提供论文结构检测规则；规范摘要（Abstract）的编写标准。
- **是否必须纳入：** 必须
- **备注：** 替代 GB/T 7713—1987 中学术论文部分。2022版新增了数字对象唯一标识符（DOI）、开放获取等要求。

### GB/T 7714—2015 信息与文献 参考文献著录规则

- **类型：** 国家标准（GB/T）
- **发布日期：** 2015年
- **实施日期：** 2015年
- **当前状态：** 现行
- **主管部门：** 全国信息与文献标准化技术委员会（TC 4）
- **适用范围：** 规定了文后参考文献的著录项目、著录格式、著录顺序、著录用符号等。适用于各学科领域的文后参考文献著录。
- **对 Skill 的用途：** 作为参考文献格式化模块的核心依据；自动生成 GB/T 7714—2015 格式的参考文献条目；支持著者—出版年制和顺序编码制两种著录体系。
- **是否必须纳入：** 必须
- **备注：** 替代 GB/T 7714—2005。等效采用 ISO 690:2010（Information and documentation — Guidelines for bibliographic references and citations to information resources）。

## 二、语言文字规范

---

### GB/T 15834—2011 标点符号用法

- **类型：** 国家标准（GB/T）
- **发布日期：** 2011年
- **实施日期：** 2012年
- **当前状态：** 现行
- **主管部门：** 教育部（语言文字信息管理司归口）
- **适用范围：** 规定了汉语书面语中标点符号的名称、形式和用法。涵盖句号、逗号、顿号、分号、冒号、引号、括号、破折号、省略号、连接号、间隔号、书名号、专名号等13种标点的用法。
- **对 Skill 的用途：** 作为文档内容检测的标点规范性依据；自动纠正全角/半角标点混用错误；规范引号、括号、书名号的嵌套规则。
- **是否必须纳入：** 必须
- **备注：** 替代 GB/T 15834—1995。

---

### GB/T 15835—2011 出版物上数字用法

- **类型：** 国家标准（GB/T）
- **发布日期：** 2011年
- **实施日期：** 2012年
- **当前状态：** 现行
- **主管部门：** 教育部（语言文字信息管理司归口）
- **适用范围：** 规定了出版物上汉字数字和阿拉伯数字的用法。明确何时使用阿拉伯数字、何时使用汉字数字，以及含统计数值、世纪、年代、日期、图表序号等场景的处理方式。
- **对 Skill 的用途：** 作为数字格式规范化检测的核心依据；区分应使用阿拉伯数字与汉字数字的场景；规范物理量、非物理量、计数数字的表示方式。
- **是否必须纳入：** 必须
- **备注：** 替代 GB/T 15835—1995。

## 三、量和单位

---

### GB/T 3100—2022 国际单位制及其应用

- **类型：** 国家标准（GB/T）
- **发布日期：** 2022年
- **实施日期：** 2023年
- **当前状态：** 现行
- **主管部门：** 全国量和单位标准化技术委员会（TC 12）
- **适用范围：** 规定国际单位制（SI）的基本单位、导出单位、组合单位、词头以及计量单位的使用规则。适用于科学技术和工程领域所有计量单位的规范化使用。
- **对 Skill 的用途：** 作为科技文档中量、单位、词头规范化的核心依据；自动检测物理量的单位正误（如 kg 不写作 Kg、MPa 不写作 mPa）；规范 SI 单位符号的大小写和正斜体。
- **是否必须纳入：** 必须
- **备注：** 替代 GB/T 3100—1993。2022版根据2019年SI重新定义（千克、安培等单位改由基本物理常数定义）进行了相应修订。

---

### GB/T 3101—1993 有关量、单位和符号的一般原则

- **类型：** 国家标准（GB/T）
- **发布日期：** 1993年
- **实施日期：** 1994年
- **当前状态：** 现行（被 GB/T 3101—202x 修订进程覆盖，但现行版本仍为1993版）
- **主管部门：** 全国量和单位标准化技术委员会（TC 12）
- **适用范围：** 规定科学技术领域中使用量、单位、符号的一般原则。包括量的符号、单位的符号和计量学中常用的符号与表示规则，以及数值表示规范。
- **对 Skill 的用途：** 作为物理量符号规范化（正斜体、大小写）的辅助依据；规范数值的表示与书写规则（如数值与单位之间的空格规范）。
- **是否必须纳入：** 必须
- **备注：** 正在修订过程中（202x版）。1993版仍为现行标准。本标准与 GB/T 3100 配合使用。

---

### GB/T 3102（所有部分） 量和单位

- **类型：** 国家标准（GB/T，包括多个部分）
- **各部分发布日期：** 陆续发布，主体为1993年前后发布
- **当前状态：** 现行（各部分独立更新）
- **主管部门：** 全国量和单位标准化技术委员会（TC 12）
- **适用范围：** GB/T 3102 是整套量与单位标准，分多个部分：
  - GB/T 3102.1—1993 空间和时间的量和单位
  - GB/T 3102.2—1993 周期及其有关现象的量和单位
  - GB/T 3102.3—1993 力学和 GB/T 3102 热学的量和单位
  - GB/T 3102.4—1993 电学和磁学的量和单位
  - GB/T 3102.5—1993 声学的量和单位
  - GB/T 3102.6—1993 光及有关电磁辐射的量和单位
  - GB/T 3102.7—1993 物理化学和分子物理学的量和单位
  - GB/T 3102.8—1993 原子物理学、核物理学、核反应和电离辐射的量和单位
  - GB/T 3102.9—1993 固体物理学的量和单位
  - GB/T 3102.10—1993 核反应和电离辐射的量和单位
  - GB/T 3102.11—1993 物理科学和技术中使用的数学符号
  - GB/T 3102.12—1993 特征数
  - GB/T 3102.13—1993 固体物理学
- **对 Skill 的用途：** 提供各学科领域的标准量符号（如 m 表示质量、t 表示时间）、标准单位符号及换算关系；用于科技文档的量符号自动检测与纠错。
- **是否必须纳入：** 必须（核心部分）
- **备注：** 部分正陆续修订。Skill 重点引用 GB/T 3102.1、GB/T 3102.3、GB/T 3102.4、GB/T 3102.6、GB/T 3102.8 等与工程写作最相关的部分。

---

### GB/T 8567—2006 计算机软件文档编制规范

- **类型：** 国家标准（GB/T）
- **发布日期：** 2006年
- **实施日期：** 2006年
- **当前状态：** 现行
- **主管部门：** 全国计算机标准化技术委员会软件工程分技术委员会（TC 28/SC 7）
- **适用范围：** 规定了软件文档的分类、编制要求、管理和维护要求。给出了软件生命周期各阶段应编制的文档清单，包括：可行性分析报告、项目需求说明书、概要设计说明书、详细设计说明书、数据库设计说明书、测试计划、测试报告、用户手册、操作手册、维护手册等。
- **对 Skill 的用途：** 作为软件工程文档模板库的建设依据；提供文档完整性检查规则（必备要素、可选要素）；规范软件文档的版本管理要求。
- **是否必须纳入：** 必须
- **备注：** 替代 GB/T 8567—1988。注：GB/T 8567—202x 正在修订中，新版尚未发布。

### GB/T 9385—2008 计算机软件需求规格说明规范

- **类型：** 国家标准（GB/T）
- **发布日期：** 2008年
- **实施日期：** 2009年
- **当前状态：** 现行
- **主管部门：** 全国计算机标准化技术委员会软件工程分技术委员会（TC 28/SC 7）

- **对 Skill 的用途：** 作为需求文档（PRD/SPEC）模板的格式基准；提供需求条目编写的规范性检测（完整性、无歧义性、可验证性）。
- **是否必须纳入：** 必须
- **备注：** 非等效采用 IEEE Std 830—1998（Recommended Practice for Software Requirements Specifications）。

### GB/T 14394—2008 计算机软件可靠性和可维护性管理

- **类型：** 国家标准（GB/T）
- **发布日期：** 2008年
- **实施日期：** 2009年
- **当前状态：** 现行
- **主管部门：** 全国计算机标准化技术委员会软件工程分技术委员会（TC 28/SC 7）
- **适用范围：** 规定了计算机软件可靠性和可维护性管理的要求，包括软件可靠性和可维护性管理计划、评审和评审记录、测试与分析、交付后活动等要求。适用于软件生命周期中的可靠性与可维护性管理活动。
- **对 Skill 的用途：** 作为可靠性相关文档（如测试计划、运维文档）的编写参考；提供可靠性指标描述的规范化规则。
- **是否必须纳入：** 可选
- **备注：** 替代 GB/T 14394—1993。侧重管理而非具体技术指标。

### GB/T 16260（所有部分） 软件工程 产品质量

- **类型：** 国家标准（GB/T）
- **当前状态：** 现行（各部分陆续发布）
- **主管部门：** 全国计算机标准化技术委员会软件工程分技术委员会（TC 28/SC 7）
- **适用范围：** GB/T 16260 是软件质量系列标准，分多个部分：
  - GB/T 16260.1—2006 第1部分：质量模型（定义了功能性、可靠性、易用性、效率、维护性、可移植性六大质量特性及其子特性）
  - GB/T 16260.2—2016 第2部分：外部度量
  - GB/T 16260.3—2016 第3部分：内部度量
  - GB/T 16260.4—2016 第4部分：使用质量度量
  - GB/T 16260.5—2023 第5部分：过程度量（2023年新增）
  - GB/T 16260.6—2023 第6部分：确定质量子模型用的度量（2023年新增）
- **对 Skill 的用途：** 作为软件质量检测模块的理论依据；为代码质量报告、测试报告提供质量特性评估框架。
- **是否必须纳入：** 可选
- **备注：** GB/T 16260.1 等效采用 ISO/IEC 25010:2011（Systems and software engineering — Systems and software Quality Requirements and Evaluation (SQuaRE) — Quality model）。

## 六、国际标准

---

### ISO/IEC/IEEE 12207:2017 系统和软件工程 软件生命周期过程

- **类型：** 国际标准（ISO/IEC/IEEE）
- **发布日期：** 2017年
- **实施日期：** 2017年
- **当前状态：** 现行
- **发布机构：** ISO（国际标准化组织）、IEC（国际电工委员会）、IEEE（电气电子工程师学会）联合发布
- **适用范围：** 规定了软件生命周期过程的框架，包括：协议协议过程组（获取、供应、开发、运营、维护、退役）、组织项目保障过程组（管理、资产管理、配置管理、质量保证）、工程过程组（需求导出、需求分析、设计、编码、集成、测试、认证、移交、支持、验收）、重用过程组等。广泛适用于各类软件系统的生命周期管理。
- **对 Skill 的用途：** 作为软件工程文档体系全景图的顶层框架；指导软件项目各阶段文档的完整覆盖；为文档版本管理提供生命周期视角。
- **是否必须纳入：** 可选（作为进阶参考）
- **备注：** 合并并替代了 IEEE/EIA 12207.0—1996 及其补充部分。ISO/IEC/IEEE 12207:2017 与 ISO/IEC/IEEE 15288:2015 构成系统与软件工程的姐妹标准（前者侧重软件，后者侧重系统）。

---

### ISO/IEC/IEEE 29148:2018 系统和软件工程 需求工程

- **类型：** 国际标准（ISO/IEC/IEEE）
- **发布日期：** 2018年
- **实施日期：** 2018年
- **当前状态：** 现行
- **发布机构：** ISO、IEC、IEEE 联合发布
- **适用范围：** 规定了需求工程的要求和指南，包括：需求获取、需求分析、需求规格说明、需求验证、需求管理、需求变更管理等全过程活动。给出了需求条目（requirement）编写质量标准（无歧义、完整、一致、可测试、可追溯等）。
- **对 Skill 的用途：** 作为需求文档编写的国际化参考标准；提供需求条目质量的评判标准（SOLID 原则的需求版）；补充 GB/T 9385—2008 的国际视野。
- **是否必须纳入：** 可选（作为进阶参考）
- **备注：** 替代 ISO/IEC/IEEE 29148:2011。合并了 IEEE 830—1998 和 ISO/IEC 15026（部分）的内容。

---

### ISO/IEC/IEEE 15288:2015 系统生命周期过程

- **类型：** 国际标准（ISO/IEC/IEEE）
- **发布日期：** 2015年
- **实施日期：** 2015年
- **当前状态：** 现行
- **发布机构：** ISO、IEC、IEEE 联合发布
- **适用范围：** 规定了系统生命周期过程的框架，包括：协议过程组（获取、供应）、组织项目保障过程组（管理、架构、配置、质量等）、技术管理过程组（规划、评估、控制）、技术过程组（需求、设计、集成、验证、确认、交付、运维、退役）。与 ISO/IEC/IEEE 12207 形成系统（15288）与软件（12207）的互补框架。
- **对 Skill 的用途：** 作为复杂系统（含嵌入式软件、软硬件一体化系统）文档编写的框架参考；为系统级文档（如系统规格说明、接口控制文档）提供编写依据。
- **是否必须纳入：** 可选（作为进阶参考）
- **备注：** 替代 IEEE Std 15288—2005。ISO/IEC/IEEE 15288:2023 版本正在推进中。
