# Agent 调研任务库

> 本模块定义 Agent 用于维护本 Skill 的周期性调研任务。当标准发生变更或新标准发布时，Agent 应主动更新对应章节。

---

## 一、周期性调研任务

### 任务 1：国家标准动态监控（月度）

```
任务名称：国家标准动态监控
触发频率：每月1次（定时任务）
任务描述：检查以下标准是否有新版本发布或状态变更
检查来源：
- 全国标准信息公共服务平台 https://std.samr.gov.cn
- 国家标准全文公开系统 https://openstd.samr.gov.cn

检查标准清单：
1. GB/T 7713.1—2025（学位论文）→ 检查是否被代替
2. GB/T 7713.2—2022（学术论文）→ 检查是否被代替
3. GB/T 7714—2015（参考文献）→ 检查是否被代替
4. GB/T 8567—2006（软件文档）→ 检查是否被代替
5. GB/T 9385—2008（需求规格）→ 检查是否被代替
6. GB/T 1.1—2020（标准编写）→ 检查是否被代替

输出：
- 若有变更：更新 01_国家标准索引.md 并在变更记录中说明
- 若无变更：仅记录"本月已检查，无变更"
```

### 任务 2：国际标准版本检查（季度）

```
任务名称：ISO/IEC/IEEE 国际标准版本检查
触发频率：每季度1次
任务描述：检查以下国际标准是否有新版发布

检查标准：
1. ISO/IEC/IEEE 12207（软件生命周期）
2. ISO/IEC/IEEE 29148（需求工程）
3. ISO/IEC/IEEE 15288（系统生命周期）

输出：
- 更新 01_国家标准索引.md 中的国际标准部分
- 若有重大变更，触发 Skill 整体审查
```

### 任务 3：模板库适用性审查（季度）

```
任务名称：模板库适用性审查
触发频率：每季度1次
任务描述：
- 检查现有模板是否有用户反馈的问题
- 检查是否有新场景需要新增模板
- 检查现有模板是否与最新标准一致

输出：
- 新增/修改 templates/ 目录下的模板
- 更新 templates/README.md
```

### 任务 4：常见问题库更新（半年度）

```
任务名称：常见问题库更新
触发频率：每半年1次
任务描述：
- 收集本 Skill 使用过程中的典型问题
- 更新 09_文档质量审查清单.md 中的"常见问题库"
- 更新各模块中的常见错误清单

输出：
- 更新对应文档
```

## 二、一次性调研任务（触发式）

### 任务 5：新标准发布响应

```
触发条件：发现新的相关国家标准/国际标准发布
任务描述：
1. 立即评估该标准是否与本 Skill 相关
2. 若相关：在 01_国家标准索引.md 中新增记录
3. 若高度相关：评估是否需要修订对应模块

标准发现渠道：
- 全国标准信息公共服务平台
- IEEE 标准协会官网
- ISO 官网
```

### 任务 6：用户反馈处理

```
触发条件：收到用户关于本 Skill 的具体反馈
任务描述：
1. 分析反馈类型（错误/缺失/建议）
2. 定位需要修改的模块
3. 执行修改并测试
4. 记录变更
```

## 三、Agent 调研执行 Prompt

当 Agent 触发上述调研任务时，使用以下 Prompt：

```
你是本 Skill 的维护 Agent。现在执行以下调研任务：

任务名称：[任务名称]
任务描述：[具体描述]

执行步骤：
1. [具体调研步骤1]
2. [具体调研步骤2]
3. [具体调研步骤3]

输出要求：
- 若发现变更：更新对应的 .md 文件，并说明变更内容
- 若无变更：创建/更新 [任务名]_log.md 记录本次检查结果
- 使用 write 工具写入文件

注意事项：
- 不得编造标准信息
- 无法确认的信息标记为"待核验"
- 变更记录必须包含日期、变更内容、依据
```

## 四、调研日志格式

每次执行调研任务后，在对应目录创建日志：

```markdown
# 调研日志：[任务名称]

- **执行日期**：[YYYY-MM-DD]
- **执行 Agent**：[Agent ID]
- **触发方式**：[定时/手动]
- **调研结果**：[有无变更]
- **变更内容**（如有）：
  - [变更1]
  - [变更2]
- **依据来源**：
  - [来源1 URL]
  - [来源2 URL]
- **下次检查日期**：[YYYY-MM-DD]
```

日志文件命名：`{年份}_{月份}_{任务名}_log.md`
