# 文档命名与版本控制

> 本模块定义文档命名规范、版本控制策略和变更记录格式。

---

## 一、文档命名规范（完整版）

### 1.1 通用命名规则

```
{类型}_{项目/课程}_{版本}_{日期}.{扩展名}

核心规则：
1. 单词之间用下划线 "_" 分隔
2. 不得包含空格
3. 不得包含特殊字符（仅允许 _ - .）
4. 版本号格式：v{major}.{minor}.{patch}
5. 日期格式：YYYYMMDD（8位）
6. 扩展名使用通用格式（.md / .docx / .pdf）
```

### 1.2 各类型文档命名对照表

| 文档类型 | 缩写 | 示例 |
|---|---|---|
| 学位论文 | Thesis | `Thesis_张三_智能推荐_v1.0_20240501.pdf` |

## 二、语义化版本（Semantic Versioning）

### 2.1 版本号格式

```
v{MAJOR}.{MINOR}.{PATCH}

示例：v1.2.3
  → MAJOR=1：做了不兼容的重大变更
  → MINOR=2：新增了功能（向后兼容）
  → PATCH=3：修复了缺陷（向后兼容）
```

### 2.2 版本变更规则

| 变更类型 | 触发条件 | 版本变更 | 示例 |
|---|---|---|---|
| **主版本变更** | 结构调整、删除章节、修改标准编号 | major+1, minor=0, patch=0 | v1.2.3 → v2.0.0 |
| **次版本变更** | 新增章节、补充功能 | minor+1, patch=0 | v1.2.3 → v1.3.0 |
| **修订版本** | 修正错误、补充说明、完善细节 | patch+1 | v1.2.3 → v1.2.4 |
| **草稿版本** | 编写过程中 | 0.1.0, 0.2.0... | draft |

### 2.3 版本状态

| 状态 | 说明 | 使用场景 |
|---|---|---|
| `draft` | 草稿，不对外 | 编写过程中 |
| `review` | 审查中 | 提交审查后 |
| `released` | 正式发布 | 审查通过后 |
| `deprecated` | 已废弃 | 被新版本替代后 |

## 三、变更记录（CHANGELOG）规范

### 3.1 变更记录位置

- **短期文档（≤10页）**：变更记录放在文档末尾
- **长期文档（>10页）**：单独的 `CHANGELOG.md` 文件
- **重要文档（项目核心文档）**：单独的 `CHANGELOG.md` + Git 提交记录

### 3.2 变更记录格式

```markdown
## 变更记录

### v1.2.0 (2024-04-20) - 次版本更新
**新增：**
- 添加第4章系统安全设计（响应需求变更申请 CRA-003）
- 增加数据库索引设计（基于性能测试结果）

**修改：**
- 修正第3章接口定义，补充错误码说明（修复缺陷 DEF-012）
- 更新第2章技术选型说明，将 Vue 替换为 React（响应评审意见 RVW-005）

**删除：**
- 删除第5章中关于 XX 模块的描述（因需求变更，模块取消）

**作者：** 张三
**审核：** 李四
**变更申请：** CRA-003, CRA-005
**缺陷单：** DEF-012
**评审意见：** RVW-005

---

### v1.1.0 (2024-04-10) - 次版本更新
**新增：**
- 添加第3章接口设计
- 添加第3.4节错误处理设计

**作者：** 张三

---

### v1.0.0 (2024-04-01) - 正式发布
**首次正式发布。**

**作者：** 张三
**审核：** 李四、王五
**评审日期：** 2024-03-28
**评审结论：** 通过（意见见附件）

---

### v0.2.0 (2024-03-20) - 内部审查版
**新增：**
- 完成第1-3章初稿

**作者：** 张三

---

### v0.1.0 (2024-03-15) - 初稿
**创建文档初稿。**

**作者：** 张三
```

### 3.3 变更类型关键词

| 类型 | 关键词 | 说明 |
|---|---|---|
| 新增功能/章节 | **新增** | 新增内容 |
| 修改内容 | **修改** | 修改已有内容 |
| 删除内容 | **删除** | 删除已有内容 |
| 缺陷修复 | **修复** | 修正错误 |
| 格式调整 | **格式** | 不涉及内容的格式调整 |
| 重构 | **重构** | 内容重新组织 |
| 澄清 | **澄清** | 使描述更清晰，不改变含义 |

## 四、Git 提交规范

### 4.1 提交信息格式

```
{类型}({范围}): {简短描述}

[可选的正文说明]

[可选的脚注，包含变更依据]
```

### 4.2 提交类型

| 类型 | 说明 | 示例 |
|---|---|---|
| `feat` | 新功能 | `feat(srs): 添加用户注册功能需求` |
| `fix` | 缺陷修复 | `fix(srs): 修正接口定义错误码` |
| `docs` | 文档更新 | `docs: 更新第3章图表规范` |
| `style` | 格式调整（不影响功能 | `style: 统一代码块格式` |
| `refactor` | 重构 | `refactor(hld): 重新组织架构图` |
| `test` | 测试相关 | `test(tp): 补充边界测试用例` |
| `chore` | 杂项 | `chore: 更新版本号` |

### 4.3 提交示例

```
feat(srs): 添加购物车功能需求规格

- 新增购物车模块的功能需求
- 新增结算流程的非功能需求（性能要求：响应时间<500ms）
- 新增相关接口需求（与支付模块的集成）

变更依据：需求变更申请 CRA-007
关联文档：#12
```

## 五、目录结构规范

### 5.1 项目文档目录结构

```
{项目名}/
├── 01_立项/
│   ├── FA_{项目名}_v1.0_{日期}.pdf
│   └── CHANGELOG_FA.md

├── HLD_{项目名}_v1.0_YYYYMMDD_归档.zip
└── ...
```

### 命名速查

| 场景 | 格式 | 示例 |
|---|---|---|

### 版本速查

| 场景 | 版本 | 说明 |
|---|---|---|
| 初稿 | v0.1.0 | 开始编写 |
| 完成初稿 | v0.x.0 | 达到基本完整 |
| 提交审查 | v0.x.0 → review | 审查中 |
| 审查通过 | v1.0.0 | 正式发布 |
| 小修复 | v1.0.1 | patch |
| 新增功能 | v1.1.0 | minor |
| 大改版 | v2.0.0 | major |
