# 02_国家标准驱动的文档管理与工程化写作规范

> 为个人项目、课程作业、毕业设计、软件工程项目、论文报告、技术文档、产品文档建立一套**可引用、可追溯、可审查、可复用、可交付**的文档生产标准。

---

## 1. Skill 定位

本 Skill 的目标不是简单规定"文档怎么写"，而是建立一套**标准驱动的文档生产体系**：

- ✅ 可溯源：每项结论都有引用来源
- ✅ 可审查：完整的质量评分模型 + 检查清单
- ✅ 可复用：统一的命名规范 + 元数据体系
- ✅ 可维护：版本控制 + 变更记录
- ✅ 可交付：符合国家/行业标准的正式文档
- ✅ 可归档：清晰的归档路径和保留期限

## 4. 核心原则

1. **标准优先**：先确定适用的国家/行业标准，再按标准框架撰写
2. **读者优先**：明确文档的目标读者，写他们能理解的深度
3. **证据可追溯**：每个重要结论必须附有引用来源
4. **模板化生产**：优先使用本 Skill 提供的模板
5. **检查清单验收**：文档完成后必须过检查清单
6. **版本可维护**：每份正式文档必须有版本号和变更记录
7. **链接优先**：引用已有文档用链接，而非复制内容
8. **归档优先**：正式文档发布后立即归档，不散落在本地

## 5. 标准体系

| 层级 | 内容 | 典型标准 |
|---|---|---|
| 国家标准 | 学位论文、参考文献、软件文档、需求规格、标点符号、量和单位 | GB/T 7713、7714、8567、9385 |
| 行业标准 | 软件工程、信息系统、测试、运维、安全 | 各行业规范 |
| 国际标准 | ISO/IEC/IEEE 软件生命周期、需求工程、系统工程 | ISO/IEC/IEEE 12207、29148 |
| 高校规范 | 本科毕业设计、课程论文、实验报告规范 | 各高校研究生院/教务处模板 |
| 企业实践 | PRD、技术方案、API 文档、项目管理文档 | 内部 SOP |

## 6. 使用流程

```
① 确定文档类型
    ↓
② 匹配标准（查看 01_国家标准索引）
    ↓
③ 选择模板（templates/ 目录）
    ↓
④ 按模板撰写（参照对应规范文档）
    ↓
⑤ 自查检查清单（checklists/ 目录）
    ↓
⑥ 提交审查
    ↓
⑦ 发布 + 归档
    ↓
⑧ 后续复用
```
