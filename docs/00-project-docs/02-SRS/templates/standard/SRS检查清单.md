# 软件需求规格说明书（SRS）检查清单

> 本清单用于 SRS 文档的审查。依据 GB/T 9385—2008《计算机软件需求规格说明规范》。

---

## A. 引言部分

- [ ] 1.1 目的：说明编写目的、预期读者、适用范围
- [ ] 1.2 范围：说明软件名称、功能范围、与外部系统的边界
- [ ] 1.3 定义与缩略语：列出文档中使用的专有术语
- [ ] 1.4 参考资料：列出引用的标准、文档、文献

---

## B. 总体描述

- [ ] 2.1 产品前景：描述产品背景和价值
- [ ] 2.2 用户特征：描述目标用户群体的特征、职责
- [ ] 2.3 约束条件：列出设计限制（硬件、软件、运行等）
- [ ] 2.4 假设与依赖：列出依赖的外部系统/组件

---

## C. 需求规格（核心部分）

### C.1 功能需求

- [ ] 每条功能需求有唯一标识符（如 FR-001）
- [ ] 功能需求描述了系统行为（而非实现方式）
- [ ] 功能需求可测试
- [ ] 功能需求之间无冲突
- [ ] 功能需求覆盖了所有用户角色和用例

### C.2 非功能需求

- [ ] **性能需求**：有量化指标（如响应时间≤2s，并发≥100）
- [ ] **安全性需求**：有认证、授权、数据保护要求
- [ ] **可靠性需求**：有可用性、容错要求（如 99.9% 可用）
- [ ] **可维护性需求**：有可扩展、可替换要求
- [ ] **易用性需求**：有人机交互、用户体验要求
- [ ] **数据需求**：有数据量、数据保留、数据备份要求

---

## D. 接口需求

### D.1 用户接口
- [ ] 有界面原型链接或描述
- [ ] 有输入/输出格式说明
- [ ] 有错误消息规范

### D.2 硬件接口
- [ ] 有设备类型和通信协议说明

### D.3 软件接口
- [ ] 有外部系统/模块接口说明
- [ ] 有接口调用方式（API/REST/gRPC 等）
- [ ] 有输入参数、输出格式、错误码说明

### D.4 通信接口
- [ ] 有网络协议说明（HTTP/WebSocket 等）
- [ ] 有安全通信要求（TLS/HTTPS 等）

---

## E. 可验证性

- [ ] 每条需求可对应到测试用例
- [ ] 性能需求有明确的量化指标
- [ ] 安全性需求有可验证的方法
- [ ] 可建立需求追踪矩阵（需求→设计→测试用例）

---

## F. 可追踪性

- [ ] 每个功能需求有来源（用例/用户故事/变更申请）
- [ ] 需求变更有记录
- [ ] 可建立前后版本的对比

---

## G. 标准符合性

- [ ] 文档结构符合 GB/T 9385—2008 要求
- [ ] 无实现细节混入需求规格（实现属于设计阶段）
- [ ] 需求描述无歧义（避免"大约""可能"等模糊词）
- [ ] 需求描述完整（覆盖正常、异常、边界情况）

---

## H. 完整性自检

- [ ] 所有"待定"项已解决或明确了责任人
- [ ] 无未完成的占位符
- [ ] 术语使用一致
- [ ] 文档内部交叉引用正确
