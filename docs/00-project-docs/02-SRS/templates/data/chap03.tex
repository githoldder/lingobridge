\chapter{需求规格}

\section{功能需求}
本平台的功能需求定义围绕“教纲课件辅助消费与发音录制”这一业务主线展开，功能范围严格限定在 MVP 开发边界之内。核心模块包含：身份认证、课程管理、课件管理、作业管理、中文 TTS 朗读辅助、录音模块、直播课堂、管理后台以及状态反馈模块。

\subsection{身份认证模块 [FR-001]}
本模块负责系统内各角色的身份验证与鉴权。系统区分教师、学生与管理员三种角色，采用基于令牌（Token）的无状态会话保持机制。系统需要根据用户的角色身份，在登录成功后进行动态角色路由，分发至对应的控制台视图。
\paragraph{用户故事}
\begin{itemize}
  \item \textbf{Given}：具有俄语背景的留学生阿合买提已在系统注册，其用户角色为学生。
  \item \textbf{When}：阿合买提在登录界面输入正确的用户名与密码，点击“登录”按钮；
  \item \textbf{Then}：系统在 1.5 秒内完成令牌验证，并将页面重定向至学生端的课程列表视图，界面文字默认加载为其首选的俄语。
\end{itemize}

\subsection{直播课堂模块 [FR-007]}
本模块主要提供在线上授课期间的师生页面同步以及受控音视频连麦机制。教师可发起翻页信令，学生的课件页面根据 Presence 状态机进行跟随；学生可进行举手连麦申请，获得教师授权后方可开启本地硬件设备。
\paragraph{用户故事}
\begin{itemize}
  \item \textbf{Given}：王老师与阿合买提已同时进入了在线课堂，王老师正在进行课件讲解。
  \item \textbf{When}：王老师在屏幕上将课件翻到了第 2 页；
  \item \textbf{Then}：阿合买提的学生端界面在 2 秒内接收到 Presence 状态变更信令，自动同步跳转并渲染第 2 页课件图片。
  \item \textbf{When}：阿合买提在学生端点击“申请举手发言”，王老师在控制面板点击“同意连麦授权”；
  \item \textbf{Then}：阿合买提的浏览器麦克风被激活，系统将阿合买提的本地音频流同步广播至在线课堂中的其他成员。
\end{itemize}

\section{非功能需求}
本系统的非功能性需求从性能、安全、可靠性、易用性以及法律数据合规等维度为系统设定了客观、可测量的验收指标。

\subsection{性能需求}
系统的性能要求致力于确保在跨国弱网网络环境下的用户流畅度。
\begin{itemize}
  \item \textbf{页面与数据加载延迟}：在香港云节点部署、留学生本地网络访问的典型测试条件下，静态页面的首屏加载耗时应优化至 500 毫秒以内；TTS 标准发音播放响应时延应控制在 1.0 秒以内；
  \item \textbf{课件转换排队耗时}：教师端提交 10 页以内的 PPTX/PDF 课件，后台自动转为高保真 PDF 静态图的转换耗时应限制在 15 秒以内；
  \item \textbf{状态同步响应时延}：直播课堂（Presence）的翻页跟随信令跨海分发延迟应控制在 2.0 秒以内；
  \item \textbf{音频文件体积约束}：单次 15 秒内的拼音跟读录制，其 WebM/Opus 封装的音频文件大小应控制在 1MB 以下，以减轻跨境网络通信带宽负荷。
\end{itemize}

\subsection{安全性需求}
系统采取多维度的安全防御机制：
\begin{itemize}
  \item \textbf{鉴权安全}：所有的 API 请求在写入和读取数据库前，必须通过身份验证令牌校验；
  \item \textbf{输入安全校验}：教师上传的课件标题、关联文案文本和输入框信息，必须在后台执行 XSS 清洗，严禁包含任何恶意脚本；
  \item \textbf{设备硬件物理停用}：在关闭麦克风、摄像头或退出直播课堂视图时，系统前端必须显式执行 \texttt{track.stop()} 切断硬件物理供电，防止发生恶意窃听和硬件占用悬挂的隐患。
\end{itemize}

\subsection{可靠性需求}
\begin{itemize}
  \item \textbf{自适应指数退避重连}：在遭遇跨境海缆网络抖动或长连接暂时阻断时，客户端信令层必须启动自适应指数退避重试，自动在后台重新建立信令连接，规避控制台报错以保护用户体验；
  \item \textbf{演示数据预置备份}：系统默认预置“拼音三课”的完整课程和学生练习的演示数据，确保系统部署后开箱即用，降低对外部实时交互环境的强依赖。
\end{itemize}

\subsection{用户界面与易用性需求}
\begin{itemize}
  \item \textbf{多语热切换适配}：系统界面必须支持三语（中文简体、俄语与哈萨克语）一键热切换。所有文本标识、导航按钮和提示信息在切换后须实时转换，不得残留非对称漏翻文本；
  \item \textbf{错误指引清晰度}：针对录音被拒、上传超时等异常，系统必须给出直观的弹框式操作步骤指引，而非暴露底层代码级报错（如 \texttt{PermissionDeniedError}）。
\end{itemize}

\subsection{数据与合规需求}
由于本产品面向跨国留学生，数据安全需严格遵循中哈两国的基础数据保护法：
\begin{itemize}
  \item \textbf{学生个人数据自主权}：系统承诺不主动收集任何学生的敏感生物识别信息，录音数据仅用于教师批改作业。学生录音文件可在个人控制台面板一键进行“彻底删除”，实现数据库与文件系统的物理擦除；
  \item \textbf{数据保留策略限制}：对于已结课班级的历史语音数据，系统设定自动归档与过期删除清理逻辑，有效降低服务器磁盘开销，防范历史数据泄露。
\end{itemize}

\section{接口需求}
LingoBridge 为保证可插拔特性，对系统与外部硬件及环境的通信交互界面进行了标准化定义：
\begin{itemize}
  \item \textbf{用户接口（User Interface）}：学生端前端采用响应式流式布局，兼容支持主流桌面 PC 浏览器及移动端平板；界面组件具备清晰的状态转换动效。
  \item \textbf{硬件设备权限接口（Hardware Interfaces）}：平台与客户端设备的交互通过 Web Audio API 接口以及 MediaRecorder 音频录制设备接口调用硬件麦克风权限。
  \item \textbf{软件外部通信接口（Software Interfaces）}：系统后端 API 采用标准的 RESTful 风格，通信协议完全基于安全套接字超文本传输协议（HTTPS）与安全套接字协议（WSS），接口契约完全由 JSON 格式进行数据序列化。
\end{itemize}
