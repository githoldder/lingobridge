\chapter{总体描述}

\section{产品背景}
随着国际中文教育在中亚及俄语区（如哈萨克斯坦等国）的高速发展，传统教学模式面临数字化转型的迫切要求。在传统的俄语背景中文教学中，经常面临着课堂教学课时少（通常每学期为 196 课时，且线下集中授课频率较低，多为每周仅一次课）、课后时间间隔长、学生缺乏高频中文拼音及韵脚校准辅助工具的痛点。现有的通用在线工具（如普通的多媒体播放器或通用网课平台）往往无法针对中文拼音的声母、韵脚与声调提供针对性的发音采集与对比功能，导致学生在课后自主复习阶段的发音盲区无法得到及时留存与反馈，严重制约了国际汉语教学的质量提升。

本产品 LingoBridge 正是在此背景下应运而生。平台依托 C2K 国际教育机构的跨境沟通渠道以及试点高校（如赤塔国立大学 CZU）的丰富学生资源，旨在为具有俄语背景的中文初学者及高校留学生提供一款高互动的课后拼音跟读、听读校准与学习轨迹沉淀工具。通过建立“课件内容消费---发音本地采集---录音持久化沉淀”的数字化闭环，LingoBridge 充当了连接课堂教学与课后自学的数字化纽带。产品研发历经初期的“全量大而全教育平台”设想，经过深入的需求访谈与可行性论证，最终收敛至当前的最小可行产品（MVP）架构，以确保系统在跨境网络环境下的高稳定性与经济可行性。

\section{产品前景}
LingoBridge 的产品前景定位在“教纲课件辅助消费与发音录制工具”，其并非一款大而全的学校教务系统（LMS）或音视频直播室，而是与现有的重型会议工具（如腾讯会议、Microsoft Teams 等）形成差异化对策与功能互补。在实际教学场景中，教师通过腾讯会议或 Teams 进行线上网课授课，而 LingoBridge 则作为课前预习、课件交互式展示、课后跟读练习分发以及音频作业归档的专用数字化资产沉淀平台。

通过这种“专业直播 + 专用辅助”的混合模式，系统避免了堆砌重型实时字幕与音视频编解码的研发开销，而是专注于优化跨国弱网环境下的课件图片分发加载性能、流式标准 TTS 发音合成拉取以及高可信度的 MediaRecorder 录音持久化。未来，随着版本的迭代，系统将逐步在云端部署轻量翻译大模型，并在后续版本中集成对接腾讯会议与 Teams 的会议预约及出勤拉取接口，形成覆盖“线上授课---课后自学---智能评估---教务跟踪”的完整数字化中文教育前景版图。

\section{运行环境}
为了确保留学生能零门槛即入即学，本系统的客户端采用 Web 响应式架构，运行环境规格定义如下：
\begin{itemize}
  \item \textbf{学生端浏览器兼容性}：支持主流移动端与 PC 端浏览器，如 Google Chrome（85 及以上版本）、Safari（14 及以上版本）、Firefox、Edge。要求浏览器完整支持 Web Audio API 与 MediaRecorder 接口以进行音频采集，且必须支持 \texttt{LanguageContext} 三语热切换及 Canvas 2D 高保真课件渲染。
  \item \textbf{教师端浏览器兼容性}：推荐使用 Google Chrome 桌面版或 Edge 桌面版，以支撑大文件上传以及屏幕捕获接口的使用。
  \item \textbf{硬件设备依赖}：客户端设备必须配备完好的麦克风（学生端）和摄像头（教师端录屏），且用户必须在首次进入课堂时显式授权媒体权限。
\end{itemize}

\section{假设与依赖}
本系统的正常运行与成功演示，建立在以下合理的假设与依赖条件之上，相关风险评估如下：
\begin{itemize}
  \item \textbf{课件数据源假设}：假设教师上传的课件格式符合规范，且已在课件中清晰标明拼音、声母与韵脚文本以供 TTS 识别。若上传课件的图片分辨率或排版出现严重偏差，可能导致前端 Canvas 渲染模糊，该风险将通过提供标准的 PDF/PPTX 上传模板及压缩工具来进行缓解。
  \item \textbf{浏览器媒体接口依赖}：系统深度依赖底层浏览器的 \texttt{MediaRecorder} 录音接口。若用户的非主流设备屏蔽了录音权限或不支持 WebM/Opus 音频封装，系统将自动提示降级为“仅听读不保存”的只读模式。
  \item \textbf{网络长连接 Presence 依赖}：假设师生之间的翻页同步依赖于持久网络信道。若跨境弱网导致长连接阻断，前端将自动启动自适应指数退避与重连重试机制，确保用户无卡顿退出，并在网络恢复后自动获取最新课件页面。
\end{itemize}

\section{需求分期}
本系统秉承“聚焦核心、快速验证、有条件可行”的敏捷思路，对需求进行了清晰的分期规划：
\begin{itemize}
  \item \textbf{V1.0 MVP 阶段（当前基线）}：单学校单租户支持。实现“PDF 课件上传与三页翻页展示---TTS 发音朗读播放---麦克风跟读录音本地试听、回放与删除---录音上传文件系统沉淀---班级积分排行”的核心拼音自学交互闭环。
  \item \textbf{V1.1 阶段}：支持多学校、多班级多租户基本隔离。上线教师一键预约及拉起腾讯会议，实现课堂出勤数据的基本拉取。
  \item \textbf{V2.0 阶段}：引入学校教务认证，在云端部署轻量翻译大模型，上线低延迟实时双语字幕与口语智能 ASR 音素评测打分。
  \item \textbf{V3.0 阶段}：上线国际付费收单接口，支持 B 端抽成与 C 端付费解耦，上线国际学生中文交流互助社区。
\end{itemize}
