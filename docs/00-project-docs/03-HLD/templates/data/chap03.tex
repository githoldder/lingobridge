\chapter{系统结构设计}

\section{系统功能模块划分}
为了贯彻软件工程中高内聚、低耦合的模块化设计思想，LingoBridge 平台在逻辑与物理结构上进行了严格的子系统解耦。系统主要划分为用户与鉴权子系统、班级与课程子系统、在线互动课堂子系统、作业与词汇子系统以及语音与合成子系统。各子系统之间职责边界分明，均通过预先定义的数据传输对象（DTO）和标准 RESTful API 路由进行安全通信，任何子系统内部的重构均不会影响其他兄弟模块的稳定运行。

\section{模块间调用关系}
系统在运行时，前端 UI 展示组件并不直接处理任何底层业务逻辑或物理存储。当用户在客户端发起操作（例如教师在交互课堂中执行 PDF 课件翻页、学生在作业面板点击确认提交录音等）时，前端组件首选调用 \texttt{apiClient.ts} 中封装 of 无状态客户端 API 方法。该方法会在 HTTP 请求头中自动附加当前用户的 Authorization JWT 鉴权令牌，并将数据流以标准的 JSON 或 multipart 数据格式推送到后端的 Express.js 控制路由。

后端路由层接收请求后，首选通过身份鉴权拦截器提取 JWT，校验用户角色是否符合该接口的权限白名单（如教师权限或管理员权限）。校验通过后，控制器将请求参数传递给专门的 Repository 仓库层（例如 \texttt{coursesRepo} 或 \texttt{assignmentsRepo}）。仓库层通过统一的数据传输对象实现双模（JSON 文件和 PostgreSQL 物理数据库连接池）底座缩放，在不影响上层控制器逻辑的前提下，完成元数据的持久化读写。最终，处理结果沿着原调用链层层返回，由前端页面接收 DTO 进行状态刷新并渲染给用户。

\section{子系统划分说明}
本系统在物理结构上清晰地被划分为前端单页面展示子系统与后端信令控制子系统。

前端子系统在浏览器环境中执行，主要基于 React 19 强类型组件体系构建，其内部通过 Context 状态机制管理用户鉴权（AuthContext）与国际化多语言（LanguageContext）上下文。前端子系统在渲染上推行轻量原则，如 PDF 课件的分页解析与 Canvas 绘制均全部下推至学生和教师端本地浏览器执行，从而杜绝了向服务器端发起渲染请求的必要性。

后端子系统则运行于 Node.js 服务端沙盒中，其内部划分为 Express 控制路由层、Repository 仓库持久化层、文件 I/O 缓冲区以及外部云引擎对接网关。这种严格的子系统隔离设计，不仅降低了服务器端高昂的计算和网络费用，也极大地提升了系统在遇到高并发、跨海长连接抖动等复杂网络情境下的弹性与稳定性能。
