\chapter{安全设计}

\section{用户认证与双令牌滑动窗口流}
LingoBridge 平台为确保凭证传输安全性并防止会话劫持，采用了基于 HMAC-SHA256 签名算法的无状态身份验证机制。

当用户通过账户密码验证后，系统后端在机密密钥保护下，为该用户签发一段不可篡改的 JSON Web Token（JWT）。为了防御针对 JWT 令牌的窃取与重放攻击，平台设计了双令牌（Access Token 与 Refresh Token）滑动窗口置换机制。

Access Token 设定为 1 小时的短生命周期，用于承载高频的 API 访问控制。Refresh Token 设定为 7 天的长生命周期，并加密存储于物理数据库或隔离的存储介质中。当前端检测到 Access Token 即将过期且用户正处于活动态时，API 拦截器会自动调度 Refresh Token 向安全网关申请置换新令牌，实现会话平滑滑动，避免了用户频繁被迫退出登录，并在数据层提供了实时的凭证吊销支持。

针对前端在 \texttt{localStorage} 中缓存 JWT 带来的潜在跨站脚本（XSS）注入窃取风险，平台推行了严格的防御策略。系统在前端脚手架与 API 客户端中内置了统一的输入字符安全过滤与 XSS 转义机制，防止任何恶意脚本通过留言、姓名修改或作业文本注入前台运行。同时，在长远演进路线中，规划采用将 JWT 写入带 \texttt{HttpOnly} 与 \texttt{Secure} 属性的 Cookie 候选安全防御架构，彻底从浏览器物理层面上隔离脚本对敏感 Token 数据的读取，保障了留学生凭证的机密性。

\section{权限控制}
系统底层深刻践行了基于角色的访问控制（RBAC）标准，将所有系统用户划分为学生（\texttt{student}）、教师（\texttt{teacher}）以及系统管理员（\texttt{admin}）三大身份角色。

在前端子系统层面，所有的路由切换与工作台导航均受到鉴权守卫拦截器的监管。如果学生角色尝试在浏览器地址栏直接输入管理员看板或教师排课详情的路径，拦截器会自动识别当前安全上下文中存储的 \texttt{UserDto.role} 属性，并在界面上弹出访问受限的语种对应提示，同时强制将路由导流回系统首页。

在后端接口控制层，系统则通过 Express 级联中间件建立起安全防火墙。例如，像课件上传接口、Excel 作业包导入接口以及班级名册更新接口等敏感通道，在其路由声明中均前置了具有角色判定功能的 \texttt{requireTeacher} 过滤器；而教师课程详情的学生作业进度检查接口，则必须通过 \texttt{assertHomeworkAccess} 安全辅助中间件来动态核验所查询的 \texttt{studentId} 是否确实属于该教师所执教的试点班级，杜绝了横向越权和纵向越权的可能性。

\section{数据安全}
本系统在持久化存储、动态传输和灾备管理三个维度，全方位构建了数据安全屏障。

在动态数据流向维度，系统所有的动态 HTTPS 接口路由和 WebSocket 实时 Presence 通信均绑定标准的安全传输协议（SSL/TLS 1.3），所有的语音字节流、用户密码及课件资源在公共网络中均以高强度的密文传输，有效抵御了旁路监听和中间人重放攻击。

在持久层维度，系统通过在 \texttt{class\_members} 及 \texttt{courses} 表中设计外键级联约束与唯一性索引，从逻辑层面达成了多租户试点班级的物理隔离。

为了应对留学生毕业或退出试点的隐私保护诉求，系统数据管理模块中内置了物理擦除通道，当管理员执行销户或级联物理删除命令时，系统会在多级关联表中执行级联擦除，防止敏感行为日志在云端残留。

\section{操作审计与隐私保护}
作为一款需要调用留学生麦克风等声学传感硬件的教学辅助平台，LingoBridge 将硬件调用安全和学生个人隐私审计作为核心设计底线，制定了麦克风硬件物理注销规范。

当留学生在前台激活口语纠音任务或连麦互动时，系统前端组件会调用浏览器的 \texttt{MediaDevices.getUserMedia()} 接口，启动本地麦克风物理硬件，并创建持续接收声学音频流的 \texttt{MediaStream} 轨道。为了杜绝潜在的物理隐私泄露风险（即防止网页跳转、网络异常断连或组件卸载后麦克风硬件仍处于采音状态，导致物理指示灯常亮），系统前端在所有相关的交互组件（如口语跟读卡片及虚拟教室主视图等）的生命周期销毁回调函数中，强行塞入了硬件轨道物理清扫闭环。

当发生学生主动点击离开教室、前台页面重定向、网络断连被踢出或组件发生垃圾回收卸载时，清扫机制会自动在底层拦截该事件，迅速遍历并抓取当前活动中所有的 \texttt{MediaStream} 物理轨道，并逐一、显式地派发底层的物理停用指令 \texttt{track.stop()}。此操作审计规范在物理硬件层面上彻底阻断了潜在的远程监听与隐私外泄通道，保障了本平台在留学生隐私保护层面完全符合技术安全标准。
