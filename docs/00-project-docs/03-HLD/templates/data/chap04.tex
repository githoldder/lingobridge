\chapter{接口设计}

\section{用户接口设计}
LingoBridge 平台的用户接口采用极简主义的设计美学，全面贯彻响应式自适应布局标准，以保证在不同终端设备上的视觉呈现一致性。前端交互开发主要依托标准的 Web 技术栈，如通过 HTML5 Canvas 2D 绘图引擎承载教师端的基础白板划线指示，并下推 PDF.js 分页库在浏览器沙盒内直接将 PDF 课件逐页高精准渲染为像素级一致的画布。系统在左侧设计了常驻式侧边导航栏，右侧则为核心功能工作区，在视觉上提供清晰的层级结构。

\section{外部接口设计}
在外部服务对接维度，平台遵循轻量运营与极低网络开销的核心诉求，设计了标准的外部 API 调度网关，并对未验证能力执行候选及演进降级描述。

针对学生口语听读跟读的纠音诉求，系统后端在接到录音解析请求后，设计了流式调用自动语音识别（ASR）评测接口的扩展占位符。在目前的 MVP 原型阶段，系统主要提供本地音频采集与回放，ASR 评测打分及音素级分析功能定位为\textbf{后续演进路线}。

针对中文标准拼音的听读合成诉求，系统调度第三方语音合成（TTS）流式 API 接口，返回高压缩率的标准音频字节流，并配合后端的双层缓存策略进行就地复用。

为了实现直播课堂中高质量的音视频通话闭环，系统在设计上规划了第三方会议接口。由于第三方会议平台存在严格的企业资质审核门槛及长周期的企业级准入限制，在 MVP 试点阶段，系统不直接集成腾讯会议及 Microsoft Teams 的官方 OAuth 2.0 鉴权通道。系统将官方 REST API 自动同步考勤和自动建会功能降级为\textbf{候选增强技术}，并通过设计非 API 依赖的外链分享降级预案来保证基本教学闭环的跑通。

\section{数据接口设计}
系统底层的数据库访问接口均由统一的 Repository 数据源适配仓库承载。数据访问层不仅在接口方法签名上对上层控制器隐藏了物理介质的差异，更在底层封装了双模式读写通道。当系统启动并检测到物理环境变量 \texttt{DB\_MODE=postgres} 时，数据访问层内部的数据流自动转换为向连接池派发包含参数占位符的 SQL 结构化语句，并严格遵循 DTO 格式进行结果返回；而在本地开发或演示模式下，仓库层数据流则切换至 JSON 读写器，直接对物理文件系统中的 JSON 元数据存储执行并发安全锁保护的排他文件 I/O 读写，在逻辑与物理上保障了数据传输对象的行为一致。
