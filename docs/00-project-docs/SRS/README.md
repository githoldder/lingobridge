## 1. 引言

### 1.1 目的
说明 LingoBridge MVP 阶段的业务边界和功能需求，指导后续的开发与测试防回归。

### 1.2 范围
本项目涵盖核心的教学互动闭环，包含：
- **包含**：多角色鉴权、课程与Live课时管理、课件（PDF）及作业（Excel）导入、课内画笔标注、跟读录音与学习记录。
- **不包含**：复杂的在线多方音视频通讯（MVP可考虑集成外部工具）、在线PPT实时编辑、复杂的多租户/支付计费系统。

### 1.3 用户特征

| 用户类型 | 描述 | 核心诉求 |
|---|---|---|
| 游客 (Guest) | 未登录用户 | 浏览课程大纲，体验 Demo 环节。点击受保护操作必须触发注册/登录弹窗。 |
| 学生 (Student) | 学习者 | 加入直播间、观看课件、跟读录音、提交课后作业，要求跨平台（主要移动/平板 Web）友好。 |
| 老师 (Teacher) | 教学者 | 创建班级/课程、上传 PDF/PPTX 课件与 Excel 题库、开启直播进行画笔标注并下发练习。 |
| 管理员 (Admin) | 平台运营方 | 直接进入独立的后台 (`/admin`)，管理老师与学生账号，监控课程资源（多媒体、JSON数据存储状态）。 |

## 2. 核心业务规则 (Business Rules)

1. **R1 身份准确**：页面不得硬编码 mock 身份，学生名必须来自 `/users/me`；游客显示游客态并拦截操作。
2. **R2 课时模型**：一个 Live Session = 一个课时节点 (`lessonNodeId`) = 一个作业节点 (`assignmentNodeId`)。
3. **R3 数据持久化**：学习记录必须持久化，跳页刷新后作业完成状态不得回退。
4. **R4 权限分离**：学生端 Live 不显示教师专用的配置按钮（例如“映射作业”、“清除全班笔迹”）。
5. **R5 UI与画笔**：关闭画笔模式不应清空笔迹，仅“清除”按钮才可清空。
6. **R6 管理端隔离**：Admin 登录后必须直接进入后台，后台不是学生 dashboard，用于底层数据和接口维护。

## 3. 功能需求规格

### 3.1 身份与权限模块
**FR-001: 游客门禁拦截 (GuestGate)**
- **处理：** 拦截游客提交表单、进入Live、发起录音等动作，触发 Login/Register 对话框。
**FR-002: 多角色分流登录**
- **处理：** 登录成功后，根据 role（admin/teacher/student）跳转至对应的 dashboard 或 Admin 面板。

### 3.2 教师端（教学管理与授课）
**FR-003: 课件与题库导入**
- **处理：** 提供 PDF 上传供板书展示，提供 Excel 上传解析以生成标准作业题库结构。
**FR-004: Live 课堂与画笔标注**
- **处理：** 以明确线条（如圆珠笔效果）实现标注，支持粗细调节，多端状态同步或本地展示。

### 3.3 学生端（上课与练习）
**FR-005: 课堂沉浸**
- **处理：** 学生不可上传课件，无教师授权时默认不可开麦/清除画板，只能接受教师屏幕及作业入口推送。
**FR-006: 录音与记录**
- **处理：** 利用 Web Audio API 记录发音并附带 `taskId` 上传，后端在 JSON `db.json` 中保存 `studentId + lessonNodeId + taskId`。

## 4. 接口需求概要
主要 RESTful 接口路径定义：
- `GET /api/v1/users/me` (当前用户状态)
- `POST /api/v1/courses/:id/lesson-nodes` (生成包含作业的课时)
- `POST /api/v1/learning-records` (Upsert 学生端提交的练习记录)
- `POST /api/v1/assignments/import` (Excel作业入库)
