\chapter{操作可行性}

\section{多端响应式与国际化支持}
LingoBridge 面向不同国别与背景的师生，在交互流程设计上采取了适配性设计：
\begin{itemize}
  \item \textbf{多端布局适配}：前端界面在开发阶段针对常见屏幕尺寸（如桌面端课程管理面板、平板端在线课件容器、移动端日程卡片）进行了响应式排版适配测试，以适应不同终端的使用习惯。
  \item \textbf{多语言资源配置}：系统提供中、英、俄、哈四语的国际化切换接口，多语言文本资源集中配置于 \texttt{LanguageContext} 中，用于支撑跨国师生的沟通与基本界面导航。
\end{itemize}

\section{双角色交互流程与功能事实验证}
本平台在原型验证阶段，主要通过如下具体的角色交互流程来论证其操作可行性：
\begin{itemize}
  \item \textbf{教师端排课与课件管理}：教师可使用图形化排课日历进行课堂日程的新增与编辑，并通过排课校验（防止课时冲突）将日程持久化。同时，系统支持教师上传演示使用的三页课件 PDF，其后端服务通过绑定课时节点（\texttt{lessonNodeId}）提供统一的获取接口。
  \item \textbf{学生端登录与课堂同步流}：学生可从登录页进入在线课堂，课堂界面包含自连接状态指示动画。在进入课堂后，学生能够实时查看当前课件 PDF 页面，并接收由教师端触发的页面同步信令（Presence）。此外，学生可发送举手申请并获得教师的连麦授权。
  \item \textbf{跟读作业录制与提交}：在线跟读作业场景下，界面为学生提供麦克风录音控制面板，支持录音的开启、关闭与本地音频的临时回放，并能够将音频流打包上传至服务器。
  \item \textbf{后续智能化扩展方向}：智能语音口语评分系统（ASR 自动评分与发音盲区高亮）属于后续功能规划，将作为下一阶段接入外部智能 API 的扩展目标，用于减轻教师的课后批改负荷。
\end{itemize}
