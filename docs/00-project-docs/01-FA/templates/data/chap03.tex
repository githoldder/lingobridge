\chapter{操作可行性}

\section{多端响应式与三语热切换配置可行性}
LingoBridge 中文自学平台面向跨国背景、多屏幕终端的试点师生，在交互界面与易用性层面设计了严格的适配性对策，以确保在不同环境下均具备极高的操作可行性：
\begin{itemize}
  \item \textbf{多端流式布局适配}：前端 UI 界面基于标准 CSS Flexbox 和 Grid 进行响应式流式布局设计，完美适配教师端大屏 PC 课程控制台、学生端平板电脑（iPad）讲义翻页容器，以及学生端移动手机课后跟读面板。前端在 320px 到 1920px 的主流屏幕分辨率下均通过了流式排版兼容性测试，保证交互组件不发生重叠或超出视口。
  \item \textbf{LanguageContext 三语热切换}：系统界面设计了面向中文简体、俄语、哈萨克语三语的一键热切换机制。国际化多语言文本资源在前端由 \texttt{LanguageContext} 统一进行动态加载与派发。在切换语言时，系统能在不重新加载页面的情况下实时刷新所有静态文本、按钮导航、Toast 提示及错误指引，绝不产生跨语言文本残留或溢出破坏排版的低级缺陷。
\end{itemize}

\section{双角色全链路闭环操作事证验证}
为了验证多端响应式下的具体操作逻辑，项目组针对核心业务流在原型开发阶段执行了全链路操作路径事实验证：
\begin{itemize}
  \item \textbf{教师端：教纲与课件的零门槛分发}：教师进入 PC 浏览器控制台，通过标准的 PDF/PPTX 上传模板一键共享讲义，系统在后台自动完成分页解析与 Canvas 2D 高保真位图转换。同时，教师可导入标准的 Excel 作业模版，系统在解析时执行 \texttt{lessonNodeId} 节点自动映射与去重 Upsert 逻辑，避免了手动录入高频声韵母时的繁复操作。
  \item \textbf{学生端：即入即学的听读跟读流}：学生使用移动端（平板/手机）浏览器，允许麦克风设备授权后即可即入即学。在课件页面翻动时，学生点击发音按钮，系统通过 HTML5 Audio 组件拉取由双层语音缓存策略存储的标准普通话发音音频文件；点击跟读时，触发浏览器 MediaRecorder 接口进行 15 秒以内的语音采集，并在停止时自动打包为高保真的 WebM 封装 Opus 编码音频流，安全传输持久化。
  \item \textbf{物理硬件强制停用与隐私保护}：当学生关闭麦克风或点击退出直播课堂/退出作业面板时，系统前端将显式强制调用 \texttt{track.stop()} 方法对所有正在占用的媒体轨道执行物理断电与置空，彻底切断物理供电通道，防止发生麦克风悬挂占用引发的系统不稳定及恶意窃听隐患。同时，个人控制台为学生提供了“彻底物理删除”按钮，确保音频从 OSS 存储和数据库索引中被物理擦除。
\end{itemize}

\section{SaaS 级三层租户架构操作路径}
系统独立运行，并不直接与哈萨克斯坦高校复杂的教务系统（LMS）进行重型接口对接，这极大降低了数据打通和技术审查的壁垒。系统通过“合作机构 (Institution) --- 试点高校 (School) --- 试点班级 (Class)”的三层 SaaS 级逻辑租户树状关系进行人员的快速绑定与数据隔离。教师以试点学校下的班级为维度分发作业，学生以注册的班级身份登录获取大纲，使得操作流独立于学校网络环境之外，保障了跨国多学校、多租户并行使用的极低操作阻力。
