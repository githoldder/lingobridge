\chapter{测试范围}

LingoBridge 中文学习平台作为一个高可用的多语种教学交互系统，其测试范围涵盖了系统所有的前端交互界面、后端微服务组件以及数据库持久化层。前端部分的测试涵盖用户登录与身份认证、课程导航与多维度进度树、交互式富媒体课件播放器、在线课后作业提交系统、基于浏览器底层音频 API 的实时录音与回放组件、高并发低延迟的直播互动室，以及功能完备的管理后台系统。后端部分的测试则聚焦于 RESTful API 接口的规范性与安全性、数据持久化的一致性、Redis 缓存的穿透与雪崩防护、基于深度学习算法的语音智能打分算法精确度，以及高并发直播信令服务器与推拉流流媒体服务器的吞吐量与稳定性。

本项目测试活动的规划与执行，严格遵循国家标准 GB/T 8567—2006《计算机软件文档编制规范》\cite{gb8567} 以及国际通用标准 IEEE 829-2008《软件与系统测试文档标准》\cite{ieee829}，旨在通过规范化的文档记录与工程化的测试流程，确保软件在功能完整性、系统可靠性、性能瓶颈以及多平台兼容性等方面均达到生产级部署的要求。

在测试流程的管理上，项目采用严密且闭环的测试生命周期活动流，以实现测试过程的可控性与缺陷的快速收敛。整个测试生命周期活动的核心流程如图 \ref{fig:test-process-activity} 所示。在项目启动初期，测试团队根据系统设计规范与需求说明，编制全面的测试计划并明确测试策略；随后进入测试用例设计阶段，通过等价类划分、边界值分析以及场景法，编写高覆盖率的测试用例并组织多方评审；用例评审通过后，在专门的集成测试环境中部署被测系统，并在执行冒烟测试通过后正式推开全量系统测试；在测试执行期间，测试人员将利用缺陷跟踪系统对发现的问题进行全生命周期的状态管理与回归验证；当所有遗留缺陷均被妥善处理且系统达到预定收敛标准后，测试团队将对整个测试过程进行多维度数据统计与分析，最终输出客观反映系统质量的测试总结报告，为系统的灰度发布与正式上线提供强有力的决策依据。
