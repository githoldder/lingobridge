\chapter{缺陷管理}

\section{缺陷定义与分类标准}

缺陷管理是 LingoBridge 中文学习平台软件测试生命周期中的核心控制环节，通过对测试过程中发现的问题进行量化分类与分级，能够客观地评估当前软件版本的质量状态并指导研发团队有序进行缺陷修复。缺陷根据其表现形式可细分为功能性缺陷、接口通信缺陷、性能瓶颈缺陷、界面排版缺陷、安全越权缺陷以及多端兼容性缺陷等大类。为了实现对缺陷的高效响应，测试团队针对缺陷的严重程度制定了严密的四级分级体系。缺陷严重程度的分级定义以及相应的开发团队响应时限要求如表 \ref{tab:defect-severity} 所示。在实际执行中，致命缺陷主要涉及服务宕机、数据库锁死或主干身份验证服务崩溃；严重缺陷指核心功能失效且无法规避；一般缺陷代表非主干业务逻辑错误或系统提示不够准确；微小缺陷则多为界面排版或学术拼写方面的瑕疵。

\section{缺陷生命周期与流转流程}

为了保障所有发现的缺陷均能得到及时、透明且可闭环的跟踪处理，LingoBridge 项目定义了规范化的缺陷生命周期流转规程。缺陷生命周期的状态变迁与流转路径如图 \ref{fig:defect-lifecycle-state} 所示。缺陷流转的起点是测试人员在执行用例时发现系统异常，并在缺陷管理平台中创建新缺陷单，此时缺陷状态被标记为新建；经过测试组长或产品经理审查确认后，缺陷单被指派给负责该模块的具体开发工程师，缺陷状态变更为已指派；开发人员在本地重现并定位原因后，提交修复代码，并将缺陷单状态修改为已解决，同时附带详细的修复说明；测试人员在接收到构建的新版本后，对照缺陷描述进行严格的回归测试与边界验证，若确认问题已完全修复，则将缺陷状态修改为已关闭，实现缺陷生命周期的终结；若在回归测试中发现问题依然存在或引入了新的关联缺陷，则测试人员会拒绝关闭，并将缺陷单状态置为重新打开，打回开发人员重新处理，如此往复，直至整个系统质量达到发布收敛标准。
