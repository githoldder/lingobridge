\chapter{测试策略}

\section{单元测试}

单元测试作为 LingoBridge 系统质量保障体系的第一道防线，旨在对系统的最小可独立测试单元（如类、函数与方法）进行静态分析与动态运行验证。在后端微服务的开发与测试过程中，团队主要依托 Go 语言原生的测试工具链以及 Node.js 环境下的 Jest 测试框架，对用户身份认证逻辑、学习进度计算算法、语音智能评分数值映射关系以及数据库核心事务逻辑进行全方位的白盒测试。单元测试设计严格遵循测试驱动开发与边界值测试原则，确保各模块分支覆盖率达到百分之九十以上。为了保证单元测试的执行速度以及对本地环境的低依赖性，开发与测试团队利用 Golang 的 mock 机制和 Jest 的模拟模块，对外部云存储服务、第三方智能语音交互接口以及数据库连接进行轻量级打桩隔离，从而实现单元测试的完全闭环，其执行时间可控制在毫秒级别，以便于持续集成系统在每次代码提交时进行高频自动化触发与回归。

\section{集成测试}

集成测试阶段重点关注 LingoBridge 系统内部各微服务模块以及前后端系统在接口通信、数据交互和协同工作时的行为一致性。为了防范服务间调用协议不兼容导致的缺陷，集成测试采用自底向上与基于核心业务路径相结合的演进策略。针对前后端系统的接口联调，测试团队引入了基于 OpenAPI 规范的契约测试，通过自动化工具生成接口测试套件，定时对后端的 RESTful API 接口进行全量参数扫描，重点验证接口在参数边界值、非法权限越权访问、Token 过期失效以及高并发调用场景下的鲁棒性。在此基础上，集成测试还着力覆盖多数据库联合事务的一致性、Redis 缓存的冷热数据转换以及消息队列在网络抖动时的消息重试与幂等性消费机制，从而确保系统各组件能够在中等负载下实现无缝且安全的集成协同。

\section{系统测试}

系统测试是对 LingoBridge 系统的完整功能、系统吞吐、网络延迟以及端到端业务闭环进行的全面黑盒验证。本阶段的核心任务是站在用户的视角，对整个平台的业务功能链进行覆盖性测试。为了实现这一目标并大幅度提升回归测试效率，团队基于 Playwright 自动化测试框架构建了端到端自动化测试套件。该自动化测试方案主要针对系统的主干用户链路开展深度覆盖，测试套件将全自动执行诸如新用户在线注册、个人账号登录、多媒体课程目录交互式检索、富媒体课件在线播放与进度上报、浏览器底层音频组件调用以及在线作业语音录制与 AI 智能打分展示等关键行为。基于 Playwright 的端到端自动化测试流如图 \ref{fig:e2e-test-flow} 所示。测试执行引擎会自动拉起无头浏览器实例，模拟真实用户的各类键鼠操作与音频输入，并在测试完成后对关键渲染元素和响应断言进行校验，若发生异常则自动捕获现场页面截图与性能日志，从而在每次软件版本打包前完成对主干流程的高效回归验证。

\section{测试环境与部署}

为了确保所有测试活动的结果具备高度的真实性与生产环境参考价值，LingoBridge 的测试工作必须在一个高度隔离且拓扑结构完全一致的测试环境中开展。测试环境的设计秉持与生产环境等比例隔离映射的原则，以防范测试数据的溢出与环境污染。测试环境的部署架构如图 \ref{fig:test-environment-deployment} 所示。测试团队利用 Docker 容器化技术与 Kubernetes 服务编排工具，在物理测试服务器上搭建起一套自闭环的测试沙盒。该环境完整部署了基于 Vite 容器构建的前端服务、基于 Go 开发的后端 API 核心服务集群、提供深度学习语音打分能力的智能评分微服务实例，以及专为测试环境服务的 PostgreSQL 关系型数据库集群和 Redis 高速缓存容器。测试环境不仅通过子网段划分实现与外网及生产环境的物理隔离，更通过自动化脚本在每次测试迭代前对测试数据库进行结构同步与脱敏数据初始化，从而保证了测试环境的纯净性与可重复部署性。
