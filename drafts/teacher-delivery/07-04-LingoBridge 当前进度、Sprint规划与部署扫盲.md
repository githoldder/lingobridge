## 二、眼前最重要的目标

老师真正关心的是四句话：

1. **能不能公网打开？**  
   要给一个能访问的网址或公网 IP。

2. **老师和学生怎么用？**  
   要有老师版、学生版操作手册。

3. **能不能组织学生采集数据？**  
   要有采集流程、录音标准、问题反馈表。

4. **需要多少钱？**  
   要有当前试用预算和后续正式运行预算。

所以接下来的 Sprint 不以“继续调研更多技术”为主，而以“演示闭环稳定、公网可访问、材料可交付”为主。

## 四、完整演示脚本

演示时建议按以下顺序：

1. 打开公网地址。
2. 说明：这是 LingoBridge MVP，可支持中文学习数据采集。
3. 教师账号登录。
4. 进入课程管理，展示课程。
5. 修改课程说明并保存。
6. 上传 PDF/PPTX 课件。
7. 上传 Excel 练习数据。
8. 创建课时和 Live，展示课堂入口。
9. 退出教师账号。
10. 学生账号登录。
11. 进入指定课程。
12. 点击中文朗读。
13. 点击录音，授权麦克风。
14. 录一条中文跟读。
15. 播放录音，删除错误录音，再重新录制。
16. 退出学生账号。
17. 管理员账号登录。
18. 查看课件、Excel 导入、录音、学习进度。
19. 总结：当前已能支持公网试用和数据采集。

## 五、公网部署扫盲

### 1. 什么是服务器？

服务器就是一台 24 小时在线的远程电脑。我们把 LingoBridge 放到这台电脑上，老师和学生就可以通过网址访问。

### 2. 什么是公网 IP？

公网 IP 是服务器在互联网上的地址。没有域名时，可以直接用：

```text
http://服务器公网IP/
```

优点是立刻能用；缺点是不够好记。

### 3. 什么是域名？

域名是更容易记的网址，例如：

```text
https://lingobridge-demo.com
```

域名需要购买并解析到服务器公网 IP。

### 4. 什么是 HTTPS？

HTTPS 是加密访问。正式给学生使用时建议配置 HTTPS，因为浏览器对麦克风、摄像头、录屏等能力在安全环境下支持更稳定。

### 5. 什么是安全组？

安全组是云服务器的防火墙。即使程序启动了，如果安全组没有开放端口，外网仍然打不开。

常见端口：

| 端口 | 用途 |
|---:|---|
| 80 | HTTP 网站访问 |
| 443 | HTTPS 网站访问 |
| 3001 | 后端 API 测试端口 |
| 4174 | Vite preview 测试端口 |

正式部署建议用 80/443 对外，后端 API 由 Nginx 转发，不直接暴露太多端口。

### 6. 什么是前端和后端？

| 名称 | 通俗解释 | 本项目例子 |
|---|---|---|
| 前端 | 老师和学生看到的网页界面 | 登录页、课程页、录音按钮 |
| 后端 | 处理数据和文件的服务 | 登录、上传、录音保存 |
| 数据库/数据文件 | 保存用户、课程、记录 | 当前 MVP 用本地 JSON 数据 |
| 存储 | 保存课件、录音、录播文件 | 当前 MVP 用服务器目录 |

### 7. 什么是 PM2？

PM2 是 Node.js 服务守护工具。服务器重启或程序崩溃后，它可以帮助服务重新拉起。MVP 部署用 PM2 比较轻量。

### 8. 什么是 Docker？

Docker 是把应用和运行环境打包的工具。正式部署更稳，但配置成本更高。当前可以先用 PM2 快速上线，后续再 Docker 化。

## 七、上线前 P0 风险

| 风险 | 影响 | 处理优先级 |
|---|---|---|
| 公网打不开 | 老师无法试用 | P0 |
| 登录失败 | 无法演示 | P0 |
| API 地址配置错误 | 前端页面能开但数据失败 | P0 |
| 麦克风权限失败 | 无法采集核心数据 | P0 |
| 上传文件失败 | 老师无法导入材料 | P0 |
| 数据未备份 | 采集数据可能丢失 | P1 |
| 手机兼容问题 | 学生使用受阻 | P1 |
