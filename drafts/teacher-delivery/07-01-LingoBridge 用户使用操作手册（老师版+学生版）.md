# LingoBridge 用户使用操作手册（老师版+学生版）

> 适用阶段：MVP 公网试用与数据采集  
> 适用对象：文科院中文教师、参与试用的留学生、项目演示人员  
> 当前目标：让老师能上传课件、组织学生练习，让学生能听中文朗读、录音提交，并能回看基础记录。

## 二、亮点闭环一：留学生“3 卡槽”高保真跟读自测

这套闭环专门解决比赛评委最关心的问题：语言学习产品如何拿到真实、清晰、可用的口语数据。传统软件“录完即交”，学生一紧张就小声读，宿舍噪音一进来就产生废片，后台虽然有提交量，却没有稳定的数据质量。

LingoBridge 把提交前的压力拆掉：每道跟读题先进入本地 3 卡槽临时缓存，学生可以连录三次、试听比较，只把最满意的一条作为最终作业证据提交。系统随后给出流畅度、声调、发音完整度等 ASR 风格评分，让“采集、筛选、提交、反馈”变成同一个防挫败闭环。

![首页今日待完成作业卡片](attachments/11-01-student-dashboard-homework-card.png)

### 1. 首页预警：把作业入口前置到学生工作台

学生登录后第一眼看到“今日待完成作业”卡片，不需要在课程、课件、日程之间来回找。这个入口把作业标题、任务数量和行动按钮压缩在同一张卡片里，降低留学生因界面陌生而放弃提交的概率。

商业价值：入口越短，真实采集率越高。对试点学校而言，这意味着老师不再依赖微信群催交，也能快速判断今天是否有足够样本进入后台。

![日程中的课时作业入口](attachments/11-02-student-schedule-homework.png)

### 2. 日程承接：从课时上下文进入作业

日程页把当天课时与“进入作业”动作绑定在一起。学生知道这不是孤立练习，而是某一节课后的明确任务，老师也能把课时、Excel 作业、录音数据和提交状态串起来追踪。

技术上，日程入口会携带 `courseId` 与 `lessonNodeId`，为后续 Focus Locator 自动定位当前课时作业做准备。

![完整通关地铁大路](attachments/11-03-student-homework-pathway.png)

### 3. 完整路径：用通关地图替代孤立题单

作业页按 `LessonNodes` 排序渲染成一条“地铁大路”，学生可以清楚看到自己处在第几课、前后还有哪些关卡。虚线路径和节点状态让作业从“待办压力”变成“通关进度”。

交互哲学：留学生发音不自信时，最怕看见一长串冰冷题目。地图式进度把任务拆成可完成的小段，降低开始录音的心理门槛。

![Focus Locator 自动弹出作业面板](attachments/11-04-student-homework-modal.png)

### 4. Focus Locator：从课时直达当前作业

学生从日程进入作业后，系统不要求再次点击地图节点，而是在完整大路背景上自动弹出当前课时面板。这个细节对演示非常关键：它证明平台不只是有作业页，而是能把“今天这节课的作业”精准定位出来。

技术上，这依赖日程入口传递的 `lessonNodeId`，作业页加载任务与学习记录后自动匹配对应课时，并展开 Modal。

![3 卡槽跟读自测与 AI 评分](attachments/11-05-student-homework-3slot.png)

### 5. 3 卡槽自测：提交前先筛掉废片

这是 LingoBridge 的核心差异化。学生每道题最多录 3 次，每次录完先进入本地卡槽，立刻生成评分和可视化反馈。学生可以比较三次结果，确认哪一次音量更稳定、声调更准、句子更完整。

它解决的不是“能不能录音”，而是“交上来的录音值不值得训练和分析”。对后台数据资产来说，这一步相当于把噪音、怯声、残缺朗读提前过滤掉，把原本需要人工清洗的废片率压低。

![提交后通关节点变绿](attachments/11-06-student-homework-completed-path.png)

### 6. 提交通关：反馈回到路径状态

当同一课时的子任务全部完成并提交后，学生回到地图，对应关卡变成绿色对勾。这个状态不是装饰，而是学习记录、录音提交和课时节点完成度共同写入后的结果。

评委可以从这一张图直接理解闭环：入口提醒、课时定位、任务完成、录音筛选、AI 反馈、状态回写，全部发生在同一个学生体验里。

## 三、MVP 可以演示什么

LingoBridge 当前不是完整教务系统，也不是正式线上课程平台。MVP 阶段重点验证一个教学闭环：

```text
老师上传课件/练习数据
-> 学生登录查看课程
-> 学生点击中文朗读
-> 学生录制跟读音频
-> 系统保存录音和学习记录
-> 老师/管理员查看采集结果
```

当前可演示功能：

| 模块 | 可演示内容 |
|---|---|
| 登录与权限 | 老师、学生、管理员三类账号登录；游客点击受保护功能时提示登录 |
| 课程管理 | 老师查看课程、创建/修改课程信息 |
| 课件上传 | 老师上传 PDF/PPTX 课件，系统生成课程页面 |
| Excel 导入 | 老师上传练习/词汇 Excel，生成学生练习任务 |
| 学生练习 | 学生进入课程，播放中文 TTS，使用 3 卡槽录制跟读音频并提交 |
| 录音管理 | 学生播放、下载、删除自己的录音 |
| 自测评分 | 学生录音确认后展示流畅度、声调、发音、完整度评分反馈 |
| Live/录课演示 | 老师创建课时与 live，演示本地投屏/多媒体课件模式 |
| 后台管理 | 管理员查看用户、课件、Excel 导入、live、录音、学习进度 |

暂不作为 MVP 承诺的功能：

| 功能 | 说明 |
|---|---|
| 云端深度语音评测 | 当前已具备学生端自测评分展示；更深的云端 ASR 诊断可在试点数据稳定后接入 |
| 实时双语字幕 | 规划为后续版本，不影响当前数据采集 |
| 腾讯会议/Teams 深度集成 | 当前先用本系统内 live/录制能力演示 |
| 多学校 SaaS | 当前为单项目、单团队试用 |

## 四、老师版操作手册

### 1. 登录

1. 打开公网地址。
2. 点击登录入口。
3. 输入教师账号 `teacher@test.com` 和密码 `Test@123456`。
4. 登录后进入教师工作台。

验收点：页面右上角或侧边栏显示教师身份；能看到课程、课件、日程、作业、live 等教师模块。

### 2. 创建或修改课程

1. 进入“课程管理”。
2. 点击课程列表中的现有课程，例如“第三课：自我介绍”。
3. 在课程详情页修改课程名称、描述或状态。
4. 保存后刷新页面，确认修改仍然存在。

建议演示话术：

> 老师可以把每一课作为一个课程单元管理，后续课件、练习、录音和学生学习记录都会挂在课程下面。

### 3. 上传课件

1. 进入教师课程详情页。
2. 切换到“课件”或“Courseware”相关标签。
3. 点击上传按钮。
4. 选择 `.pdf` 或 `.pptx` 文件。
5. 等待上传完成。
6. 上传后进入学生端或课堂页，检查课件页面是否可浏览。

文件要求：

| 类型 | 建议规格 | 说明 |
|---|---|---|
| PDF | 单文件不超过 50MB | MVP 阶段最稳定，建议优先使用 PPT 导出的 PDF |
| PPTX | 单文件不超过 50MB | 可上传，但复杂动画和嵌入媒体不作为 MVP 保证 |
| Excel | 建议不超过 1MB | 用于导入词汇、跟读句子和练习任务 |

### 4. 上传 Excel 练习数据

1. 进入课程详情页的作业/练习相关区域。
2. 点击“上传 Excel”或“导入作业”。
3. 选择按模板整理好的 Excel 文件。
4. 上传完成后检查导入结果：任务数、词汇数、错误提示。
5. 用学生账号登录，确认学生端能看到对应练习。

Excel 建议字段：

| 字段 | 示例 | 是否必填 |
|---|---|---|
| `course_code` | `TEST-01` | 建议填写 |
| `unit` | `1` | 是 |
| `lesson` | `1` | 是 |
| `task_id` | `TEST-01-L01-001` | 是 |
| `task_type` | `pronunciation` / `vocabulary` | 是 |
| `zh_text` | `大家好，我叫阿合买提。` | 是 |
| `pinyin` | `Dajia hao, wo jiao Ahemaiti.` | 建议填写 |
| `translation_ru` | `Здравствуйте...` | 建议填写 |
| `translation_kk` | `Сәлеметсіз бе...` | 建议填写 |
| `publish_to_homework` | `TRUE` | 是 |
| `publish_to_vocab` | `FALSE` | 是 |

### 5. 创建课时与 Live 演示

1. 进入课程详情页。
2. 打开“日程/课时”区域，创建一个课时，例如“第三课课后跟读采集”。
3. 打开“Live”标签。
4. 选择刚创建的课时。
5. 点击创建 Live。
6. 点击进入课堂。
7. 演示两种模式：本地屏幕共享、多媒体 PDF/课件模式。

注意：

- 直播能力当前主要用于演示课堂互动和录制，不建议一开始就组织大规模同步直播。
- 采集学生录音数据时，优先使用“课后练习+录音提交”的方式，稳定性更高。

### 6. 查看学生数据

老师或管理员可以查看：

| 数据 | 位置 |
|---|---|
| 学生录音 | 学生端录音列表、管理员录播/录音记录 |
| 学习记录 | 管理员后台学习进度 |
| 上传课件 | 管理员后台课件管理 |
| Excel 导入 | 管理员后台 Excel 导入 |
| Live 记录 | 管理员后台 Live 管理 |

### 7. 退出登录

1. 点击头像或侧边栏底部账号区域。
2. 点击退出登录。
3. 回到首页或登录页。

## 五、学生版操作手册

### 1. 登录

1. 打开公网地址。
2. 点击登录。
3. 输入学生账号 `student_a@test.com` 和密码 `Test@123456`。
4. 登录后进入学生首页。

### 2. 进入课程

1. 点击“课程”或“Courses”。
2. 选择老师指定的课程，例如“第三课：自我介绍”。
3. 进入课程学习页。
4. 浏览老师上传的课件页面。

### 3. 使用中文朗读

1. 在课程页找到中文句子或练习任务。
2. 点击朗读按钮。
3. 听系统朗读。
4. 跟读前可以多听几遍。

说明：如果云端 TTS 没有配置，系统会使用浏览器自带朗读能力。不同电脑和手机的声音可能略有差异，不影响录音采集。

### 4. 录制跟读音频

1. 点击录音按钮。
2. 浏览器弹出麦克风授权时，选择允许。
3. 按老师要求朗读中文句子。
4. 点击停止录音。
5. 点击确认，把本次录音放入 3 卡槽自测区。
6. 重复录制最多 3 次，比较评分和试听结果。
7. 选择最满意的一次完成提交。

录音要求：

| 项目 | 要求 |
|---|---|
| 环境 | 尽量安静，避免多人同时说话 |
| 时长 | 每条 5-30 秒即可，短句跟读可控制在 10 秒内 |
| 内容 | 按页面中文句子或老师指定句子朗读 |
| 设备 | 手机耳机麦克风或电脑麦克风均可 |

### 5. 管理录音

学生可以对自己的录音执行：

| 操作 | 用途 |
|---|---|
| 播放 | 检查录音是否清楚 |
| 下载 | 保存本地备份 |
| 删除 | 删除错误录音后重新录制 |

建议：如果录音太小、没有声音、背景噪音太大，请删除后重新录。

### 6. 切换语言

系统支持中文、俄语、哈萨克语界面切换。学生可根据自己的阅读习惯选择界面语言。课程里的中文学习内容不会因为界面切换而改变。

### 7. 退出登录

完成采集后，点击账号区域退出登录。公共电脑上务必退出。

## 六、推荐演示流程（给老师看）

一次完整演示建议控制在 8-12 分钟：

| 顺序 | 操作 | 预计时间 |
|---|---|---|
| 1 | 打开公网地址，说明这是可访问的 MVP 测试环境 | 30 秒 |
| 2 | 用教师账号登录 | 30 秒 |
| 3 | 查看课程，修改课程说明 | 1 分钟 |
| 4 | 上传 PDF/PPTX 课件 | 1-2 分钟 |
| 5 | 上传 Excel 练习数据 | 1 分钟 |
| 6 | 创建课时和 Live，展示课堂入口 | 1-2 分钟 |
| 7 | 退出教师账号，切换学生账号 | 30 秒 |
| 8 | 学生进入课程，听 TTS，录音并播放 | 2 分钟 |
| 9 | 学生删除错误录音并重新录制 | 1 分钟 |
| 10 | 切换管理员账号，查看课件/录音/学习进度 | 1-2 分钟 |

演示结论话术：

> 当前版本已经可以支持小范围公网试用：老师上传课件和练习，学生登录后完成中文朗读、跟读录音和数据提交。下一阶段重点不是堆新功能，而是组织真实学生录入数据、收集问题、修复体验细节。
