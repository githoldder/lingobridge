# Sprint 09 — LingoBridge v2.0 AI 学习分析与个性化推荐里程碑

Last Updated: 2026-06-01 10:46

## Object
将 LingoBridge 从 MVP 教学闭环升级为 v2.0 数据驱动学习平台，在学生端、教师端与 Admin 端显式呈现“数据分析”和“智能推荐”能力，并形成可用于《人工智能》课程大作业的 Python 机器学习子系统、演示流程与报告证据链。

> Agent 执行必须以 `prds/json/sprint09-prd-260601-v2.0.json` 为详细设计拆解文件；该 JSON 已包含现有 MVP 代码边界、可改/禁改文件、数据收集/清洗/导出步骤、CLI 验证命令和 `.agent/` 规则引用。

## Version Positioning
版本号：v2.0

选题名称：《基于机器学习的国际中文学习行为分析与个性化练习推荐系统》

v2.0 不改变 LingoBridge 的核心业务边界，仍围绕国际中文教学、课程学习、课堂同步、作业练习与录音提交展开；新增能力集中在学习行为数据采集、学习画像、风险分层、个性化练习推荐、教师教学干预建议和 Admin 数据中台。该版本的 AI 能力以可解释的传统机器学习优先，允许后续扩展深度学习或语音特征模型。

## Key-Results
- KR-1: 学生首页新增 AI 学习画像模块，展示学习状态、薄弱点、推荐练习和下一步行动建议。
- KR-2: 学生个人排名页新增数据分析视图，展示个人排名、进步趋势、同伴对比和推荐目标。
- KR-3: 教师学生总览页新增 AI 班级洞察模块，展示风险学生、共性薄弱点、作业完成质量和干预建议。
- KR-4: Admin 端升级为数据中台，支持全局学习数据、课程数据、活跃度、推荐系统运行状态和演示监控。
- KR-5: 新增 Python 机器学习服务或离线分析模块，完成数据预处理、特征工程、模型训练、预测、推荐和结果导出。
- KR-6: 建立 v2.0 演示数据集与测试用例，覆盖学生端、教师端、Admin 端三条演示路径。
- KR-7: 形成课程大作业报告素材，包括背景综述、总体方案、可行性分析、数据预处理、算法设计、交互设计、测试结果和成员分工。

## Scope
### In Scope
- 学生端首页 AI 学习助手卡片。
- 学生端个人排名页的数据分析与推荐目标。
- 教师端学生总览页的班级学习洞察。
- Admin 端数据中台首页与监控看板。
- Python ML 模块，包括样例数据、预处理、模型、推荐逻辑和评估脚本。
- 前后端接口契约，用于读取分析结果、推荐结果和聚合指标。
- 演示数据种子脚本与答辩演示剧本。

### Out of Scope
- 云端 ASR 自动评分闭环。
- 大规模实时深度学习训练平台。
- 复杂多模态语音情感识别。
- 商业化计费、学校级多租户结算和正式隐私合规认证。
- 将推荐结果作为强制教学决策；v2.0 只提供辅助建议。

## Product Narrative
v2.0 的核心叙事是“学习行为数据进入平台，平台反向服务教学”。学生在课程学习、课堂互动、作业提交、录音练习和排名竞争中产生行为数据；系统通过预处理和特征工程生成学习画像，再利用分类、聚类或推荐算法识别薄弱点、预测学习风险，并把结果分别送回学生端、教师端和 Admin 端。学生看到的是个性化下一步练习，教师看到的是班级管理和干预优先级，Admin 看到的是应用级平台运行与演示数据中台。

## Tasks

### S9-T00: 执行环境、进程、Git 与记忆治理
- Python 环境同时支持 uv 与 Anaconda；默认推荐 uv 管理轻量依赖，课堂本地演示可用 conda 环境兜底。
- 本地演示优先使用 PM2 管理前后端进程，避免 Vite、tsx、backend server 残留僵尸进程。
- 严格遵循 Ralph 机制，每个小 feature 或可独立回滚的 step 都要本地 commit 留痕。
- Sprint 完成后先 review，确认无误后再 push。
- 短期踩坑和执行记忆写入 `context/context.txt`，长期规则经确认后沉淀到 `.agent/rules` 或 `prds/current`。
- 验收：后续 agent 开工前能从 JSON PRD 直接获得环境、进程、提交、review、context/memory 的执行纪律。

### S9-T01: v2.0 数据指标体系与埋点口径
- 定义学生学习行为事件：登录、课程进入、课件浏览、课堂参与、作业提交、录音次数、重试次数、完成时间、排名变化。
- 定义核心分析指标：完成率、活跃度、练习稳定性、薄弱知识点、进步速度、风险等级、推荐命中状态。
- 输出前后端共享的数据字典与字段命名规范。
- 验收：`docs` 或 `prds/current` 中有可追踪的数据指标说明；JSON PRD 中字段可被后续 agent 直接使用。

### S9-T02: Python ML 分析与推荐模块
- 建立 `analysis/ai-learning-recommendation/` 或同级目录，放置数据集、预处理脚本、模型脚本和导出结果。
- 实现基础特征工程：完成率、平均重试次数、最近活跃天数、作业得分、录音覆盖率、知识点错误率。
- 实现至少两类算法：学习风险分类或聚类、个性化练习推荐。
- 输出 JSON 结果文件或 API 可读结构，供前端页面消费。
- 验收：脚本可用 Python 运行，能基于样例数据生成学生画像、风险等级和推荐练习。

### S9-T03: 学生首页 AI 学习助手
- 在学生首页加入 AI 学习助手模块。
- 展示学习状态、今日推荐任务、薄弱知识点、预计提升方向。
- 提供可点击的推荐练习入口，跳转到课程、作业或练习详情。
- 验收：学生可在首页直接看到“数据分析”和“智能推荐”的明确表达。

### S9-T04: 学生个人排名页数据分析升级
- 在个人排名页展示个人排名、历史趋势、班级分位、连续练习天数和进步幅度。
- 增加“超过下一名需要完成什么”的推荐目标。
- 增加同伴对比但避免羞辱式文案，强调自我进步。
- 验收：排名页既有竞争感，又能体现 AI 推荐的行动建议。

### S9-T05: 教师学生总览 AI 班级洞察
- 在教师学生总览页加入班级分析模块。
- 展示风险学生列表、共性薄弱知识点、作业完成质量、录音练习覆盖率。
- 为教师生成可解释干预建议，例如补充拼音练习、提醒未提交作业、安排复习任务。
- 验收：教师能一眼判断“谁需要关注、为什么、下一步做什么”。

### S9-T06: Admin 数据中台
- 将 Admin 首页升级为全局数据中台。
- 展示平台总览：学生数、教师数、课程数、作业数、录音数、推荐任务数、活跃趋势。
- 展示 AI 模块运行状态：数据更新时间、模型版本、预测样本数、异常数据数。
- 支持演示模式，能稳定展示完整平台级数据闭环。
- 验收：Admin 端可以用于答辩时说明 LingoBridge 是应用级教学平台，不只是单页面 demo。

### S9-T07: 接口与数据流打通
- 设计前端读取分析结果的 API 或本地 mock 数据结构。
- 保持 MVP 原有功能不回退。
- 明确 Python 分析模块与 TypeScript 应用之间的边界：离线 JSON、轻量 FastAPI 服务或后端聚合接口三选一。
- 验收：学生端、教师端、Admin 端都能读取同一套分析结果或 mock 结果。

### S9-T08: v2.0 演示数据与测试
- 构造至少 1 个班级、1 名教师、1 名 Admin、8-12 名学生、3-5 个课程/作业的演示数据。
- 覆盖高活跃学生、低活跃学生、薄弱知识点明显学生、进步型学生等典型画像。
- 为三端页面写 smoke test 或手工验收清单。
- 验收：答辩演示可稳定复现“数据产生 -> AI 分析 -> 推荐展示 -> 教师干预 -> Admin 监控”。

### S9-T09: 人工智能大作业报告素材沉淀
- 建立报告目录或文档草稿，覆盖指导书要求的背景综述、设计指标、总体方案、可行性分析、数据预处理、功能设计、算法设计、交互设计、编码实现、测试结果和分工说明。
- 保留关键截图、模型评估结果、数据样例和运行输出。
- 验收：报告素材能直接转写为课程大作业报告，不需要重新倒推项目逻辑。

## Acceptance Criteria
- v2.0 PRD 的每个任务都能映射到前端页面、后端/数据接口或 Python 分析模块。
- 学生端、教师端、Admin 端均出现可解释的数据分析结果。
- 推荐逻辑可追溯到真实或样例学习行为数据，不是纯静态文案。
- Python 代码可独立运行并输出模型结果。
- 不破坏现有 MVP 登录、课程、课堂、作业和录音流程。
- 产物能支撑《人工智能》大作业“代码实现 30%、答辩演示 20%、文档报告 30%”的评分结构。
