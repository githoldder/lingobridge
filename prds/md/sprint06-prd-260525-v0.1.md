## 1. Objective

Solidify all core deployment runbooks, workflow boundaries, and DevOps automation scripts into standard handoff assets. Ensure LingoBridge remains fully demo-ready, highly reproducible by prospective developers, and verified by standard CI routines without altering any business logic.

---

## 2. Key-Results (Milestones)

* **KR1 (Local Demo Stability)**: Deliver an actionable runbook enabling full camera, microphone, and direct file upload testing on `localhost`.
* **KR2 (Vercel Boundary Clarity)**: Document strict Vercel HTTPS reverse-proxy and rewrite limits (especially for 20MB+ courseware).
* **KR3 (Tencent Direct Operations)**: Ensure public IP backend direct access is fully documented with TAT patch scripts and rollback procedures.
* **KR4 (DevOps Automation)**: Ship a fully functioning GitHub Actions pipeline checking lint, build, and backend test suites automatically.
* **KR5 (Agent-Ops Governance)**: Establish strict workflows for product prototyping and alternative technical research to govern future agents.
